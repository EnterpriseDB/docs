Released: 17 June 2026

The edbctl 1.7.0 release includes new features, improvements, bug fixes, and deprecations to enhance the functionality and performance of the Hybrid Manager CLI tool.

| Type              | Description                                                                                                                                                                                                              |
|-------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Feature           | Added `edbctl chat` command for interacting with the AI assistant from the CLI, including single-message prompts, interactive sessions, and conversation resume.                                                          |
| Feature           | Added AI model management commands for listing models, editing model metadata, and listing available AI model functions.                                                                                                  |
| Feature           | Added read-only connection configuration for PSR and PGD clusters.                                                                                                                                                       |
| Feature           | Added `edbctl disaster-recovery` command group with `create-restore-objectstores` and `prepare-cluster-configs` subcommands for managing disaster recovery operations on HM-internal clusters.                            |
| Feature           | Added `edbctl image cleanup-registry` command to remove unused images from the HM registry and reclaim storage after an upgrade.                                                                                          |
| Feature           | Added `edbctl hybrid-manager upgrade-operator` command to prepare Hybrid Manager environments for migration from bootstrap-based installation to operator-based installation.                                              |
| Improvement       | Improved cluster edit previews so users can review more accurate proposed changes before applying updates.                                                                                                                |
| Improvement       | Improved Hybrid Manager registry cleanup reliability by refreshing the registry after garbage collection.                                                                                                                 |
| Improvement       | Improved `create-install-secrets` behaviour to preserve existing portal administrator identities during upgrades and credential rotations.                                                                                 |
| Improvement       | Added support for retrying Hybrid Manager components even when a component is already in a successful state.                                                                                                              |
| Improvement       | Updated `latest-lts` handling for Hybrid Manager 1.4.                                                                                                                                                                    |
| Improvement       | Added support for `dbaas` as an accepted scenario placeholder when creating install secrets.                                                                                                                              |
| Bug&nbsp;fix      | Fixed non-interactive secret creation guidance so static administrator user examples include all required environment variables.                                                                                           |
| Bug&nbsp;fix      | Fixed cluster edit diff behaviour so root-level and detail-level changes are reported more accurately.                                                                                                                    |
| Bug&nbsp;fix      | Fixed registry cleanup behaviour to avoid stale registry state after garbage collection.                                                                                                                                  |
| Deprecation       | Disabled create and restore operations for deprecated AHA cluster types in Hybrid Manager 1.4.                                                                                                                            |
| Deprecation       | Disabled create and restore operations for deprecated Analytics cluster types in Hybrid Manager 1.4.                                                                                                                      |
