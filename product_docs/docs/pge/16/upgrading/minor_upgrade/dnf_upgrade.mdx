---
title: "Minor EDB Postgres Extended Server upgrade with dnf"
description: "Perform a minor version upgrade of EDB Postgres Extended Server with the dnf package manager"
---

If you used `yum` or `dnf` to install EDB Postgres Extended Server, use `dnf` to perform a minor version upgrade of the packages. 

1.  To list the package updates available for your system, open a command line, assume root privileges, and enter the command:

    ```shell
    sudo dnf check-update <package_name>
    ```

    For example, if you want to upgrade to the latest version of EDB Postgres Extended Server 16, run: 

    ```shell
    sudo dnf check-update edb-postgresextended16
    ```

    !!!Note
       You can include wildcard values in the search term. For example, if you are looking for EDB Packages, you could run `sudo dnf check-update edb-*`. For more information about using `dnf` commands and options, enter `dnf --help` at the command line.

1.  Once you have figured the name and version of the package you want to install, use `dnf update` to install the package:

    ```shell
    sudo dnf update edb-postgresextended16
    __OUTPUT__
    ================================================================================
    Package                        Arch   Version    Repository               Size
    ================================================================================
    Upgrading:
     edb-postgresextended16         x86_64 16.3-2.el9 enterprisedb-enterprise 1.7 M
     edb-postgresextended16-contrib x86_64 16.3-2.el9 enterprisedb-enterprise 724 k
     edb-postgresextended16-libs    x86_64 16.3-2.el9 enterprisedb-enterprise 330 k
     edb-postgresextended16-server  x86_64 16.3-2.el9 enterprisedb-enterprise 6.8 M

    Transaction Summary
    ================================================================================
    Upgrade  4 Packages
    
    Total download size: 9.5 M
    Is this ok [y/N]
    ```

1.  Confirm with `y`. The output displays an overview of all performed processes, where you can see the packages that have been upgraded:

    ```shell
    y
    __OUTPUT__
    edb-postgresextended16-16.3-2.el9.x86_64
    edb-postgresextended16-contrib-16.3-2.el9.x86_64
    edb-postgresextended16-libs-16.3-2.el9.x86_64
    edb-postgresextended16-server-16.3-2.el9.x86_64
    ```

1.  Restart the server to finalize the upgrade. Replace <server_name> with the name of the server or servers you are upgrading:

    ```shell
    /usr/edb/pge16/bin/pg_ctl -D /var/lib/edb-pge/16/<server_name> restart
    ```

1.  Verify the expected database version is running by connecting to psql: 

    ```shell
    psql
    ```
    
    And check the server version: 

    ```psql
    SHOW server_version;
    __OUTPUT__
                   server_version
    --------------------------------------------
     16.3 (EDB Postgres Extended Server 16.3.0)
    ```
