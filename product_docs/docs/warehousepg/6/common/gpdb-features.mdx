Source code for the WarehousePG documentation is shared between WarehousePG and
WarehousePG. Some features of WarehousePG are not present in the
open source WarehousePG code but are described in the WarehousePG documentation. 

The term "WarehousePG" refers to VMware's commercial, supported product offering that is 
based on open source WarehousePG.  "WarehousePG" is used in this documentation when 
describing features that are available only in the commercial offering. The term "WarehousePG
Database" refers to features available in open source WarehousePG (and in the commercial VMware 
WarehousePG product). Commercial support for "WarehousePG" is offered by VMware as "Open Source WarehousePG by VMware."

The following is a list of WarehousePG features that are not provided or supported in open
source WarehousePG, but may be documented or referenced in this documentation.

-   Code to produce the WarehousePG binary installer packages is not included
    in the open source WarehousePG repository. 

-   WarehousePG does not include EMC DD Boost integration. Backing up to an EMC Data
    Domain appliance is only supported when the Data Domain system is mounted as an NFS share on
    the WarehousePG hosts. 

-   WarehousePG does not include Symantec NetBackup integration. 

-   The QuickLZ compression method is not available in WarehousePG. QuickLZ <codeph>compresstype</codeph> values are ignored when specified in a <codeph>

    CREATE TABLE


    [AS]

    </codeph> operation, but will cause an error on subsequent <codeph>INSERT</codeph> or
    <codeph>SELECT</codeph> operations on the table.

> **Note**
> Support for the QuickLZ compression algorithm is deprecated and will be removed in the next major release of WarehousePG.

-   The deprecated <codeph>gpcheck</codeph> management utility and its replacement <codeph>gpsupport</codeph> are only supported with WarehousePG. 

-   Suggestions to contact VMware Technical Support in this documentation are intended only for WarehousePG customers. 

Source code for the following open source WarehousePG components are located in repositories outside of the WarehousePG repo:

-   To use Backup and Restore with open source WarehousePG, you must separately build and install the backup/restore software. Refer to the build instructions in the [README](https://github.com/greenplum-db/gpbackup/blob/master/README.md) file located in the WarehousePG Backup and Restore github repository.

-   To use the WarehousePG Platform Extension Framework (PXF) with open source WarehousePG, you must separately build and install the PXF software. Refer to the build and packaging instructions in the [PXF Packaging README](https://github.com/greenplum-db/pxf/blob/master/package/README.md) file located in the PXF github repository.
