This topic identifies the upgrade paths for upgrading a WarehousePG 6.x release to a newer 6.x release. The topic also describes the migration paths for migrating WarehousePG 4.x or 5.x data to WarehousePG 6.x.

WarehousePG 6 supports upgrading from a WarehousePG 6.x release to a newer 6.x release. Direct upgrade from Tanzu WarehousePG 5.28.10 to Tanzu WarehousePG 6.24.0 and later is provided via gpupgrade; for more information, see the [gpupgrade documentation](https://docs.vmware.com/en/VMware-Greenplum-Upgrade/index.html). Direct upgrade from WarehousePG 4.3, or from WarehousePG 5.27 and earlier, to WarehousePG 6 is not supported; you must migrate the data to WarehousePG 6.

-   **[Upgrading from an Earlier WarehousePG 6 Release](upgrading.mdx)**  
    The upgrade path supported for this release is WarehousePG 6.x to a newer WarehousePG 6.x release.

-   **[PXF Upgrade and Migration](../pxf/pxf_upgrade_migration.mdx)**  

-   **[Migrating Data from WarehousePG 4.3 or 5 to WarehousePG 6](migrate.html)**  
    You can migrate data from WarehousePG 4.3 or 5 to WarehousePG 6 using the standard backup and restore procedures, `gpbackup` and `gprestore`, or by using `gpcopy` for WarehousePG.
