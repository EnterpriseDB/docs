AIDB provides a set of SQL functions for performing AI inference directly in queries. These functions let you encode data into vector representations, generate text with language models, and control inference behavior — without setting up a pipeline.

## `aidb.encode_text()`

Encodes a single text string into a vector using a registered embedding model.

```sql
SELECT aidb.encode_text(
    input   => 'What is pgvector?',
    options => '{"model": "my_embedder"}'
);

__OUTPUT__
                         encode_text
--------------------------------------------------------------
 [0.021,-0.043,0.117, ...]
(1 row)
```

### Parameters

| Parameter | Type   | Description                                                       |
|-----------|--------|-------------------------------------------------------------------|
| `input`   | `TEXT` | The text string to encode.                                        |
| `options` | `JSONB`| Configuration object. Must include `"model"`: the name of a registered embedding model. |

### Returns

A single `VECTOR` value representing the embedding of the input text.

---

## `aidb.encode_text_batch()`

Encodes multiple text strings into vectors in a single batched request. More efficient than calling `encode_text()` in a loop when processing many inputs.

```sql
SELECT * FROM aidb.encode_text_batch(
    inputs  => ARRAY['What is pgvector?', 'How does RAG work?', 'Explain embeddings.'],
    options => '{"model": "my_embedder"}'
);

__OUTPUT__
 id |          value
----+-------------------------
  0 | [0.021,-0.043,0.117,...]
  1 | [0.008, 0.091,-0.034,...]
  2 | [0.055,-0.012, 0.076,...]
(3 rows)
```

### Parameters

| Parameter | Type     | Description                                                         |
|-----------|----------|---------------------------------------------------------------------|
| `inputs`  | `TEXT[]` | Array of text strings to encode.                                    |
| `options` | `JSONB`  | Configuration object. Must include `"model"`: the name of a registered embedding model. |

### Returns

A set of rows, one per input, with columns:

| Column  | Type     | Description                              |
|---------|----------|------------------------------------------|
| `id`    | `INTEGER`| Zero-based index corresponding to the input array position. |
| `value` | `VECTOR` | The embedding vector for that input.     |

## `aidb.encode_image()`

Encodes a single image into a vector using a registered multimodal embedding model (for example, a CLIP model).

```sql
SELECT aidb.encode_image(
    input   => pg_read_binary_file('/path/to/image.png')::BYTEA,
    options => '{"model": "my_clip_model"}'
);

__OUTPUT__
                         encode_image
--------------------------------------------------------------
 [-0.012, 0.084, 0.031, ...]
(1 row)
```

### Parameters

| Parameter | Type    | Description                                                         |
|-----------|---------|---------------------------------------------------------------------|
| `input`   | `BYTEA` | Raw binary image data.                                              |
| `options` | `JSONB` | Configuration object. Must include `"model"`: the name of a registered image-capable embedding model (for example, a CLIP model). |

### Returns

A single `VECTOR` value representing the embedding of the input image.

## `aidb.decode_text()`

Generates text from a language model (LLM) given a prompt. This is the core function for LLM inference in AIDB.

```sql
SELECT aidb.decode_text(
    input   => 'Explain what a knowledge base is in two sentences.',
    options => '{"model": "my_llm"}'
);

__OUTPUT__
                                        decode_text
--------------------------------------------------------------------------------------------
 A knowledge base is a structured repository of information used to answer queries or ...
(1 row)
```

To control generation behavior, pass an `inference_config` object in options:

```sql
SELECT aidb.decode_text(
    input   => 'Summarize the following text: ...',
    options => jsonb_build_object(
        'model',           'my_llm',
        'inference_config', aidb.inference_config(
            system_prompt => 'You are a concise technical writer.',
            temperature   => 0.2,
            max_tokens    => 256
        )
    )
);
```

### Parameters

| Parameter | Type   | Description                                                                                              |
|-----------|--------|----------------------------------------------------------------------------------------------------------|
| `input`   | `TEXT` | The prompt to send to the language model.                                                                |
| `options` | `JSONB`| Configuration object. Must include `"model"`. Optionally includes `"inference_config"` (see [`aidb.inference_config()`](#aidbinference_config)). |

### Returns

A single `TEXT` value containing the model's generated response.

## `aidb.decode_text_batch()`

Generates text responses for multiple prompts in a single batched request.

```sql
SELECT * FROM aidb.decode_text_batch(
    inputs  => ARRAY[
        'What is a vector database?',
        'What is retrieval-augmented generation?'
    ],
    options => '{"model": "my_llm"}'
);

__OUTPUT__
 id |                          value
----+----------------------------------------------------------
  0 | A vector database stores high-dimensional vectors ...
  1 | Retrieval-augmented generation (RAG) combines a ...
(2 rows)
```

### Parameters

| Parameter | Type     | Description                                                                                              |
|-----------|----------|----------------------------------------------------------------------------------------------------------|
| `inputs`  | `TEXT[]` | Array of prompt strings to send to the language model.                                                   |
| `options` | `JSONB`  | Configuration object. Must include `"model"`. Optionally includes `"inference_config"` (see [`aidb.inference_config()`](#aidbinference_config)). |

### Returns

A set of rows, one per input, with columns:

| Column  | Type      | Description                              |
|---------|-----------|------------------------------------------|
| `id`    | `INTEGER` | Zero-based index corresponding to the input array position. |
| `value` | `TEXT`    | The generated text response for that prompt. |

## `aidb.inference_config()`

Builds an inference configuration object to control language model generation behavior. Pass the result as the `"inference_config"` key inside the `options` JSONB of `decode_text()`, `decode_text_batch()`, or `summarize_text()`.

```sql
SELECT aidb.decode_text(
    input   => 'Write a haiku about PostgreSQL.',
    options => jsonb_build_object(
        'model',            'my_llm',
        'inference_config', aidb.inference_config(
            temperature => 0.9,
            max_tokens  => 64,
            seed        => 42
        )
    )
);
```

### Parameters

All parameters are optional.

| Parameter        | Type               | Description                                                              |
|------------------|--------------------|--------------------------------------------------------------------------|
| `system_prompt`  | `TEXT`             | System prompt prepended to each request to set the model's behavior.    |
| `temperature`    | `DOUBLE PRECISION` | Sampling temperature. Higher values produce more varied output.         |
| `max_tokens`     | `INTEGER`          | Maximum number of tokens to generate in the response.                   |
| `top_p`          | `DOUBLE PRECISION` | Nucleus sampling threshold. Limits sampling to the top-p probability mass. |
| `seed`           | `BIGINT`           | Random seed for reproducible outputs.                                   |
| `repeat_penalty` | `REAL`             | Penalty applied to repeated tokens to reduce repetition.                |
| `repeat_last_n`  | `INTEGER`          | Number of recent tokens to consider when applying the repeat penalty.   |
| `thinking`       | `BOOLEAN`          | Enable extended reasoning mode (supported models only).                 |
| `extra_args`     | `JSONB`            | Additional provider-specific inference arguments passed through to the model API. |

### Returns

A `JSONB` object for use as the `"inference_config"` value in function options.
