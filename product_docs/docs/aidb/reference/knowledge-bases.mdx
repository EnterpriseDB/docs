Reference for knowledge base functions and views. For guide-style documentation, see [Knowledge bases](../knowledge-bases).

## Views

### `aidb.knowledge_bases`

Also accessible as `aidb.kbs`. Lists all knowledge bases in the database with their full configuration.

| Column | Type | Description |
|---|---|---|
| `id` | integer | Internal identifier |
| `name` | text | Knowledge base name |
| `model_name` | text | Embedding model used |
| `topk` | integer | Default number of results returned by retrieve |
| `distance_operator` | [aidb.DistanceOperator](#aidbdistanceoperator) | Vector distance function used for retrieval |
| `auto_processing` | [aidb.PipelineAutoProcessingMode](#aidbpipelineautoprocessingmode) | Auto-processing mode |
| `batch_size` | integer | Records processed concurrently per pipeline run |
| `background_sync_interval` | interval | Polling interval for background auto-processing |
| `source_type` | text | `Table` or `Volume` |
| `source_schema` | text | Schema of the source table |
| `source_table` | text | Source table name |
| `source_data_column` | text | Column containing the data to embed |
| `source_key_column` | text | Unique key column in the source |
| `source_data_format` | [aidb.PipelineDataFormat](#aidbpipelinedataformat) | Format of source data |
| `source_volume_name` | text | Volume name (volume sources only) |
| `vector_schema` | text | Schema of the embeddings table |
| `vector_table` | text | Embeddings storage table |
| `vector_key_column` | text | Key column in the embeddings table |
| `vector_data_column` | text | Embeddings column |
| `owner_role` | text | Postgres role that owns the pipeline |

---

### `aidb.knowledge_base_stats`

Also accessible as `aidb.kbstat`. Provides current processing statistics for all knowledge base pipelines.

| Column | Type | Description |
|---|---|---|
| `knowledge base` | text | Knowledge base name |
| `auto processing` | [aidb.PipelineAutoProcessingMode](#aidbpipelineautoprocessingmode) | Configured processing mode |
| `table: unprocessed rows` | bigint | Backlog size (table sources) |
| `volume: scans completed` | bigint | Full scans completed (volume sources) |
| `count(source records)` | bigint | Total records in the source |
| `count(embeddings)` | bigint | Total embeddings stored |

#### Example

```sql
SELECT * FROM aidb.kbstat;

__OUTPUT__
     knowledge base     | auto processing | table: unprocessed rows | volume: scans completed | count(source records) | count(embeddings)
------------------------+-----------------+-------------------------+-------------------------+-----------------------+-------------------
 kb_table_text_bg       | Background      |                       0 |                         |                    15 |                15
 kb_volume_text_bg      | Background      |                         |                       6 |                     7 |                 7
(2 rows)
```

## Types

### `aidb.DistanceOperator`

Vector distance function used during retrieval.

| Value | Description |
|---|---|
| `L2` | Euclidean distance |
| `InnerProduct` | Inner product |
| `Cosine` | Cosine similarity |
| `L1` | L1 distance |
| `Hamming` | Hamming distance |
| `Jaccard` | Jaccard distance |

---

### `aidb.PipelineDataFormat`

Format of data in a pipeline source or volume.

| Value | Description |
|---|---|
| `Text` | Plain text |
| `Image` | Binary image data |
| `Pdf` | PDF documents |

---

### `aidb.PipelineAutoProcessingMode`

Auto-processing mode for a knowledge base pipeline.

| Value | Description |
|---|---|
| `Live` | Immediate processing via Postgres triggers on each data change |
| `Background` | Periodic processing via a Postgres background worker |
| `Disabled` | No automatic processing; run manually with `aidb.bulk_embedding()` |

## Knowledge base functions

### `aidb.create_table_knowledge_base`

Creates a knowledge base backed by a Postgres table source.

#### Parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `name` | TEXT | Required | Name of the knowledge base. |
| `model_name` | TEXT | Required | Registered model to use for embeddings. |
| `source_table` | REGCLASS | Required | Source table. |
| `source_data_column` | TEXT | Required | Column containing data to embed. |
| `source_data_format` | [aidb.PipelineDataFormat](#aidbpipelinedataformat) | Required | Format of source data (`Text`, `Image`, `Pdf`). |
| `source_key_column` | TEXT | `'id'` | Unique key column in the source table. |
| `vector_table` | TEXT | NULL | Name for the embeddings table (auto-generated if NULL). |
| `vector_data_column` | TEXT | `'embeddings'` | Embeddings column name. |
| `vector_key_column` | TEXT | `'id'` | Key column in the embeddings table. |
| `topk` | INTEGER | `1` | Default number of retrieve results. |
| `distance_operator` | [aidb.DistanceOperator](#aidbdistanceoperator) | `'L2'` | Distance function for retrieval. |
| `auto_processing` | [aidb.PipelineAutoProcessingMode](#aidbpipelineautoprocessingmode) | `'Disabled'` | Auto-processing mode. |
| `index_type` | TEXT | `'vector'` | Vector index type: `vector` (HNSW), `ivfflat`, or `disabled`. |
| `batch_size` | INTEGER | `100` | Records processed per batch. |
| `background_sync_interval` | INTERVAL | `'30 seconds'` | Background worker polling interval. |
| `invoker_role` | TEXT | NULL | Role for table ownership and background execution. |

The `vector` index type creates an HNSW index automatically. It supports up to 2000 dimensions. Use `disabled` for higher-dimensional models and manage the index manually.

#### Example

```sql
SELECT aidb.create_table_knowledge_base(
    name               => 'my_kb',
    model_name         => 'bert_local',
    source_table       => 'products',
    source_data_column => 'description',
    source_key_column  => 'product_id',
    source_data_format => 'Text'
);
```

### `aidb.create_volume_knowledge_base`

Creates a knowledge base backed by a PGFS volume (external storage).

#### Parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `name` | TEXT | Required | Name of the knowledge base. |
| `model_name` | TEXT | Required | Registered model to use for embeddings. |
| `source_volume_name` | TEXT | Required | AIDB volume name. |
| `vector_table` | TEXT | NULL | Name for the embeddings table (auto-generated if NULL). |
| `vector_data_column` | TEXT | `'embeddings'` | Embeddings column name. |
| `vector_key_column` | TEXT | `'id'` | Key column in the embeddings table. |
| `topk` | INTEGER | `1` | Default number of retrieve results. |
| `distance_operator` | [aidb.DistanceOperator](#aidbdistanceoperator) | `'L2'` | Distance function for retrieval. |
| `auto_processing` | [aidb.PipelineAutoProcessingMode](#aidbpipelineautoprocessingmode) | `'Disabled'` | Auto-processing mode. |
| `index_type` | TEXT | `'vector'` | Vector index type: `vector`, `ivfflat`, or `disabled`. |
| `batch_size` | INTEGER | `100` | Records processed per batch. |
| `background_sync_interval` | INTERVAL | `'30 seconds'` | Background worker polling interval. |
| `invoker_role` | TEXT | NULL | Role for table ownership and background execution. |

#### Example

```sql
SELECT aidb.create_volume_knowledge_base(
    name               => 'my_volume_kb',
    model_name         => 'bert_local',
    source_volume_name => 'pdf_volume'
);
```

---

### `aidb.set_auto_knowledge_base`

Updates the auto-processing mode for an existing knowledge base.

#### Parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `knowledge_base_name` | TEXT | Required | Name of the knowledge base. |
| `mode` | [aidb.PipelineAutoProcessingMode](#aidbpipelineautoprocessingmode) | Required | New auto-processing mode. |
| `batch_size` | INTEGER | NULL | Records processed per batch (Background and Disabled modes). |
| `background_sync_interval` | INTERVAL | NULL | Polling interval (Background mode). |

#### Examples

```sql
SELECT aidb.set_auto_knowledge_base('my_kb', 'Live');
SELECT aidb.set_auto_knowledge_base('my_kb', 'Background', background_sync_interval => '1 minute');
SELECT aidb.set_auto_knowledge_base('my_kb', 'Disabled', batch_size => 200);
```

### `aidb.bulk_embedding`

Processes all source records and stores embeddings in the destination table. Behavior depends on the auto-processing mode: Live and Disabled modes process records immediately; Background mode marks records for the background worker to process.

#### Parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `knowledge_base_name` | TEXT | Required | Name of the knowledge base. |
| `silent` | BOOL | `false` | Suppress progress log output. |

#### Example

```sql
SELECT aidb.bulk_embedding('my_kb');
```

---

### `aidb.retrieve_key`

Returns the source record keys and distances for the top matching embeddings, without fetching source data.

#### Parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `knowledge_base_name` | TEXT | Required | Name of the knowledge base. |
| `query_string` | TEXT | Required | Query text to search with. |
| `number_of_results` | INTEGER | `0` | Number of results to return. Uses `topk` default if `0`. |

#### Returns

| Column | Type | Description |
|---|---|---|
| `key` | text | Source record key |
| `distance` | double precision | Vector distance from the query |

#### Example

```sql
SELECT * FROM aidb.retrieve_key('my_kb', 'running shoes', 5);

__OUTPUT__
  key  |      distance
-------+--------------------
 43941 | 0.2938963414490189
 19337 | 0.3023805122617119
(2 rows)
```

---

### `aidb.retrieve_text`

Returns source text and distances for the top matching embeddings by joining the embeddings table with the source table.

#### Parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `knowledge_base_name` | TEXT | Required | Name of the knowledge base. |
| `query_string` | TEXT | Required | Query text to search with. |
| `number_of_results` | INTEGER | `0` | Number of results to return. Uses `topk` default if `0`. |

#### Returns

| Column | Type | Description |
|---|---|---|
| `key` | text | Source record key |
| `value` | text | Source text content |
| `distance` | double precision | Vector distance from the query |

#### Example

```sql
SELECT * FROM aidb.retrieve_text('my_kb', 'waterproof jacket', 3);

__OUTPUT__
  key  |                       value                        |      distance
-------+----------------------------------------------------+--------------------
 19337 | Men Stripes Waterproof Shell Jacket                | 0.2994317672742334
 55018 | Women All-Weather Hiking Anorak                    | 0.3804609668507203
(2 rows)
```

---

### `aidb.delete_knowledge_base`

Removes a knowledge base and its configuration. Does not delete the underlying source table or embeddings table.

#### Parameters

| Parameter | Type | Description |
|---|---|---|
| `knowledge_base_name` | TEXT | Name of the knowledge base to delete. |

#### Example

```sql
SELECT aidb.delete_knowledge_base('my_kb');
```

## Volume functions

### `aidb.create_volume`

Creates an AIDB volume from a PGFS storage location for use as a knowledge base data source. See [External data sources](../integrating-models/external-data-sources) for setting up the PGFS storage location first.

#### Parameters

| Parameter | Type | Description |
|---|---|---|
| `name` | TEXT | Name for the volume. |
| `server_name` | TEXT | Name of the PGFS storage location. |
| `path` | TEXT | Sub-path within the storage location. |
| `mime_type` | TEXT | Data type: `Text`, `Image`, or `Pdf`. |

#### Example

```sql
SELECT aidb.create_volume('pdf_volume', 'my_s3_location', '/', 'Pdf');
```

---

### `aidb.list_volumes`

Lists all AIDB volumes in the database.

```sql
SELECT * FROM aidb.list_volumes();
```

---

### `aidb.delete_volume`

Deletes an AIDB volume. Note: deleting the underlying PGFS storage location also deletes all volumes built on top of it.

#### Parameters

| Parameter | Type | Description |
|---|---|---|
| `volume_name` | TEXT | Name of the volume to delete. |

```sql
SELECT aidb.delete_volume('pdf_volume');
```
