AIDB connects to AI models through a model registry. Models are referenced by name in pipeline steps and SQL functions. A small set of models are built in and available without any registration — for everything else, use `aidb.create_model()` to register the model before use.

- [Integrated models](integrated-models) — Built-in local models (BERT, CLIP, T5, Llama) that run inside your Postgres host, and how to register custom variants.
- [External model connections](external-model-connections) — Connecting to OpenAI-compatible APIs, NVIDIA NIM, Google Gemini, and OpenRouter.
- [AIDB volumes](external-data-sources) — Using S3-compatible object stores and local file systems as pipeline data sources via PGFS.
- [Background workers and PG settings](background-workers) — Postgres configuration required for asynchronous pipeline processing.
