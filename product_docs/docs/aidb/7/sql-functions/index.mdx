AIDB functions are available as standalone SQL functions that you can call directly in queries for one-off transformations, testing and exploration, or to build custom workflows outside of the pipeline framework.

Each page here focuses on usage context and examples. For terse parameter tables and return types, see the API reference: 
- Transformation functions (`chunk_text`, `parse_html`, `parse_pdf`, `perform_ocr`, `summarize_text`) are in [Reference → Functions](../reference/functions)
- Inference functions (`encode_text`, `decode_text`, `rerank_text`) are in [Reference → Models](../reference/models#inference-functions).

| Page                                                | Functions                                                  | What it covers                                                        |
|-----------------------------------------------------|------------------------------------------------------------|-----------------------------------------------------------------------|
| [Chunking](chunk-text)                              | `aidb.chunk_text()`                                        | Splitting text into segments for LLM context windows                  |
| [Parsing](parse)                                    | `aidb.parse_html()`, `aidb.parse_pdf()`                    | Extracting readable text from HTML and PDF inputs                     |
| [OCR](perform-ocr)                                  | `aidb.perform_ocr()`                                       | Extracting text from images using an OCR-capable model                |
| [Summarization](summarize-text)                     | `aidb.summarize_text()`, `aidb.summarize_text_aggregate()` | Generating and aggregating AI-powered text summaries, use cases, and cookbook |
| [Encoding and decoding](encode-decode)              | `aidb.encode_text()`, `aidb.decode_text()`, and variants   | Embedding text and images into vectors; generating completions        |
| [Reranking](rerank-text)                            | `aidb.rerank_text()`                                       | Scoring and reordering candidate passages by relevance                |

!!!Note
To use these operations as steps inside a pipeline, see [Pipeline steps](../data-pipelines/pipeline-steps).
