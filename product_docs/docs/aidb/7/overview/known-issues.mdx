The following known issues and limitations apply to the current release of AIDB.

## `aidb.retry_pipeline_errors()` not yet available

**Affects:** All versions

The function for retrying failed pipeline records from the error log, `aidb.retry_pipeline_errors()`, is not yet available. It is planned for a future release.

**Workaround:** Use [`aidb.get_error_logs()`](../data-pipelines/orchestration/error-log#querying-errors) to inspect failed records and identify the cause of failure. Once you've resolved the underlying issue (for example, replacing a corrupt source file or correcting a model configuration), use [`aidb.clear_error_logs()`](../data-pipelines/orchestration/error-log#clearing-errors) to remove the error entries and then re-run the pipeline with `aidb.run_pipeline()` to reprocess the affected data.
