BM25 isn't managed by AI pipelines. You set it up independently on your source table, then reference it in your hybrid queries.

VectorChord-BM25 provides keyword ranking on top of PostgreSQL using the BM25 algorithm. 
It consists of two extensions — `pg_tokenizer` (tokenizes text into BM25-compatible vectors) and `vchord_bm25` (stores and queries those vectors). 
Both must be installed as OS packages from the EDB repository before the SQL setup steps below will work. See [Installing VectorChord-BM25](/pg_extensions/vectorchord_bm25/installing/) for package installation instructions.

1. Install extensions

```sql
CREATE EXTENSION IF NOT EXISTS pg_tokenizer;
CREATE EXTENSION IF NOT EXISTS vchord_bm25;
```

1. Create a tokenizer

```sql
SELECT tokenizer_catalog.create_tokenizer('bert', $$
    model = "bert_base_uncased"
$$);
```

1. Add a `bm25vector` column and index to your source table

```sql
ALTER TABLE my_source ADD COLUMN bm25_tokens bm25_catalog.bm25vector;
UPDATE my_source SET bm25_tokens = tokenizer_catalog.tokenize(description, 'bert');

CREATE INDEX my_source_bm25 ON my_source USING bm25 (bm25_tokens bm25_catalog.bm25_ops);
```

1. Keep the index current

New rows must be tokenized before they'll appear in BM25 results. Use a trigger or update the column as part of your data ingestion process:

```sql
UPDATE my_source SET bm25_tokens = tokenizer_catalog.tokenize(description, 'bert') WHERE bm25_tokens IS NULL;
```

For full tokenizer configuration options, see [VectorChord-BM25](../vectorchord/vectorchord-bm25) and [VectorChord-BM25 in pg_extensions](/pg_extensions/vectorchord_bm25/).

## Worked example: BM25 with RRF

This example shows how to combine semantic vector search with BM25 keyword search using Reciprocal Rank Fusion (RRF) to merge the two result sets. It uses the `products` table and `food_products` pipeline set up in the [relational predicate filtering example](query-patterns#worked-example-relational-predicate-filtering).

**Add a BM25 index to the source table**

BM25 isn't managed by AI pipelines — you set it up directly on your source table. This section adds the tokenizer and index that the keyword leg of the RRF query relies on.

1. Enable the `pg_tokenizer` and `vchord_bm25` extensions:

```sql
CREATE EXTENSION IF NOT EXISTS pg_tokenizer;
CREATE EXTENSION IF NOT EXISTS vchord_bm25;
```

1. Create a tokenizer using an available model:

We'll use the BERT model in this example. You can choose any model supported by the `pg_tokenizer` extension, but make sure to use the same model for both tokenizing the source data and the query.

```sql
SELECT tokenizer_catalog.create_tokenizer('bert', $$
    model = "bert_base_uncased"
$$);
```

1. Add a BM25 vector column to the source table and populate it with tokenized values:

```sql
ALTER TABLE products ADD COLUMN embedding_bm25 bm25_catalog.bm25vector;
UPDATE products SET embedding_bm25 = tokenizer_catalog.tokenize(product_name, 'bert');
```

1. Create a BM25 index on the new column:

```sql
CREATE INDEX products_bm25_idx ON products USING bm25 (embedding_bm25 bm25_catalog.bm25_ops);
```

**Hybrid query with RRF**

This query runs semantic and keyword searches as separate CTEs, then merges the two result sets using Reciprocal Rank Fusion (RRF). Items that rank well in either search surface at the top; items that rank well in both receive the highest scores.

```sql
SET hnsw.iterative_scan = relaxed_order;
SET search_path TO public, tokenizer_catalog, bm25_catalog;

WITH semantic AS (
    SELECT source.id,
           source.product_name,
           source.category,
           ROW_NUMBER() OVER (ORDER BY vector.value <=> aidb.kb_query_encode('public.pipeline_food_products', 'fast food')::vector) AS rank
    FROM public.pipeline_food_products AS vector
    JOIN products AS source ON source.id::TEXT = vector.source_id
    ORDER BY vector.value <=> aidb.kb_query_encode('public.pipeline_food_products', 'fast food')::vector
    LIMIT 20
),
keyword AS (
    SELECT id,
           product_name,
           ROW_NUMBER() OVER (ORDER BY embedding_bm25 <&> to_bm25query('products_bm25_idx', tokenize('fast food', 'bert'))) AS rank
    FROM products
    ORDER BY embedding_bm25 <&> to_bm25query('products_bm25_idx', tokenize('fast food', 'bert'))
    LIMIT 20
)
SELECT
    COALESCE(s.id, k.id)                                                         AS id,
    COALESCE(s.product_name, k.product_name)                                     AS product_name,
    s.category,
    s.rank                                                                       AS semantic_rank,
    k.rank                                                                       AS keyword_rank,
    COALESCE(1.0 / (60 + s.rank), 0) + COALESCE(1.0 / (60 + k.rank), 0)        AS rrf_score
FROM semantic s
FULL OUTER JOIN keyword k ON s.id = k.id
ORDER BY rrf_score DESC
LIMIT 10;
__OUTPUT__
 id | product_name  |  category  | semantic_rank | keyword_rank |       rrf_score
----+---------------+------------+---------------+--------------+------------------------
  1 | Hamburger     | meat       |             6 |            1 | 0.03154495777446597119
  4 | Fries         | vegetarian |             3 |            4 | 0.03149801587301587302
  6 | Pizza         | meat       |             2 |            6 | 0.03128054740957966765
  8 | Veggie Burger | vegetarian |             1 |            8 | 0.03109932497589199614
  3 | Fish n Chips  | fish       |             7 |            3 | 0.03079838900734423123
  5 | Burrito       | meat       |             5 |            5 | 0.03076923076923076924
  2 | Cheeseburger  | meat       |             9 |            2 | 0.03062178588125292193
  7 | Sandwich      | meat       |             4 |            7 | 0.03055037313432835821
  9 | Kebab         | meat       |             8 |            9 | 0.02919863597612958227
```

The RRF score combines both rankings: items that rank highly in either semantic or keyword search surface at the top, with items that rank well in both receiving the highest scores.
