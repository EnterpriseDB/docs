You can configure your HM installation across multiple data centers (multi-DC topology) to meet your high availability requirements. This topology includes a primary HM and one or more data-only HMs (Secondaries) across multiple Kubernetes clusters.

- [Overview](overview) &mdash; Reviews multi-DC concepts, explores the available multi-dc topologies, and provides an architecture review of a multi-DC HM installation.

- [Configuring a multi-dc deployment](multi-dc) &mdash; Provides a step-by-step guide of how to configure your HM installation across multiple data centers, and then provision a Postgres cluster across locations.
