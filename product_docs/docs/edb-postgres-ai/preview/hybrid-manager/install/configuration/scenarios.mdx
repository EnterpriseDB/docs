## Overview

The Hybrid Manager installation requires many components to be running in order to cover the full range of functionality provided by default.  In the event that you do not require some of this functionality, it's possible to disable various component groups in order to reduce installation size and therefore also running costs.  These component groups are known as 'scenarios'.

The available scenarios are as follows:
- `core` &mdash; the minimum set of components for Hybrid Manager to function; this is always required.
- `migration` &mdash; all components relating to the Migration Portal.
- `ai` &mdash; all AI-related components, such as KServe and Langflow.
- `analytics` &mdash; all data analytics functionality, such as Lakehouse clusters and the Spark functionality.

You many have already seen this list and chosen your scenarios whilst customising the `values.yaml` file at installation time.  However, as explained here, it is also possible to change which scenarios are running after installation.

## Finding out which scenarios are installed

To see the scenarios currently installed &mdash; which, by default, should be all of them &mdash; you can either:
- Click on the 'About' link found at the bottom of all pages of the Hybrid Manager portal.
- Run `kubectl get hybridcontrolplane edbpgai -o yaml | yq .spec.scenarios` to return the list of installed scenarios.

## Changing which scenarios are installed

To change the scenarios installed, run `kubectl edit hybridcontrolplane edbpgai` and move down to the `scenarios` section of the spec (example shown below) and remove/add items as appropriate.  Remember that `core` should always be included in the list.

```yaml
scenarios:
  - core
  - migration
  - ai
  - analytics
```

Now save the changes and exit your editor.  It will take approximately 5 minutes for the changes to take effect.  You can confirm which scenarios are now installed via the "About" link at the bottom of all Hybrid Manager portal pages.
