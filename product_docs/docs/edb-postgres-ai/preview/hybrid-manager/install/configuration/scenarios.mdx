## Overview

The Hybrid Manager (HM) installation requires several components to cover the full range of functionality provided by default. To optimize resource usage, HM supports modular installations via `scenarios`. This allows you to deploy only the components necessary for your specific use case, while omitting heavier, unused services (e.g. AI or Analytics) to minimize the installation's size and overhead.

| Module      | Description                                               | Included capabilities                                                                                  |
|-------------|-----------------------------------------------------------|--------------------------------------------------------------------------------------------------------|
| `core`      | **Required**. The foundational layer for all deployments. | Estate management, Observability, DBaaS provisioning, and system services (Cert-manager, Istio, etc.). |
| `migration` | Enables schema and data migration tools.                  | Migration Portal integration, Data Migration Service (DMS) and other migration services.               |
| `ai`        | Tools for building and serving generative AI.             | Sovereign AI, model serving (kserve), Langflow, and GenAI builders.                                    |
| `analytics` | Large-scale data processing and cataloging.               | Lakehouse cluster management, Data Catalog, and Spark-based processing.                                |

You may have already seen this list and chosen your scenarios while customizing the `values.yaml` file [at installation time](/edb-postgres-ai/preview/hybrid-manager/install/planning_arch/#installation-modules). However, as explained here, it is also possible to change which modules to enable after installation.

## Verifying installed modules

To see the modules (or `scenarios`) currently installed &mdash;which, by default, is all of them&mdash; you can either:
- In the HM console, select the **About** at the bottom of any page.
- In your terminal, with your kubeconfig pointing to the HM installation, run `kubectl get hybridcontrolplane edbpgai -o yaml | yq .spec.scenarios` to return the list of installed scenarios.

## Modifying installed modules

To change the modules installed, with your kubeconfig pointing to the HM installation, run `kubectl edit hybridcontrolplane edbpgai` and move down to the `scenarios` section of the configuration file (example shown below). Remove or add items as appropriate. Remember that `core` should always be included in the list.

```yaml
...
scenarios:
  - core
  - migration
  - ai
  - analytics
```
