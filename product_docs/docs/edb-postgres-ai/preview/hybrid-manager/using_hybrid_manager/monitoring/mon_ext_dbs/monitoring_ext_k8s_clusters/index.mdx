Use the EDB Postgres AI agent for centralized monitoring, metrics collection and resource management for your [EDB Postgres® AI for CloudNativePG™](/postgres_for_kubernetes/latest/) or [CloudNativePG™](https://cloudnative-pg.io/) clusters and integrate them on your HM console. By integrating these clusters with your HM console, you gain a unified view of your cloud-native environment. This integration is deployed using a Helm chart.

The EDB Postgres AI agent can be configured to run anywhere. To activate specific capabilities, assign it one or more roles: Webhooks, Providers, Dispatchers, or Runners. A single agent can be configured to perform any combination of these functions simultaneously.

## Architecture

The EDB Postgres AI agent used for Kubernetes deployments consists of the following components.

### EDB Postgres AI agent

The EDB Postgres AI agent serves as a centralized tool for resource discovery and monitoring, while also streamlining the collection and forwarding of your metrics.

**Key functions:**
  - Resource discovery and monitoring
  - Metrics collection and forwarding

### OpenTelemetry collector (optional)

The OpenTelemetry (OTEL) collector acts as a centralized data pipeline that gathers logs and exports telemetry data from your environment.

**Key functions:**
  - Container metrics (cAdvisor): The cAdvisor service collects and processes resource usage data (CPU, memory) from active containers.
  - Kubelet metrics: Monitors the health and performance of Kubernetes nodes and pods.
  - Application logs: Aggregates and forwards logs from your database applications.
  - Custom metrics: Allows for the collection of user-defined metrics tailored to specific application needs.

**Optional component:**
- Kube state metrics: Generates metrics based on the state of Kubernetes objects.

!!! important Important:
When deploying [CloudNativePG™](/supported-open-source/cloud_native_pg/) or [EDB Postgres® AI for CloudNativePG™](/postgres_for_kubernetes/latest/) clusters or Postgres Distributed (PGD) groups in different namespaces, you must [create the required secrets](/edb-postgres-ai/preview/hybrid-manager/using_hybrid_manager/monitoring/mon_ext_dbs/mon_ext_k8s_clusters/installation_requirements/#namespace-and-secrets) in each namespace separately.
