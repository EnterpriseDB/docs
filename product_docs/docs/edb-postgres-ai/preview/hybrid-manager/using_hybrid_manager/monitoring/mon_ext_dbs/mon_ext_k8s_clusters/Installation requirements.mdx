## Kubernetes Cluster Requirements

### HM console

- You need access to a HM Console instance with:
    - Machine user: After your HM cluster is ready, create a machine user.
    - Estate Ingest Role: Assign the Estate Ingest Role to the machine user you created.
    - Access Key: Save the access key generated during the machine user creation.
    - Project ID: Note the Project ID where the agent will report metrics.
- Kubernetes Version: 1.21 or higher
- Helm Version: 3.8 or higher
- Cluster Access: Admin privileges to create namespaces, RBAC resources, and deployments.
- Required Kubernetes Operators to install the necessary operator based on the metrics you want to monitor.

### Installing the appropriate PostgreSQL operator

Option A: [EDB CloudNativePG Operator](/postgres_for_kubernetes/latest/installation_upgrade/)

Option B: [CloudNativePG Operator](https://cloudnative-pg.io/docs/devel/installation_upgrade/)

### Requirements for each namespace in CloudnativePG clusters or PGD Groups

- Beacon Access Key Secret: Contains the HCP access key for the cluster.
- Beacon Server CA Secret: Contains the TLS certificate for connecting to HCP.
