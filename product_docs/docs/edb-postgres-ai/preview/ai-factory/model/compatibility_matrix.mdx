## Test Data & Methodology

| Test Type | Data Source | Avg Input Size | Input Range | Latency Requests | Throughput Requests | Concurrency |
|---|---|---|---|---|---|---|
| Completion | Alpaca instructions + input | ~19 tokens (77 chars) | 29тАУ234 chars | 30 | 200 | 100 |
| Embedding (Text) | Alpaca instructions + output | ~77 tokens (308 chars) | 46тАУ993 chars | 30 | 200 | 100 |
| Embedding (Image) | Test image (PNG) | 10.3 KB | тАФ | 100 | 200 | 100 |
| Reranking | 5 hardcoded passages | тАФ | тАФ | 100 | 200 | 100 |
| OCR | Test image (PNG) | 10.3 KB | тАФ | 100 | 200 | 100 |

- **Completion inputs**: Instruction + input fields from the [Stanford Alpaca](https://github.com/tatsu-lab/stanford_alpaca) dataset
- **Embedding inputs**: Instruction + output fields concatenated for longer, more realistic texts (up to ~1024 tokens)
- **Completion tests**: Responses limited to max_tokens=1024, streaming enabled for latency measurement (TTFT + full request)
- **Latency**: Sequential requests measuring per-request response time
- **Throughput**: Concurrent requests measuring requests per second (RPS)
