Use these fixes to resolve common Hybrid Manager (HM) Chat Agent issues for users and administrators. If you don't see your issue here, contact EDB Support with the request-tracing ID shown in the HM console.

## The Sparkle icon doesn't appear

Problem: the rainbow **Sparkle** icon is missing from the HM header.

Likely causes and fixes:

- **The AI scenario isn't installed.** The Chat Agent is part of the AI scenario. Check with your HM administrator that the AI scenario was selected at install or upgrade time. See [HM installation scenarios](../../install/install_guide/install_hm/#installation-scenarios).

## The Sparkle icon is disabled with a tooltip

Problem: the icon is visible but grayed out. Hovering shows a tooltip such as _The administrator needs to configure an inference service to enable the Chat Agent_ — or another reason.

Likely causes and fixes:

- **No model is registered as a Chat Agent provider.** The administrator must register a chat-completion model and apply the `chatbot-gen-content` function. See [Chat Agent model configuration](enable/#chat-agent-model-configuration).
- **The registered inference service isn't ready.** Open **Estate > Inference Services** and confirm the **Status** column shows **Ready**. The Chat Agent only surfaces inference services in the **Ready** state — services in any other state (for example, **Unknown**) can't be used by the Chat Agent.
- **The Chat Agent service is unavailable.** The tooltip says so, along with a reason. Wait a moment and try again, or check with your administrator that the supporting HM workloads are healthy.

## Generation stalls or fails

Problem: the **Waiting…** placeholder bubble stays for a long time, or the response ends with _Request failed, please try again!_

Likely causes and fixes:

- **The configured model is unreachable or has reached a usage limit.** Try a different model from the model selector if more than one is registered. Otherwise, check the model server's status in **AI Factory > Model Serving** ([Monitor InferenceService](../../../ai-factory/model/monitor-inferenceservice)).
- **The prompt is too long.** Each user message is capped at 10,240 characters. Trim the prompt and retry.
- **Network egress is blocked.** For third-party models, confirm that the HM cluster has outbound access to the provider's API.
- **The model rejects the configured `chatbot-default-thinking-level`.** Not every reasoning model accepts a per-request thinking-level setting, so the turn fails when one is passed. Remove `chatbot-default-thinking-level` from the model's **Functions** field to fall back to the model's default, or switch to a model that documents support for per-request thinking-level configuration. See [Configuring model functions](enable/#configuring-model-functions).

## The Chat Agent doesn't answer my question

Problem: the Chat Agent says it can only help with HM topics.

Cause: the Chat Agent answers HM-related questions only. Off-topic questions — general knowledge, math, creative writing, and other products — are out of scope.

Solution: rephrase your question to be about your HM estate, an HM feature, or HM documentation.

## The Chat Agent says it can't perform an action

Problem: the Chat Agent reports a **403** or _not authorized_ error and stops, or says you don't have permission to perform the action.

Cause: the Chat Agent runs as you. If you don't have the role to perform the action in the HM console, the Chat Agent gets the same answer.

Solution:

- Confirm with an administrator that you hold the right organization or project role.
- Check that you are a member of the project the action targets.
- For action types that require approval workflow (such as some automation tasks), wait for an administrator to approve.

See [Authorization model](security_and_data/#authorization-model).

## The Chat Agent doesn't see my cluster or project

Problem: the Chat Agent says it cannot find a project or cluster you know exists.

Likely causes and fixes:

- **You aren't a member of the project that owns the resource.** The Chat Agent only sees resources within projects you can access. Ask an administrator to add you to the project.
- **You misspelled the resource name.** Resource names are unique within a project. The Chat Agent resolves a name to an ID by listing the resources first. A typo can cause it to find no match. Try copying the name from the HM console.

## The Chat Agent says a topic post-dates its knowledge

Problem: the Chat Agent says the topic you asked about is newer than its bundled documentation snapshot.

Cause: each HM release ships with a snapshot of the official HM documentation, with a stamped knowledge cutoff date. The Chat Agent reports when the documentation snapshot does not contain an answer.

Solution: refer to the official EDB Hybrid Manager documentation site for newer information. To benefit from a fresher in-product snapshot, upgrade HM to a more recent release.

## The Chat Agent keeps asking follow-up questions

Some Chat Agent flows follow a predefined sequence of inputs to complete a specific task — Postgres Cluster lifecycle management is one example. When the Chat Agent doesn't have all the required parameters, it keeps asking until it does, rather than guessing or proceeding with incomplete data.

To stop a pending form, cancel it or type an unrelated message in chat — the question clears.

## Charts don't render in the response

Problem: the response shows raw chart syntax or a malformed placeholder instead of a rendered chart.

Cause: chart generation requires the model to emit precise structured output. Smaller models often struggle with the syntax and produce charts the renderer can't display.

Solution: switch to a Large or Extra Large model from the model picker if your administrator registered more than one. Otherwise, ask the administrator to register a stronger model. See [Model selection](enable/#model-selection).

## Reduced capability

Problem: the Chat Agent doesn't offer cluster creation, doesn't ask structured follow-up questions, or doesn't start the full interactive flow you've seen before.

Cause: Chat Agent capability scales with the strength of the configured model. A small or medium model handles fewer flows than a frontier model, and structured follow-up forms are only used when the model supports them.

Solution: select a stronger model from the model picker if your administrator registered more than one, or ask the administrator to register a stronger model. See [Model selection](enable/#model-selection).

## Contacting support

If the issue persists, capture the following before contacting EDB Support:

- HM troubleshooting information — open **About** from the HM console footer and select **Copy troubleshooting information** to capture the version, build, scenarios, organization, session, and platform details.
- The model and model provider in use, if known.
- A copy of the prompt you sent and the response you received.
