The EDB Hybrid Manager (HM) **Chat Agent** is an in-product, conversational assistant that helps you operate HM in plain language. Ask it about your estate, provision and inspect Postgres clusters, look up active alerts, troubleshoot performance, and consult the HM documentation — all without leaving the HM console.

## Supported operations

The Chat Agent covers estate browsing, cluster lifecycle management, monitoring and alerts, configuration recommendations, 24-hour cluster reports, Oracle-to-Postgres SQL translation, Data Migration Service status, and documentation Q&A — all scoped to your HM permissions.

For the complete catalog with example prompts and scope boundaries, see [What the Chat Agent can do](./capabilities).

## Accessing the Chat Agent

The Chat Agent is available on every authenticated page in the HM console.

1. Look for the rainbow **Sparkle** icon at the top right of the HM header.
2. Select the icon to open the Chat Agent.

If the icon is missing or disabled, see [Troubleshooting](./troubleshooting).

The Chat Agent opens in two layouts:

- **Full screen** — the default. The conversations list appears on the left.
- **Floating panel** — a compact, draggable panel docked at the bottom right of the HM console. Use the expand or shrink button to switch between modes.

## Using the Chat Agent

A typical interaction looks like:

1. (Optional) Select a **model** if your deployment exposes more than one.
2. Type your question into **Ask me anything…** and press **Enter** to send. Press **Shift+Enter** to add a new line.
3. The Chat Agent streams its response. To stop generation early, select the stop button.
4. To start a new conversation, select **New Chat**.
5. To revisit a saved conversation, open the conversations sidebar with the menu icon.

When the Chat Agent needs more information to act safely, it pauses and presents a structured form — for example, _which project?_ or _the connection password?_. Answer the form to continue. Cancel or send an unrelated message to clear the pending question.

### Conversations

Your conversations are persisted to your HM organization so you can resume them later. Each conversation is private to you — other users, including organization owners, can't read or list it. See [Conversation storage and isolation](./security_and_data#conversation-storage-and-isolation) for the full access model.

The Chat Agent auto-generates a short subject when a conversation begins. From the conversations sidebar, you can:

- Resume an earlier conversation.
- **Rename** a conversation.
- **Delete** a conversation. You cannot undo this action.

### Page context

Open the Chat Agent on a cluster, project, or other detail page, and it picks up the page context. Typically the project and cluster IDs appear as small tags above the input. Asking _show me the active alerts_ then implicitly applies to that resource.

## Model-dependent capabilities

The Chat Agent's effective capability scales with the strength of the model your administrator configures. A small, low-cost model handles estate browsing and metric lookups, while more complex cluster-creation flows and structured follow-up questions need a stronger model. See [Choosing a model](./enable#choosing-a-model) for guidance on model selection.

## Documentation scope

The Chat Agent answers HM questions from a versioned documentation snapshot shipped with each release. For details on citation behavior and the knowledge cutoff, see [HM documentation Q&A](./capabilities#hm-documentation-qa).
