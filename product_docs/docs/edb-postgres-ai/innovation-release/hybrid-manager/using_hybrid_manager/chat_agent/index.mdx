The EDB Hybrid Manager (HM) **Chat Agent** is an in-product, conversational assistant that helps you operate HM in plain language. Ask it about your estate, provision and inspect Postgres clusters, look up active alerts, troubleshoot performance, and consult the HM documentation — all without leaving the HM console.

## Supported operations

The Chat Agent covers estate browsing, cluster lifecycle management, monitoring and alerts, configuration recommendations, 24-hour cluster reports, Oracle-to-Postgres SQL translation, Data Migration Service status, and documentation Q&A — all scoped to your HM permissions.

For the complete catalog with example prompts and scope boundaries, see [What the Chat Agent can do](./capabilities).

## Accessing the Chat Agent

The Chat Agent is available on every authenticated page in the HM console.

1. Look for the rainbow **Sparkle** icon at the top right of the HM header.
1. Select the icon to open the Chat Agent.

If the icon is missing or disabled, see [Troubleshooting](./troubleshooting).

The Chat Agent opens in two layouts:

- **Full screen** — the default. The conversations list appears on the left.
- **Floating panel** — a compact, draggable panel docked at the bottom right of the HM console. Use the expand or shrink button to switch between modes.

## Using the Chat Agent

### Ask a question

1. (Optional) Select **Model** from the picker if your administrator registered more than one. The model determines which capabilities are available — see [Model-dependent capabilities](#model-dependent-capabilities).
1. Type your question in the **Ask me anything** input and press **Enter**.
1. Use **Shift+Enter** to add a line break within your message.
1. The Chat Agent streams a response in the chat window. Documentation answers include inline citations so you can open the source page directly. 
1. Select the stop button to interrupt generation early.

If the Chat Agent needs more information before completing your request — for example, which project to target or a connection password — it pauses and presents a structured form. Fill in the form to continue, or cancel it (or type an unrelated message) to dismiss it.

### Use page context

When you open the Chat Agent from a cluster, project, or other resource page, it picks up that page's context automatically. The project and cluster IDs appear as small tags above the input, so you can ask about that resource without naming it — for example, _show me the active alerts_ applies to the cluster you're currently viewing.

### Manage conversations

Each conversation is saved to your HM organization so you can return to it later. Select **New Chat** to start a fresh conversation, or open the sidebar with the menu icon to return to an earlier one.

The Chat Agent gives each new conversation a short auto-generated title. From the sidebar you can also **Rename** a conversation or **Delete** one — deletion can't be undone.

For conversation privacy details, see [Conversation storage and isolation](./security_and_data#conversation-storage-and-isolation).

## Model-dependent capabilities

The Chat Agent's effective capability scales with the strength of the model your administrator configures. A small, low-cost model handles estate browsing and metric lookups, while more complex cluster-creation flows and structured follow-up questions need a stronger model. See [Model selection](enable/#model-selection) for guidance on model selection.

## Documentation scope

The Chat Agent answers HM questions from a versioned documentation snapshot shipped with each release. For details on citation behavior and the knowledge cutoff, see [HM documentation Q&A](./capabilities#hm-documentation-qa).
