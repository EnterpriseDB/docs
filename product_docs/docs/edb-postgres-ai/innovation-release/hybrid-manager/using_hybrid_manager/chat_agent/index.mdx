The EDB Hybrid Manager (HM) **Chat Agent** is an in-product, conversational assistant that helps you operate HM in plain language. Ask it about your estate, provision and inspect Postgres clusters, look up active alerts, troubleshoot performance, and consult the HM documentation — all without leaving the HM console.

## What you can do with it

The Chat Agent helps you with a focused set of HM tasks. At a glance:

- Browse and manage projects, clusters, users, and roles.
- Provision a new Postgres cluster — Single Node, Primary/Standby HA, Advanced HA, Distributed HA (PGD), or Lakehouse Analytics — by describing what you want.
- Inspect cluster health: current and historical metrics, active alerts, and recommendations.
- Get configuration, indexing, statistics, and security recommendations for a cluster.
- Compare and analyze 24-hour per-node cluster reports.
- Translate Oracle SQL to EDB Postgres-compatible SQL.
- View Data Migration Service status (read-only).
- Ask conceptual or how-to questions about HM and get answers grounded in the official documentation, with citation links.

For the complete catalog, see [What the Chat Agent can do](./capabilities).

## What it does not do

The Chat Agent is intentionally scoped:

- It answers HM-related questions only. Off-topic questions — general knowledge, math, creative writing, or other products — are politely declined.
- It can't perform actions you yourself can't perform. The Chat Agent uses your identity and inherits your roles. See [Security and data handling](./security_and_data).
- It asks for explicit confirmation before any destructive action (create, update, delete, or DDL).
- The Data Migration Service capability is read-only. The Chat Agent directs you to the HM console to start, pause, resume, or delete migrations.

## Where to find it

The Chat Agent is available on every authenticated page in the HM console.

1. Look for the rainbow **Sparkle** icon at the top right of the HM header.
2. Select the icon to open the Chat Agent.

If the icon is missing or disabled, see [Troubleshooting](./troubleshooting).

The Chat Agent opens in two layouts:

- **Full screen** — the default. The conversations list appears on the left.
- **Floating panel** — a compact, draggable panel docked at the bottom right of the HM console. Use the expand or shrink button to switch between modes.

## Talking to the Chat Agent

A typical interaction looks like:

1. (Optional) Select a **model** if your deployment exposes more than one.
2. Type your question into **Ask me anything…** and press **Enter** to send. Press **Shift+Enter** to add a new line.
3. The Chat Agent streams its response. To stop generation early, select the stop button.
4. To start a new conversation, select **New Chat**.
5. To revisit a saved conversation, open the conversations sidebar with the menu icon.

When the Chat Agent needs more information to act safely, it pauses and presents a structured form — for example, *which project?* or *the connection password?*. Answer the form to continue. Cancel or send an unrelated message to clear the pending question.

### Conversations

Your conversations are persisted to your HM organization so you can resume them later. Each conversation is private to you — other users, including organization owners, can't read or list it. See [Conversation storage and isolation](./security_and_data#conversation-storage-and-isolation) for the full access model.

The Chat Agent auto-generates a short subject when a conversation begins. From the conversations sidebar, you can:

- Resume an earlier conversation.
- **Rename** a conversation.
- **Delete** a conversation. This action can't be undone.

### Page context

Open the Chat Agent on a cluster, project, or other detail page, and it picks up the page context. Typically the project and cluster IDs appear as small tags above the input. Asking *show me the active alerts* then implicitly applies to that resource.

## Capabilities depend on the configured model

The Chat Agent's effective capability scales with the strength of the model your administrator configures. A small, low-cost model handles estate browsing and metric lookups, while richer cluster-creation flows and structured follow-up questions need a stronger model. See [Choosing a model](./enable#choosing-a-model) for guidance on model selection.

## Knowledge currency

The Chat Agent answers conceptual and how-to questions about HM from a snapshot of the official HM documentation that ships with each HM release. The snapshot is stamped with the date it was taken.

When the Chat Agent answers from the snapshot, it cites the doc pages it read so you can open them in full. For topics that post-date the snapshot, the Chat Agent says so and refers you to the official EDB Hybrid Manager documentation site for further reading.
