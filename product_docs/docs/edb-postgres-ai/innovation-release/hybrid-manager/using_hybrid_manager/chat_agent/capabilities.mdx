The Chat Agent helps you operate Hybrid Manager (HM) in plain language, grouped by HM area. Available capabilities depend on the configured model's strength tier — see [Choosing a model](./enable#choosing-a-model). All capabilities respect your HM permissions and project scope — see [Security and data handling](./security_and_data).

## HM documentation Q&A

Answer conceptual or how-to questions about HM, AI Factory, Lakehouse, and migrations. Answers come from a snapshot of the official HM documentation that ships with each HM release, stamped with a knowledge cutoff date. The Chat Agent cites the pages it read. For topics that post-date the snapshot, the Chat Agent says so and refers you to the official EDB Hybrid Manager documentation site.

Try:

- *How does Distributed HA differ from Advanced HA?*
- *What's the recommended retention setting for backups in production?*

## Estate, projects, users, and roles

Browse and manage projects, organization or project users, and role assignments. Includes organization and project activity feeds. Doesn't delete users or configure identity providers — use [Configuring IdPs in the HM console](../config_idp_hm) for IdP setup.

Try:

- *Show me my projects.*
- *Add `alice@example.com` to `marketing-sandbox` as a Project Editor.*

## Postgres Cluster lifecycle management

List, inspect, create, delete, and restore Postgres clusters in any HM-supported topology: Single Node, Primary/Standby HA, Advanced HA (AHA), Distributed HA (PGD), and Lakehouse Analytics. Restore supports point-in-time recovery (PITR) and targeted-database restore. The Chat Agent inspects backups and lists supported Postgres image versions. Long-running actions run as background tasks — the Chat Agent returns a task reference and can check progress.

Try:

- *Create a Single Node Postgres 17 cluster in `marketing-sandbox`, location `aws-eu-west-1`, 4 vCPU, 16 GB memory.*
- *Restore `customers-prod` to 12:00 UTC yesterday into a new cluster called `customers-restored`.*

## Cluster monitoring, alerts, and observability

Get current or historical values of standard cluster metrics. Available metrics cover CPU, memory, storage, I/O, throughput, network, connections, transactions, query rate and latency, buffer cache hit ratio, top databases, write-ahead logs, and checkpoints. Render metrics over time as charts. View active alerts per cluster or per project. Mirrors the **Cluster Monitoring** page in the HM console. Doesn't create or modify dashboards, acknowledge alerts, or silence alerts — use the HM console for those.

Try:

- *Plot CPU usage on `customers-prod` over the past 24 hours.*
- *Compare query latency on `orders-prod` between yesterday and today.*

## Recommendations and advisories

Get configuration, indexing, statistics, and security recommendations for a cluster, plus aggregated health scores across a project. Apply an index recommendation directly — the Chat Agent asks for confirmation, then runs the work as a background task.

Try:

- *Are there any pending index recommendations on `orders-prod`?*
- *Apply the recommended index on `orders.line_items.customer_id`.*

## 24-hour cluster reports

List, fetch, analyze, and compare the 24-hour per-node performance reports for a cluster. HM produces these on a fixed cadence — newly created clusters have no reports for about a day.

Try:

- *Compare yesterday's and today's cluster reports for `customers-prod` and tell me what changed.*

## Schema design recommendations

Get table or schema improvement suggestions for a specific cluster, database, and table, driven by the top running queries. Requires query traffic — clusters without query traffic yield no recommendations.

Try:

- *Suggest schema improvements for table `public.orders` on `customers-prod`, database `app`.*

## Lakehouse and analytics catalogs

List, inspect, create, update, and delete Iceberg catalogs.

Try:

- *List the Iceberg catalogs in project `analytics-shared`.*

## Data Migration Service (read-only)

View existing migrations, sources, destinations, table mappings, error history, and assessment reports. Translate Oracle SQL to EDB Postgres-compatible SQL. To start, pause, resume, or delete migrations, use the HM console — the Chat Agent points you to the right page.

Try:

- *Show me the failing migrations in project `migrations-prod` and the most recent error for each.*
- *Translate this Oracle SQL: `SELECT NVL(name, 'unknown') FROM customers …`*

## Cross-cutting limits

These rules apply to every capability:

- **HM-only scope.** Off-topic questions (general knowledge, math, other products) are out of scope.
- **Confirmation gates.** Create, update, delete, drop, DDL, and DML operations require an explicit *yes* before execution.
- **Permission-honest.** On a permission-denied error, the Chat Agent stops and reports the error rather than retrying or routing around it.
- **Status comes from a status check.** Listing a resource doesn't imply it's healthy or active — the Chat Agent calls a status endpoint when needed.

For the security model behind these guarantees, see [Security and data handling](./security_and_data).
