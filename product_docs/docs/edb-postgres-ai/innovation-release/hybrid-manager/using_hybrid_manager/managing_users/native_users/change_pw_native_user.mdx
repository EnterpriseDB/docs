To avoid unauthorized access to your environment, change the password of the default native user immediately after deployment.

Native user credentials are stored in the `hm-portal-bootstrap` Kubernetes secret. To change a password, you update this secret with a new bcrypt-hashed password.

## Changing a native user password

1. Generate a new bcrypt-hashed password:

   ```shell
   NEW_PASSWORD="your-new-password"
   NEW_HASH=$(echo -n "$NEW_PASSWORD" | htpasswd -BinC 10 admin | cut -d: -f2)
   echo "New hash: $NEW_HASH"
   ```

   The output displays the new hashed password, for example:
   ```
   New hash: $2y$10$YDHMEnc8VZKnhvze538ePOLKol7dFH.S7IgpqC3.X3R/3P2Q9zRcG
   ```

2. Retrieve the current secret content:

   ```shell
   kubectl get secret hm-portal-bootstrap -n default -o jsonpath='{.data.static-passwords\.yaml}' | base64 -d > static-passwords.yaml
   ```

3. Edit the `static-passwords.yaml` file and replace the `hash` value for the user you want to update:

   ```yaml
   staticPasswords:
     - email: "owner@mycompany.com"
       hash: "$2y$10$YDHMEnc8VZKnhvze538ePOLKol7dFH.S7IgpqC3.X3R/3P2Q9zRcG"
       username: "Owner MyCompany"
       userID: "c5998173-a605-449a-a9a5-4a9c33e26df7"
   ```

   !!! important
   Do not change the `userID` value. Changing it creates a new user identity and orphans any existing data associated with the original user.
   !!!

4. Update the secret with the modified content:

   ```shell
   kubectl patch secret hm-portal-bootstrap -n default \
     --type merge \
     -p "{\"data\":{\"static-passwords.yaml\":\"$(base64 < static-passwords.yaml | tr -d '\n')\"}}"
   ```

5. The secret is automatically replicated to the `upm-dex` namespace, and the Dex deployment automatically restarts to apply the changes.

6. Verify the Dex pods have restarted and are running:

   ```shell
   kubectl get pods -n upm-dex -l app=dex
   ```

The user can now sign in with the new password.
