## Health assessment reports

Health assessment reports give you a point-in-time snapshot of the performance and health of a database cluster node over a given period. You can use them to spot anomalies, track how your system evolves over time, and share a consistent artifact with others — similar to an Oracle AWR report.

Reports are generated automatically: one report per node immediately after a cluster is connected to HM, then once every 24 hours thereafter. Reports are stored persistently, so they remain available even after the underlying data used to generate them has been purged.

### What a report contains

Each report covers a fixed time window and includes aggregate statistics for that period, such as session counts, transaction stats, wait events, top SQL statements, and WAL and buffer metrics.

### Viewing and downloading reports

To access reports:

1. In the HM navigation, go to **Reports**.
   The Reports page lists all reports across your estate, sorted by date. You can filter by project, cluster, date range, and report interval.
2. To download a report, select the report row and click **Download**.
   Reports are available as PDF files.

You can also view reports scoped to a single cluster from the cluster detail page.

### Comparing two reports

You can ask the HM chatbot to compare two reports for the same cluster. This surfaces the delta between two snapshots — useful for tracking how performance has changed over time.

To compare reports using the chatbot:

1. Open the **ChatBot** in HM.
2. Ask the chatbot to compare reports for a specific node and cluster — for example: *"Compare the last two reports for the primary node of cluster X."*

The chatbot returns a summary of the differences between the two reports.
