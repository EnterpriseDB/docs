When you configure an automation with **Approval Required**, Hybrid Manager (HM) doesn't run the operation immediately. Instead, it creates a pending task and waits for a team member to review it. Task Manager is where that review happens — it gives you a centralized view of all pending, running, and completed automation tasks across your project, so you can stay in control of what runs and when.

## Accessing Task Manager

Task Manager is available at two levels:

- **Project level** — In the left navigation, select **Task Manager**. This view shows all tasks across every cluster in the project.
- **Cluster level** — Open a cluster and select the **Task Manager** tab. This view is filtered to tasks for that cluster only.

## Understanding task status

Each task moves through a lifecycle that you can track in the **Status** column:

| Status | Meaning |
|---|---|
| Pending | The task is waiting — either for approval (if approval is required) or for its maintenance window to open. |
| Gate Satisfied | Approval was granted. The task is scheduled and will execute when its maintenance window opens, or immediately if no window is set. |
| Running | The task is currently executing. |
| Completed | The task finished. The result is shown as **Success** or **Failed** in the task detail. |
| Failed | The task encountered an error during execution. |
| Cancelled | The task was cancelled before it ran. |

## Approving or rejecting a task

When a task is in **Pending** status and requires your approval:

1. Select the task name or its status badge to open the task detail sidebar.
1. Review the task details — including the automation type, target cluster or data group, and the approval description set by the automation owner.
1. Select **Approve** to allow the task to proceed, or **Deny** to reject it.
1. If the automation was configured with **Require Reason**, add an explanation in the **Reason** field before confirming your decision. If reason is optional, you can leave it blank.

After approval, the task moves to **Gate Satisfied** and executes according to the automation's maintenance window configuration. After rejection, the task is closed. If the automation's cooldown period has passed, HM recreates the task on the next evaluation cycle.

!!! note

    Only users with the appropriate role in the target cluster can approve tasks. HM's RBAC system enforces this — if you don't see the Approve and Deny buttons, you don't have the required permissions.

## Monitoring task history

Task Manager retains a history of all completed, failed, and cancelled tasks. Use it to audit what ran, when, and with what outcome.

To narrow down the list, use the available filters:

- **Status** — Filter by one or more task phases (for example, show only Pending or Failed tasks).
- **Date range** — Limit results to a specific time window. Preset ranges of 7, 15, 30, and 90 days are available, or set a custom range.
- **Search** — Find tasks by name.

Select any task to open its detail sidebar. For completed tasks, the sidebar shows execution start and end times, the number of attempts, and a result summary. For failed tasks, it also includes an error code and message.

All task activity is also recorded in the [Activity Log](../../monitoring/activity_log/), which provides an org- or project-wide audit trail across all resource types, not just tasks.
