The Activity Log is a chronological record of everything that happens in your Hybrid Manager (HM) environment — cluster changes, automation task executions, user actions, access control changes, and more. Use it to understand what changed, who made the change, and when.

The Activity Log is also the event source for [system notifications](../notifications/). When a platform event is recorded in the Activity Log — such as an automation task completing or a recommendation being applied — HM uses that event to trigger any notification channels you've configured at the project or organization level.

## Accessing the Activity Log

The Activity Log is available at two levels:

- **Project level** — In the left navigation, select **Activity Log**. Shows all events scoped to that project.
- **Organization level** — In the organization navigation, select **Activity Log**. Shows events across all projects in your organization, with a **Project** column to identify the source.

## Reading the log

Each row in the Activity Log represents a single event. The columns are:

| Column | Description |
|---|---|
| Activity | What happened — for example, a cluster was created, a task was approved, or a user role was changed. For task-related events, the activity name includes the task name for easy identification. |
| User | The user or system process that triggered the event. |
| Date | When the event occurred. |
| Resource | The type of object affected, such as Cluster, Task, User, or Project. Clusters and tasks are shown as clickable links that navigate to the relevant detail page. |
| Project | (Organization-level view only) The project the event belongs to. |

## Filtering and searching

Use the filters at the top of the log to narrow down results:

- **Date range** — Preset ranges of 7, 15, 30, and 90 days are available, or set a custom range.
- **Resource** — Filter by resource type (for example, show only Cluster or Task events) and optionally by a specific resource within that type.
- **Project** — (Organization-level view only) Filter events to one or more specific projects.
- **Search** — Find events by name or resource.

## Exporting activity data

Select **Export** to download the current filtered view as a CSV file. This is useful for compliance audits or sharing activity history with stakeholders outside HM.

## Connecting activity to notifications

Any event recorded in the Activity Log can trigger a system notification if you've configured a notification channel (webhook, Pager Duty, or email) at the project or organization level. This means you don't need to monitor the Activity Log manually — configure your channels and HM will push the relevant events to your external tools automatically.

See [Notifications](../notifications/) for steps on configuring channels and managing which events you receive.
