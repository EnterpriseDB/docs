Event markers show activity log events as visual indicators on time-series charts in Hybrid Manager (HM). Use them to correlate metric changes with specific cluster events, or as boundaries when selecting a time period to analyze.

## Marker types

HM displays two types of event markers:

- **Point-in-time markers** — Displayed as a vertical line at the event's timestamp. Use these to mark a single moment, such as a configuration change or an alert.
- **Interval markers** — Displayed as a shaded area between a start and end time. Use these to mark a duration, such as a maintenance window or a period of degraded performance.

## Viewing event markers

Event markers appear on monitoring charts in the **Monitoring** tab. You can view them at the cluster or node level.

To view event markers:

1. In the HM console, select your project.
2. Select **Clusters > Postgres Clusters**.
3. Select the **Monitoring** tab.
4. From the event type selector, select the event types you want to display.

Hovering over a marker displays its metadata — name, type, and description.

## Toggling markers

Use the event type selector to show or hide markers by section, chart, or marker type. Marker visibility can be controlled independently per type.

!!! Note
Alerts appear as event markers but aren't logged in the activity log. Alert markers reflect only the alert's start time. When the underlying issue is resolved, the marker disappears.
!!!

## Using markers as time delimiters

Instead of the date and time pickers, you can use event markers to define the time range displayed on monitoring charts:

- For a **point-in-time marker**: use it as the start or end of the time range, or set it as both ends with a time buffer. You can also pair a marker on one end with a datetime on the other.
- For an **interval marker**: zoom the monitoring page to the marker's full duration. You can add an optional time buffer on either side.

## System-generated annotations

HM automatically generates annotations for certain events so they appear on your monitoring charts without manual action. These include cluster spec updates — whether triggered by a user, autoscaling, or an agentic automation.

## Custom event markers

You can create a custom event marker at any point in time by sending a POST request to `/api/v1/projects/<project_id>/custom-event`:

```json
{
  "message": "The text to attach to your event",
  "clusterId": "<cluster_id>"
}
```

!!! Note
Events with a `clusterId` appear in the activity log and as annotations on charts. Events without a `clusterId` appear only in the activity log.
!!!
