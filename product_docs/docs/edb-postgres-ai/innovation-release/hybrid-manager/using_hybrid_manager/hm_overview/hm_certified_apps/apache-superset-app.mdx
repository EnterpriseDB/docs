Apache Superset is an open-source business intelligence web application that provides data visualization, dashboard creation, and SQL-based data exploration for databases connected to EDB Hybrid Manager.

Key features:
- No-code interface for building charts
- Web-based SQL Editor for querying
- Semantic layer for defining custom dimensions and metrics
- Support for SQL databases and data engines
- A range of visualization types, including bar charts and geospatial visualizations
- A configurable caching layer to reduce database load
- Configurable security roles and authentication options
- An API for programmatic customization
- Cloud-native architecture designed for scale



## Apache Superset components

- **Core application** - Includes a Python (Flask) backend, an API layer, and a React frontend. When a user loads a chart or dashboard, Superset queries the configured data warehouse and renders the result as a visualization.
- **Metadata database** — Stores chart and dashboard definitions, user information, and logs. PostgreSQL and MySQL support production use. Avoid SQLite for production; some installation methods use it by default.
- **Caching layer (optional)** — Reduces data warehouse load by storing query results, and acts as a message broker for the worker process. Redis is the most common choice.
- **Worker and beat (optional)** — Handles asynchronous task execution and scheduling, typically via Celery.

Enable the caching layer and worker to access the following features:

- Alerts and reports
- Caching
- Async queries
- Dashboard thumbnails


## Requirements

### Technical requirements:

1. External PostgreSQL: Required for metadata storage (PostgreSQL 12+).
1. Resources: Worker pods default to 1Gi Memory and 250m CPU.

### Configuration requirements:

1. Secret Key: Provide a unique `secretKey` for cryptographic operations.
1. Database Backend: Host, port, and credentials for the metadata database.

### Supported data sources: PostgreSQL, and other SQLAlchemy-compatible databases.

### Installation steps:

1. Configure the `secretKey` and PostgreSQL connection in the Beacon values schema.
1. Set the replica count for the Superset nodes (default 1).
1. Install via Beacon or Helm CLI.

### Post-installation steps:

1. Access the UI via HTTPS.
1. Go to **Settings** > **Database Connection** to add your own data clusters for analysis.


### Application database

Superset requires an external database to store its application metadata, including questions, dashboards, and settings. You must create this database before deploying Superset.
EDB recommends PostgreSQL for production. Superset also supports MySQL 8.0.17 or later and MariaDB 10.2.2 or later.

Have these connection details ready before installation:
- Host and port
- Database name
- Username and password


### Memory

Superset uses approximately one quarter of available system RAM by default. Allocate at least 2 GB for Superset, reserving additional capacity for other system processes.

### Supported data sources

Superset officially supports drivers for the following databases:
- PostgreSQL (any currently supported version)
- MySQL 8.0.17 or later and MariaDB 10.2.2 or later
- MongoDB, SQL Server, SQLite
- Cloud data warehouses: Amazon Athena, Amazon Redshift, BigQuery, Databricks, Snowflake, SparkSQL, Starburst
- Additional databases: ClickHouse, Druid, Oracle, Presto, Vertica
- Cloud-managed services such as Amazon RDS and Aurora

## Installing Superset in HM

Install Superset directly in Hybrid Manager (HM).
1. Navigate to **Asset Library** then select **Apps** in the HM console.
1. Select Superset and enter required values in the fields.

!!!
Once you have deployed Superset in HM, you can access it in the HM dashboard under **Apps** or **Asset library**.
!!!

## Initial setup

Once you deploy Superset, open the Superset URL from the HM dashboard. A setup wizard in the Superset interface guides you through these steps:

1. Create an admin account. The first account created is an admin account in Superset. This account manages all subsequent user additions, database connections, and system configuration.
1. Select your use case. Choose between self-service analytics, analytics embedded in applications, or both. You can change this later.
1. Connect a database. Provide your database hostname, port, name, and credentials. You can also skip this step and explore the sample database included with Superset.
1. Usage analytics consent. Choose whether to allow anonymous usage data collection. Superset never collects anything about your data or question results.
