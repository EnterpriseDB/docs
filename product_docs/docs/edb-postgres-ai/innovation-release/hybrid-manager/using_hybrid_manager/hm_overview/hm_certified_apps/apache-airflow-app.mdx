Apache Airflow® is an open-source workflow orchestration platform deployed and managed within Hybrid Manager (HM). Its extensible Python framework supports integration with external technologies, and a web-based UI accessible from the HM dashboard provides visibility into workflow execution and debugging. Airflow supports both single-node and distributed deployments.

### Key features:

- Web-based UI for monitoring, managing, and debugging workflow execution.
- Python-based workflow definitions using Directed Acyclic Graphs (DAGs).
- Built-in scheduler supporting cron-based and event-driven triggers.
- Extensible operator library for integrating with external systems and services.
- Git synchronization for deploying DAG files from a repository.
- Distributed task execution across multiple workers.

## Workflows as code in Apache Airflow

Airflow workflows are defined entirely in Python:

- Pipelines are defined in code, supporting dynamic DAG generation and parameterization.
- Built-in operators can be extended for custom integrations.
- Jinja templating is supported for runtime variable substitution.

### Supported data sources

Airflow connects to nearly any data source via [Connections](https://airflow.apache.org/docs/apache-airflow/stable/authoring-and-scheduling/connections.html), including PostgreSQL, MySQL, AWS, and Snowflake. Airflow's Connection stores credentials and other information necessary for connecting to external services.

## Requirements

### Technical requirements:

- Python: v3.10+
- **(Required) Database:** Airflow requires an external database to store workflow state, task instances, DAG definitions, and run history. You must create this database before deploying Airflow.
  - PostgreSQL v13+ is recommended for production. Have the host, port, database name, username, and password ready before installation. You can obtain these from the **Connect** tab of your cluster in the HM console.
  - Airflow also supports MySQL 8.0.17+ and MariaDB 10.2.2+.
- **(Required) Security keys:** Airflow uses [Fernet](https://airflow.apache.org/docs/apache-airflow/stable/security/secrets/fernet.html) to encrypt passwords in the connection configuration and the variable configuration (minimum 32 characters).
  - Generate the **Fernet key** by running: `openssl rand -base64 32`.
  - Generate the **APIsecretkey** by running: `openssl rand -base64 32`.
- **(Optional) Storage:** Git Sync configuration for syncing DAGs from public or private Git repositories using SSH keys. If using Git sync, prepare the repository URL, branch name, and SSH private key.

## Deploying Airflow

1. In the HM console, navigate to **Asset Library**, then select **Apps**.
1. Select **Apache Airflow** and select **Deploy**.
1. Under **Identity**, enter a name for your deployment and select the target project.
1. Under **Parameters**, fill in the required fields:

   Where:

   * `apiSecretKey` is the secret key for API authentication. Generate with `openssl rand -base64 32`.
   * `fernetKey` is the key for encrypting metadata database connections. Generate with `openssl rand -base64 32`.
   * `data.metadataConnection` is the external PostgreSQL connection details: database name, host, port, password, username, SSL mode (`require` recommended), and protocol (`postgresql`).
   * `dags.gitSync.enabled` is the toggle to enable syncing DAGs from a Git repository.
   * `dags.gitSync.repo` is the Git repository URL containing your DAGs.
   * `dags.gitSync.branch` is the branch to sync from (default: `main`).
   * `dags.gitSync.subPath` is the subfolder in the repository containing DAG files (default: `dags`).
   * `dags.gitSync.sshKey` is the SSH private key for authenticating to a private Git repository.
   * `dags.gitSync.wait` is the seconds between Git sync intervals (default: `60`).
   * `dagProcessor.replicas` is the number of DAG processor replicas — Airflow 3.0+ only (min: 1, max: 3).
   * `scheduler.replicas` is the number of scheduler replicas (min: 1, max: 3).
   * `web.defaultUser` is the default admin username for the Airflow UI.

1. Select **Deploy** to install Airflow.

### Post-deployment configuration

Once Airflow is deployed, complete the following steps to finish your setup:

1. Open **Airflow** from the HM console.
1. Log in with the admin username you set in `web.defaultUser` during deployment.
1. Navigate to **Admin > Connections** to add credentials for any external databases or services your DAGs will connect to.
1. If you enabled Git sync, verify that your DAGs are appearing in the Airflow UI.
