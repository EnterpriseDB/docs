!!! important
Regular upstream releases deliver security updates. Keep your pgAdmin instance up to date to receive the latest patches.
!!!

pgAdmin 4 is an open-source web-based interface for administering PostgreSQL databases. When deployed as a certified app in Hybrid Manager, it provides remote access to your PostgreSQL clusters for managing database objects and executing queries.

!!! warning
pgAdmin 4 is not supported on Red Hat OpenShift (RHOS).
!!!

## Requirements

### Application database

pgAdmin requires an external database to store its application metadata, including questions, dashboards, and settings. You must create this database before deploying pgAdmin.
EDB recommends PostgreSQL for production. pgAdmin also supports MySQL 8.0.17 or later and MariaDB 10.2.2 or later.

Have these connection details ready before installation:
- Host and port
- Database name
- Username and password

## Memory
pgAdmin uses approximately one quarter of available system RAM by default. Allocate at least 2 GB for pgAdmin, reserving additional capacity for other system processes.


## Supported data sources

pgAdmin supports drivers for the following databases:
- PostgreSQL (any currently supported version)
- MySQL 8.0.17 or later and MariaDB 10.2.2 or later
- MongoDB, SQL Server, SQLite
- Cloud data warehouses: Amazon Athena, Amazon Redshift, BigQuery, Databricks, Snowflake, SparkSQL, Starburst
- Additional databases: ClickHouse, Druid, Oracle, Presto, Vertica
- Cloud-managed services such as Amazon RDS and Aurora


## Installing pgAdmin in HM

Install pgAdmin directly in Hybrid Manager (HM).
1. Navigate to **Asset Library** then select **Apps** in the HM console.
1. Select **pgAdmin** and enter required values in the fields.

!!! note
Once you have deployed pgAdmin in HM, you can access it in the HM dashboard under Apps or Asset library.
!!!

## Initial setup

Once you deploy pgAdmin, open the pgAdmin URL from the HM dashboard. A setup wizard in the pgAdmin interface guides you through these steps:

1. Create an admin account in pgAdmin to manage all subsequent user additions, database connections, and system configuration.
1. Choose between self-service analytics, analytics embedded in applications, or both. You can change this later.
1. Provide your database hostname, port, name, and credentials. 

!!! note
You can also skip this step and explore the sample database included with pgAdmin.
!!!

1. Choose whether to allow anonymous usage data collection. pgAdmin never collects anything about your data or question results.
