The Hybrid Manager Marketplace enables the automated deployment of curated Open Source Software (OSS) directly into your K8s environment. Using standardized .yaml package files, Hybrid Manager streamlines container image retrieval and Kubernetes orchestration for tools like Apache Airflow, pgAdmin4, and Metabase. This framework ensures consistent, repeatable installation and lifecycle management across your Hybrid Manager estate.

## Supported application versions

- pgAdmin4: v9.11
- Metabase: 0.58.7
- Apache Airflow: v3.0.2
- Apache Superset: v6.0. 1rc1
- HammerDB: v5.0
- PGBadger: v13.2

## Technical requirements

- pgAdmin4: 512Mi Memory, 500m CPU, (Limits: 1Gi / 1000m).
- Metabase: 1Gi memory, 800m CPU (Limits: 2Gi / 1000m).
- Superset/Airflow: 1Gi Memory, 800m CPU (Limits: 2Gi / 1000m).

## Configuration requirements

- **External Database**: Both Metabase and Superset require an existing PostgreSQL instance. Customers need to have the Host, Port, Database Name, and Credentials ready.
- **Security Keys**: For Airflow and Superset, customers will need to generate their own Fernet Key and SecretKey (using openssl) to secure their installation.
- **Namespace Requirements**: The apps appear to be designed to run within specific Kubernetes namespaces (e.g., default or upm-beacon).
