Metabase is an open-source tool that provides overview over your data. It is available in the Hybrid Manager Marketplace as a community-supported OSS application.

## Deployment and architecture

Metabase deploys as a containerized application within your Hybrid Manager Kubernetes environment using a `.yaml` package file. This deployment ensures database traffic remains entirely within your private network (VPC/VNET) and security perimeter. The architecture supports horizontal scaling and external PostgreSQL application databases for high availability.

## Requirements

### Application database

Metabase requires an external database to store its application metadata, including questions, dashboards, and settings. You must create this database before deploying Metabase.

PostgreSQL is recommended for production. MySQL 8.0.17 or later and MariaDB 10.2.2 or later are also supported.

Have these connection details ready before installation:

- Host and port
- Database name
- Username and password

### Memory

Metabase uses approximately one quarter of available system RAM by default. Allocate at least 2 GB for Metabase, reserving additional capacity for other system processes.

### Supported data sources

Metabase officially supports drivers for the following databases:

- PostgreSQL (any currently supported version)
- MySQL 8.0.17 or later and MariaDB 10.2.2 or later
- MongoDB, SQL Server, SQLite
- Cloud data warehouses: Amazon Athena, Amazon Redshift, BigQuery, Databricks, Snowflake, SparkSQL, Starburst
- Additional databases: ClickHouse, Druid, Oracle, Presto, Vertica
- Cloud-managed services such as Amazon RDS and Aurora

## Installing Metabase in HM

Metabase is installed directly in Hybrid Manager (HM). 

1. Navigate to **Asset Library** then select **Apps** in the HM console.
1. Select **Metabase** and enter required values in the fields.

!!!note
Once you have deployed Metabase in HM, you can access it in the HM dashboard under **Apps** or **Asset library**.
!!!

## Initial setup

Once Metabase is deployed, open the Metabase URL from the HM dashboard. A setup wizard in the Metabase interface guides you through these steps:

1. **Create an admin account.** The first account created is an admin account in Metabase. This account manages all subsequent user additions, database connections, and system configuration.
2. **Select your use case.** Choose between self-service analytics, analytics embedded in applications, or both. You can change this later.
3. **Connect a database.** Provide your database hostname, port, name, and credentials. You can also skip this step and explore the sample database included with Metabase.
4. **Usage analytics consent.** Choose whether to allow anonymous usage data collection. Metabase never collects anything about your data or question results.

## Support resources

As Metabase OSS is community-supported, use these resources for technical guidance and updates:

- [Metabase discussion forum](https://discourse.metabase.com/) — peer-to-peer support from the community.
- [Metabase documentation](https://www.metabase.com/docs/latest/) — self-service deployment and feature guides, as well as additional information on Metabase installation and configuration.
