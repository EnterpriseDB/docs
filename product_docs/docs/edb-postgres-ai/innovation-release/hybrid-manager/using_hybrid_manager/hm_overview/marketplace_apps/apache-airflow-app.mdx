Apache Airflow® is an open-source workflow orchestration platform deployed and managed within Hybrid Manager (HM). Its extensible Python framework supports integration with a wide range of external technologies, and a web-based UI accessible from the HM dashboard provides visibility into workflow execution and debugging. Airflow scales from single-node deployments to distributed configurations for large-scale workloads.

## Workflows as code in Apache Airflow

Airflow workflows are defined entirely in Python. This “workflows as code” approach brings several advantages:

- **Dynamic:** Pipelines are defined in code, enabling dynamic Dag generation and parameterization.
- **Extensible:** The Airflow framework includes a wide range of built-in operators and can be extended to fit your needs.
- **Flexible:** Airflow leverages the Jinja templating engine, allowing rich customizations.


## Requirements

### Technical Requirements:
  1. External PostgreSQL: Required for Airflow's metadata store.
  1. Storage: Git repository access for DAG management.

### Configuration Requirements:
  1. Security Keys: fernetKey and apiSecretKey (minimum 32 characters) are required for encryption.
  1. Git Sync: Optional configuration for syncing DAGs from public or private Git repositories using SSH keys.

### Supported Data Sources
Airflow connects to nearly any data source via "Connections," including PostgreSQL, MySQL, AWS, and Snowflake.

### Installation Steps:
  1. Generate a Fernet key: openssl rand -base64 32.
  1. Provide the external PostgreSQL connection string in the Beacon UI.
  1. (Optional) Provide Git repository URL and SSH key for DAG syncing.

### Post-Installation Steps:
  1. Log in to the Airflow UI via Beacon.
  1. Navigate to Admin -> Connections to add credentials for your external databases.
  1. Verify DAGs are appearing via the Git Sync process.


### Application database
Airflow requires an external database to store its application metadata, including questions, dashboards, and settings. You must create this database before deploying Airflow.
PostgreSQL is recommended for production. MySQL 8.0.17 or later and MariaDB 10.2.2 or later are also supported.

Have these connection details ready before installation:
- Host and port
- Database name
- Username and password

### Memory
Airflow uses approximately one quarter of available system RAM by default. Allocate at least 2 GB for Airflow, reserving additional capacity for other system processes.

### Supported data sources
Airflow officially supports drivers for the following databases:
- PostgreSQL (any currently supported version)
- MySQL 8.0.17 or later and MariaDB 10.2.2 or later
- MongoDB, SQL Server, SQLite
- Cloud data warehouses: Amazon Athena, Amazon Redshift, BigQuery, Databricks, Snowflake, SparkSQL, Starburst
- Additional databases: ClickHouse, Druid, Oracle, Presto, Vertica
- Cloud-managed services such as Amazon RDS and Aurora

## Installing Airflow in HM
Airflow is installed directly in Hybrid Manager (HM).
Navigate to Asset Library then select Apps in the HM console.
Select Airflow and enter required values in the fields.

!!! note
Once you have deployed Airflow in HM, you can access it in the HM dashboard under **Apps** or **Asset library**.
!!!

### Initial setup
Once Airflow is deployed, open the Airflow URL from the HM dashboard. A setup wizard in the Airflow interface guides you through these steps:
1. Create an admin account. The first account created is an admin account in Airflow. This account manages all subsequent user additions, database connections, and system configuration.
1. Select your use case. Choose between self-service analytics, analytics embedded in applications, or both. You can change this later.
1. Connect a database. Provide your database hostname, port, name, and credentials. You can also skip this step and explore the sample database included with Airflow.
1. Usage analytics consent. Choose whether to allow anonymous usage data collection. Airflow never collects anything about your data or question results.
