## Deployment and Architecture
Metabase is a production-ready application that deploys via Docker or Helm within your private network (VPC/VNET) to ensure database traffic remains within your security perimeter. The architecture supports horizontal scaling and external PostgreSQL system databases for high availability. By leveraging Private Link and Reverse Proxy configurations, Metabase enables zero-trust connectivity and centralized access management trough providers such as Azure AD, Okta, or Google Workspace.

## Requirements
To ensure a successful deployment, your environment must meet the following hardware and software specifications: 
- Deployment requires an external PostgreSQL or MySQL instance to store application metadata, such as questions and dashboards.
- The platform is compatible with any OCI-compliant orchestrator, including Kubernetes and OpenShift, and requires a minimum of 1GB of persistent storage for configuration settings.
- Supported data sources include PostgreSQL 12 or higher, EDB, and cloud-managed services such as RDS, Aurora, and Azure.


Metabase maintains rigorous security standards to protect data integrity and ensure regulatory compliance: The platform ensures data sovereignty by operating entirely within your private Hybrid Manager environment, preventing sensitive information from leaving the secure network. Identity management is handled through integrated SSO (OIDC/SAML), while SSL/TLS encryption secures all database connections. The environment is further hardened through daily CVE scans to ensure consistent compliance with the CIS benchmark.

## Integration in HM console

Follow these steps to integrate the application with your existing infrastructure: Users can deploy Metabase into the Hybrid Manager environment using the provided YAML manifests. Once the installation is complete, the instance is accessible and manageable directly through the Hybrid Manager dashboard.

## Getting Started with Metabase OSS

To ensure your Metabase instance is correctly configured and connected, follow this initialization sequence: Initialize the container and create an administrator account before connecting your database via a standard connection string, such as Snowflake or PostgreSQL. Once connected, allow Metabase to synchronize the schema and automatically map your tables. You can verify connectivity by selecting Ask a Question and choosing a table to generate an initial automatic visualization.

### Exploration Tools

A pre-configured H2 sample dataset is included for immediate feature exploration. Users can view pre-built dashboards such as Daily Active Users and Revenue Growth, or select Show Editor on any sample chart to inspect the specific filters and aggregations used to build it.

### Troubleshooting
Review these solutions for the Community OSS edition to resolve common connectivity and performance issues: Ensure the database firewall permits traffic from the Metabase container IP to resolve connection failures. To address dashboard latency, implement data caching to improve response times for frequent queries. If memory errors occur in large-scale deployments, adjust the JAVA_OPTS environment variable to optimize resource allocation.
