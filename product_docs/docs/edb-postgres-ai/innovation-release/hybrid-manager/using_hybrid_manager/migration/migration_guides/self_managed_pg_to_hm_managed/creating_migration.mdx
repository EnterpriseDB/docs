After you configured the Data Migration Service (DMS) agent in reader mode to establish connections to Hybrid Manager (HM), create a migration in the HM console. A migration establishes a sync between the source database and a destination database with the help of DMS.

<TabContainer syncKey="pg_schema_path">
<Tab title="Simplified (recommended)">

1.  In the HM console, select the project from which you want to manage the migration.

1.  In the project, select **Migrate > Migrations**.

1.  In the **Migrations** page, select **Create New Migration**.

1.  In the **Create OLTP Migration** page, assign a name to the migration.

1.  Select the source of the migration. The ID for the DMS agent is listed in the **Source** menu.

1.  Under **Scope**, select **Schema and data**. When you select this option, HM:

    1. Sources the schema from the Migration Portal project that was created automatically when you registered your database with the EDB Postgres AI agent.
    1. Creates tables on the destination without constraints, so the initial snapshot copy isn't slowed by constraint validation.
    1. After the snapshot completes, applies primary key and unique constraints (for tables that have them) so DMS can deliver streaming changes exactly-once. Tables without a PK or UK fall back to at-least-once delivery and require deduplication during verification.
    1. Defers the remaining schema (indexes, foreign keys, check constraints, `NOT NULL`, views, triggers) until cutover. You can optionally apply most of this work concurrently with streaming via **Finalize Schema Concurrently** to shrink the cutover window — see [Finalizing migration](finalizing_migration).
    1. After you mark the migration as completed, validates deferred foreign keys, rebuilds sequences, and applies anything still pending in a final **Schema Finalization** stage.
    1. Excludes foreign key, check, and exclusion constraints during migration to avoid performance degradation.
    1. Reapplies the excluded constraints automatically after the migration completes.

1.  Under **Destination Type**, select an HM-managed destination database cluster and database for the migration. Then, select **Next**.

1.  Select the tables and columns to migrate. Modify the table and column names if needed.

1.  Select **Create Migration**.

The HM console displays a new migration. If you chose **Snapshot + Streaming**, the DMS agent streams data continuously while the migration is in the **Running** state.

!!!note
The DMS agent in reader mode streams data changes. It doesn't stream changes in DDL objects.
!!!

</Tab>
<Tab title="Manual">

Before creating the migration, ensure you have already exported your schema, excluded foreign key, check, and exclusion constraints, and imported the schema to the destination database. See the previous **Migrating schemas** step for instructions.

1.  In the HM console, select the project from which you want to manage the migration.

1.  In the project, select **Migrate > Migrations**.

1.  In the **Migrations** page, select **Create New Migration**.

1.  In the **Create OLTP Migration** page, assign a name to the migration.

1.  Select the source of the migration. The ID for the DMS agent is listed in the **Source** menu.

1.  Select the migration type: [**Snapshot**](/edb-postgres-ai/current/hybrid-manager/using_hybrid_manager/migration/terminology/#snapshot-migration) or [**Snapshot + Streaming**](/edb-postgres-ai/current/hybrid-manager/using_hybrid_manager/migration/terminology/#snapshot--streaming-migration).

1.  Under **Scope**, select **Data only**. The schema was already imported manually in the previous step.

1.  Under **Destination Type**, select an HM-managed destination database cluster and database for the migration. Then, select **Next**.

1.  Select the tables and columns to migrate. Modify the table and column names if needed.

1.  Select **Create Migration**.

The HM console displays a new migration. If you chose **Snapshot + Streaming**, the DMS agent streams data continuously while the migration is in the **Running** state.

!!!note
The DMS agent in reader mode streams data changes. It doesn't stream changes in DDL objects.
!!!

</Tab>
