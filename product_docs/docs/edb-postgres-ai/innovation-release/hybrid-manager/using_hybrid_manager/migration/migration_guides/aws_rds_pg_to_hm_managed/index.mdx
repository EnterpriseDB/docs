These are all the steps you need to perform a migration from an AWS RDS for PostgreSQL or Aurora PostgreSQL database to a database managed by Hybrid Manager (HM).

## Set up your auxiliary machine

Configure the machine that hosts both migration agents. This machine acts as the bridge between your Postgres source and HM.

1.  [Configure your auxiliary machine](auxiliary_machine).

## Prepare your source database

Create the required users and configure your source database for both agents. This process covers creating the HM machine user, the agent OS user, the `beacon-agent` database user and schema extraction permissions, enabling logical replication via an AWS parameter group, the migration role and its grants, and configuration validation.

1.  [Prepare the source AWS RDS/Aurora PostgreSQL database](preparing_source_pg).

## Configure the EDB Postgres AI agent

Connect the agent to your source database and to HM to enable schema ingestion and migration assessment.

1.  [Configure the EDB Postgres AI agent to assess your database](agent_postgres).

## Create a destination database

Create a cluster in the HM instance to host the destination database.

1.  [Create a destination database in HM](creating_destination_db).

## Migrate your schema

Handle incompatibilities and transfer your schemas.

1.  [Handle constraints for schema migration](handling_schema). 

1.  [Migrate schemas](migrating_schema).

## Migrate your data

Execute the data transfer and complete the final cleanup steps.

1.  [Create project credentials for the data migration](creating_project_credentials). 

1.  [Configure the Data Migration Service agent (DMS agent) in reader mode for a data migration](configuring_reader). 

1.  [Verify your source database appears in the HM console](viewing_source).

1.  [Create and perform the data migration](creating_migration).

1.  [Monitor your data migration](monitoring_migration).

1.  [Finalize the migration](finalizing_migration) (for streaming migrations only).

1.  [Apply excluded constraints](applying_constraints).
