After you have configured a DMS agent in reader mode and a DMS agent in writer mode to establish connections to Hybrid Manager (HM), create a migration in the HM console. A migration establishes a sync between the source database and a destination database with the help of DMS.

<TabContainer syncKey="pg_schema_path">
<Tab title="Simplified (recommended)">

1.  In the HM console, select the project from which you want to manage the migration.

1.  In your project, select **Migrate > Migrations**.

1.  In the **Migrations** page, select **Create New Migration**.

1.  In the **Create OLTP Migration** page, assign a name to the migration.

1.  Select the source of the migration. The ID for the DMS agent is listed in the **Source** menu.

1.  Select the migration type: [**Snapshot**](/edb-postgres-ai/current/hybrid-manager/using_hybrid_manager/migration/terminology/#snapshot-migration) or [**Snapshot + Streaming**](/edb-postgres-ai/current/hybrid-manager/using_hybrid_manager/migration/terminology/#snapshot--streaming-migration).

1.  Under **Scope**, select **Schema and data**. When you select this option, HM:

    - Sources the schema from the Migration Portal project created automatically when you registered your database with the EDB Postgres AI agent.
    - Imports primary key, unique, and NOT NULL constraints before migration begins — required for exactly-once delivery.
    - Excludes foreign key, check, and exclusion constraints during migration to avoid performance degradation.
    - Reapplies the excluded constraints automatically after the migration completes.

1.  Under **Destination Type**, select a self-managed instance as the destination for the migration. Then, select **Next**.

1.  Select the tables and columns to migrate. Modify the table and column names if needed.

1.  Select **Create Migration**.

The HM console displays a new migration. If you chose **Snapshot + Streaming**, the DMS agents stream data continuously while the migration is in the **Running** state.

!!!note
The DMS agent in reader mode streams data changes. It doesn't stream changes in DDL objects.
!!!

</Tab>
<Tab title="Manual">

Before creating the migration, ensure you have already exported your schema, excluded foreign key, check, and exclusion constraints, and imported the schema to the destination database. See the previous **Migrating schemas** step for instructions.

1.  In the HM console, select the project from which you want to manage the migration.

1.  In your project, select **Migrate > Migrations**.

1.  In the **Migrations** page, select **Create New Migration**.

1.  In the **Create OLTP Migration** page, assign a name to the migration.

1.  Select the source of the migration. The ID for the DMS agent is listed in the **Source** menu.

1.  Select the migration type: [**Snapshot**](/edb-postgres-ai/current/hybrid-manager/using_hybrid_manager/migration/terminology/#snapshot-migration) or [**Snapshot + Streaming**](/edb-postgres-ai/current/hybrid-manager/using_hybrid_manager/migration/terminology/#snapshot--streaming-migration).

1.  Under **Scope**, select **Data only**. The schema was already imported manually in the previous step.

1.  Under **Destination Type**, select a self-managed instance as the destination for the migration. Then, select **Next**.

1.  Select the tables and columns to migrate. Modify the table and column names if needed.

1.  Select **Create Migration**.

The HM console displays a new migration. If you chose **Snapshot + Streaming**, the DMS agents stream data continuously while the migration is in the **Running** state.

!!!note
The DMS agent in reader mode streams data changes. It doesn't stream changes in DDL objects.
!!!

</Tab>
</TabContainer>
