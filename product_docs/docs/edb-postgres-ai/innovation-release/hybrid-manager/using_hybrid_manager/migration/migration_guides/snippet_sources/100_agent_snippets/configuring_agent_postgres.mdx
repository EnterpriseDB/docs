---
title: Registering EDB Postgres AI agent to assess your database
navTitle: Configuring EDB Postgres AI agent
description: Configure the EDB Postgres AI agent to connect to your Postgres database, register it with HM, and enable schema migration and data migration assessment.
---

Install and configure the EDB Postgres AI agent to register your source database with Hybrid Manager (HM). This enables schema migration within the HM landscape and provides observability data and high-level migration metrics — such as schema count and names, table count, and object count. You can also inspect configuration items in your Postgres database that could affect migration complexity, such as extensions installed, foreign servers, foreign tables, tablespaces, and installed procedural languages.

To register a database with the EDB Postgres AI agent, see:

- [Monitoring self-managed deployments](/edb-postgres-ai/current/hybrid-manager/using_hybrid_manager/monitoring/mon_ext_dbs/mon_with_agent/)

- [Monitoring Cloud Service Provider managed Postgres deployments](/edb-postgres-ai/current/hybrid-manager/using_hybrid_manager/monitoring/mon_ext_dbs/mon_ext_csp/)
