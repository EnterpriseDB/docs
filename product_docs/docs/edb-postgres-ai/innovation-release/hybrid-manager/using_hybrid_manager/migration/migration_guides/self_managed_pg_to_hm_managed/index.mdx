These are all the steps you need to perform a migration from a self-managed Postgres database to a database cluster managed by Hybrid Manager (HM).

## Set up your auxiliary machine

Configure the machine that hosts both migration agents. This machine acts as the bridge between your Postgres source and HM.

1.  [Configure your auxiliary machine](auxiliary_machine).

## Prepare your source database

Create the required users and configure your source database for both agents. This process covers creating the HM machine user, the agent OS user, the `beacon-agent` database user and schema extraction permissions, WAL configuration for change data capture (CDC), the migration role and its grants, SSL, and configuration validation.

1.  [Prepare the source Postgres database](preparing_source_pg).

## Configure the EDB Postgres AI agent

Connect the agent to your source database and to HM to enable schema ingestion and migration assessment.

1.  [Configure the EDB Postgres AI agent to assess your database](agent_postgres).

## Prepare a destination database

Create or identify the cluster in HM that will host the destination database.

1.  [Prepare a destination database in HM](creating_destination_db).

## Migrate your schema

Choose your approach and migrate schemas to the destination database.

1.  [Migrate schemas](migrating_schema).

## Configure the EDB DMS agent

Create migration credentials, configure the DMS agent in reader mode, and verify your source appears in HM.

1.  [Configure the EDB DMS agent](configuring_dms_agent).

## Migrate your data

Create the migration, monitor its progress, finalize streaming, and apply post-migration constraints.

1.  [Migrate your data](migrating_data).
