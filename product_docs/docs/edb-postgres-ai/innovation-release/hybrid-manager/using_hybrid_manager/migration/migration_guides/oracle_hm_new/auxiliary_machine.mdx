You need an auxiliary machine or bastion host to set up the EDB Postgres AI agent (`beacon-agent`) and the DMS agent (`cdcagent`). This host acts as the bridge between your source Oracle database and Hybrid Manager.

## Software

Install the following tools on the auxiliary machine you're using to connect to the Oracle database.

- [Install the EDB Postgres AI agent](/edb-postgres-ai/current/hybrid-manager/using_hybrid_manager/migration/migration_guides/oracle_to_hm_managed/agent_installing/) (`beacon-agent`). Handles database registration and schema assessment.
- [Install the DMS agent](/edb-postgres-ai/current/hybrid-manager/using_hybrid_manager/migration/installing_agent/) (`cdcagent`). Handles data migration and Change Data Capture (CDC).

## Required dependencies

Before running the agents, ensure the following environment requirements are met:

- Java Runtime: The DMS agent requires Java 17 or higher.
- Oracle Instant Client: Both agents require the Oracle Instant Client libraries.
- SQL Plus: The EDB Postgres AI agent uses `sqlplus` to extract DDL from the Oracle source. [Install Oracle's SQL*Plus utility](https://www.oracle.com/database/technologies/instant-client/downloads.html) (the SQL Plus package from Oracle Instant Client downloads) on the auxiliary machine.

## Environment configuration

The migration agents must be able to locate the Oracle libraries and binaries at runtime. Set the following environment variables on your auxiliary machine, replacing `</path/to/instantclient>` with the actual Instant Client directory (e.g., `/opt/oracle/instantclient_19_19`):

```shell
export ORACLE_HOME=</path/to/instantclient>
export PATH=$ORACLE_HOME:$PATH
export LD_LIBRARY_PATH=$ORACLE_HOME:$LD_LIBRARY_PATH
```

!!!note
For Oracle Instant Client, the `sqlplus` binary and the shared libraries (`.so` files) are both located directly in the Instant Client root directory — not in a `bin/` or `lib/` subdirectory. Set both `PATH` and `LD_LIBRARY_PATH` to the Instant Client root.
!!!

Verify the setup before proceeding:

```shell
which sqlplus
sqlplus -V
```

These environment variables must be set in the same shell session where you run the agents. For persistent configuration, add them to your shell profile (e.g., `~/.bashrc` or `~/.bash_profile`).

## Connectivity

Ensure network connectivity (ports, firewall, SSH) from your auxiliary machine is open to:

- Hybrid Manager host
- Oracle source host

Verify that the auxiliary machine can resolve the hostname and reach the Oracle host:

```shell
ping -c 4 <oracle-host-name-or-ip>
```

Ensure the Oracle Listener is accepting connections on port 1521:

```shell
# Example: Testing connectivity to the Oracle Listener
nc -zv <oracle-host-name-or-ip> 1521
```
