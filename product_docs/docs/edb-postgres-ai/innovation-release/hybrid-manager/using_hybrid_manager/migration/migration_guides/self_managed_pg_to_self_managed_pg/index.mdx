These are all the steps you need to perform a migration from a self-managed Postgres database to a different self-managed Postgres database using the Data Migration Service (DMS) of Hybrid Manager (HM).

## Set up your auxiliary machine

Configure the machine that hosts both migration agents. This machine acts as the bridge between your Postgres source and HM.

1.  [Configure your auxiliary machine](auxiliary_machine).

## Prepare your source database

Create the required users and configure your source database for both agents. This process covers creating the HM machine user, the agent OS user, the `beacon-agent` database user and schema extraction permissions, WAL configuration for change data capture (CDC), the migration role and its grants, SSL, and configuration validation.

1.  [Prepare the source Postgres database](preparing_source_pg).

## Configure the EDB Postgres AI agent

Connect the agent to your source database and to HM to enable schema ingestion and migration assessment.

1.  [Configure the EDB Postgres AI agent to assess your database](agent_postgres).

## Prepare your destination database

Create a server to host the destination database. Then, alter the destination database's configuration to enable replication features.

1.  [Create a destination database in your server](creating_destination_db).

1.  [Prepare the destination Postgres database](preparing_destination_selfpg_db).

## Migrate your schema

Handle incompatibilities and transfer your schemas.

1.  [Handling constraints for schema migration](handling_schema). 

1.  [Migrating schemas](migrating_schema). 

## Migrate your data

Execute the data transfer and complete the final cleanup steps.

1.  [Create project credentials for the data migration](creating_project_credentials). 

1.  [Configure a DMS agent in reader mode for a data migration](configuring_reader). 

1.  [Verify your source database appears in the HM console](viewing_source).

1.  [Configure a DMS agent in writer mode for a data migration to a self-managed Postgres database](configuring_writer).

1.  [Verify your destination appears in the HM console](viewing_destination_console).

1.  [Create and perform the data migration](creating_migration).

1.  [Monitor your data migration](monitoring_migration).

1.  [Finalize the migration](finalizing_migration) (for streaming migrations only).

1.  [Apply excluded constraints](applying_constraints).
