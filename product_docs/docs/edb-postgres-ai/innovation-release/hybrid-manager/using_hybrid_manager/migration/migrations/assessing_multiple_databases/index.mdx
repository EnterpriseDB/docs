When you have a large database estate — dozens of Oracle or Postgres databases — manually constructing a DSN string and adding a `beacon_agent.yaml` entry for each one is time-consuming and error-prone. The EDB Postgres AI agent's `configure-onprem-dbs` command addresses this by reading your existing database inventory from a file (or discovering databases directly from AWS) and automatically populating `beacon_agent.yaml` with the correct entries.

The result is a configuration file ready to run: once you start the agent, all enrolled databases are registered with HM and assessed for migration. This is the recommended approach when you want an estate-wide migration readiness report before committing to individual migration projects.

!!!note
`configure-onprem-dbs` prepares the configuration file only. It doesn't start the agent or connect to HM — you run the agent separately after reviewing and finalizing the generated configuration.
!!!

## Supported input sources

The `configure-onprem-dbs` command can populate your configuration from three sources.

- CSV file

- Oracle `tnsnames.ora` file

- AWS RDS discovery

## Full command reference

For all available flags — including RDS-specific options such as `--region`, `--rds-engine`, and `--tag-filter` — see [`configure-onprem-dbs`](/tools/agent/agent_ref#configure-onprem-dbs) in the EDB Postgres AI agent CLI reference.
