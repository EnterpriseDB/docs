Hybrid Manager (HM) accepts three types of SQL input files for application assessment. Each captures your application's SQL differently, and the best choice depends on your application's architecture and the Oracle features it uses.

| File type | Best for | Extension |
|---|---|---|
| [Oracle trace files](oracle_trace_files) | Applications where you can enable Oracle SQL tracing against a test or non-production database. Provides the most complete picture of runtime SQL, including bind variable values. | `.trc` |
| [EDB Query Extractor output](edb_query_extractor_files) | Applications where Oracle tracing isn't available but Unified Auditing is enabled. Captures SQL persistently across restarts. | `.out` |
| [Plain SQL files](plain_sql_files) | Existing SQL scripts — initialization scripts, maintenance jobs, or SQL extracted from source code — that don't require any Oracle-side configuration. | `.sql` |
