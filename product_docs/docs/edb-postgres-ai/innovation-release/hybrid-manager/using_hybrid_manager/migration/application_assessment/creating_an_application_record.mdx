An application record is how Hybrid Manager (HM) represents a software application. It maps the application to the source databases and schemas it depends on, and holds the SQL files you upload to assess their compatibility with EDB Postgres Advanced Server (EPAS). Creating an application record is the first step in the application assessment workflow.

## Prerequisites

Before creating an application record:

-   You have access to the HM project where you want to create the application.
-   Your user role includes the `migration_applications` permission for that HM project.
-   You have at least one Oracle migration database registered with the EDB Postgres AI agent to associate with the application. To register an Oracle database, see [Configuring the EDB Postgres AI agent to register your Oracle database](/edb-postgres-ai/latest/hybrid-manager/using_hybrid_manager/migration/migration_guides/oracle_to_hm_managed/configuring_beacon_agent/).
-   You have your SQL artifacts ready to upload. See [supported import formats](/edb-postgres-ai/latest/hybrid-manager/using_hybrid_manager/migration/application_assessment/#supported_import_formats/) for the accepted file types.

## Creating an application record

The HM console guides you through defining the application's general metadata and associating it with your migration databases.

1.  In HM, open your project.

1.  In the left navigation, go to **Migrate** > **Applications**.

1.  Select **+New Application**.

1.  Enter the application's metadata:

    | Field | Required | Description |
    |---|---|---|
    | **Name** | Yes | A descriptive name for the application. This name appears in the **Applications** table and on the details page. |
    | **Description** | No | A short explanation of what the application does or its business purpose. |
    | **Target database version** | Recommended | The EPAS version your SQL will be assessed against, ensuring the correct compatibility rules are applied. Becomes the default for every file you upload to this application, so you don't have to select it again at upload time. |
    | **Programming language** | Recommended | The primary language or framework the application uses to interact with the database, for example, Java, Go (sqlx), or Python. |

1.  Continue to the **Next: Databases (Optional)** step.

1.  Select the checkbox next to each database you want to include. You can select multiple databases across pages.

1.  Select **Create application**.

HM creates the application record and takes you to your application details page.

## Uploading SQL files

After creating the application record, upload your SQL artifacts for assessment.

1.  On your application details page, go to the **Overview** tab.
1.  In the **SQL File Assessments** section, select **Upload New File(s)**.
1.  Drag and drop your files into the upload area, or select them using the file picker. You can select up to 20 files in a single upload operation. Individual file size limits apply. If a file exceeds the limit, it's rejected at upload with an error message.
1.  Review the list of files. Any pre-validation errors appear here.
1.  Select **Next: Databases** to continue.
1.  For each uploaded file, configure the following settings:

    | Field | Required | Description |
    |---|---|---|
    | **Source database** | No | The migration database the file's SQL was extracted from. Linking a file to a source database lets HM map queries to specific schemas for more precise assessment. |
    | **Target database version** | Yes | The EPAS version to assess against, for example, EPAS 18. If you set a target version at the application level, it prefills here. |
    | **Oracle User UIDs** | No | One or more Oracle user IDs whose objects appear in the SQL file. Used to scope the assessment to the relevant schema owners. |

1.  Select **Confirm** to upload the files and queue them for assessment.

!!!note Security
Uploaded files are write-only. You can't redownload the raw source files after upload. This prevents the accidental exposure of sensitive data or secrets embedded in SQL scripts.
!!!
