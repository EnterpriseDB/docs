## What you'll build

This quickstart points you to the fastest path for two common goals: building an AI flow in Langflow backed by a knowledge base, and deploying a model endpoint your applications can call directly.

## Prerequisites

-   GPU node groups provisioned for model serving: [GPU recommendations](gpu-recommended)
-   A model cluster deployed from the Model Library: [Model management](models)

## Option A — Build an AI flow with Langflow

Use Langflow to build a RAG flow that retrieves from a knowledge base and generates answers with an LLM — all running inside your HM cluster.

-   [Langflow quickstart](langflow/quickstart) — end-to-end walkthrough: knowledge base setup, component wiring, and running the flow in the Playground

## Option B — Create a knowledge base pipeline

Use Pipeline Designer to ingest and embed documents into a vector knowledge base through a no-code UI.

-   [Pipeline Designer getting started](pipeline-designer/getting-started)
-   [Creating pipelines](pipeline-designer/creating-pipelines)

## Deploy a model endpoint

Deploy a model using KServe so your applications can call it directly over REST.

-   [Deploy with HM UI](deploy-with-hm-ui)
-   [Access KServe endpoints](/edb-postgres-ai/current/ai-factory/model/access-kserve-endpoints)
-   [Quickstart (Python)](/edb-postgres-ai/current/ai-factory/model/quickstart-python-using-inference-endpoint)
