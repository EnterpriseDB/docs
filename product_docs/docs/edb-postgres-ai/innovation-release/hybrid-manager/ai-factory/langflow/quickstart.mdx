This quickstart walks you through building a simple retrieval-augmented generation (RAG) flow in Langflow on Hybrid Manager. By the end you'll have a working flow that accepts a user question, retrieves relevant content from a knowledge base, and returns a grounded answer from an LLM — all running within your HM cluster.

## Prerequisites

Before starting, make sure you have:

- AI Factory enabled on your HM project. See [How to enable AI Factory](../how-to-enable-ai-factory).
- A GPU node deployed for model serving. See [GPU recommendations](../gpu-recommended).
- At least one model cluster deployed from the Model Library — you need an LLM for generation and an embedding model for the knowledge base. See [Creating a model cluster](./components/embeddings#creating-a-model-cluster).
- A Postgres cluster in your HM project with a table containing the content you want to query.
- An access key (machine user) with the following roles:
  - **AI Model Manager** (organization level)
  - **Project Viewer** (project level)

---

## Step 1 — Access Langflow

1. Log into the HM console and select your project.
1. Select **Launchpad** in the navigation and open **Langflow**.
1. On the Langflow home screen, select **New Flow** → **Blank Flow**.

---

## Step 2 — Set up Global Variables

EDB components read credentials from Langflow Global Variables. Set these up once and reuse them across all components in the flow.

1. In Langflow, go to **Profile** → **Settings** → **Global Variables**.
1. Create the following variables:

   | Variable name | Value |
   |---|---|
   | `hm_access_key` | Your HM machine user access key. |
   | `db_user` | Postgres user for your HM cluster. |
   | `db_password` | Password for the Postgres user. |

See [Configure Global Variables](https://docs.langflow.org/configuration-global-variables#create-a-global-variable) for instructions.

---

## Step 3 — Create a knowledge base

Before building the flow, create a knowledge base in your HM cluster so Langflow has content to retrieve from.

Run the following in your HM Postgres cluster, replacing the table name and column with your own data:

```sql
CREATE EXTENSION IF NOT EXISTS aidb CASCADE;

SELECT aidb.create_table_knowledge_base(
    name              => 'my_knowledge_base',
    model_name        => 'bert',
    source_table      => 'public.my_content_table',
    source_data_column => 'content',
    source_data_format => 'Text'
);

SELECT aidb.bulk_embedding('my_knowledge_base');
```

Verify the knowledge base is ready:

```sql
SELECT * FROM aidb.kbstat;
```

---

## Step 4 — Add a Chat Input

1. In the Langflow canvas, select **Inputs** in the component panel.
1. Drag **Chat Input** onto the canvas. This captures the user's question.

---

## Step 5 — Add an EDB Knowledge Base component

1. Select **EDB** in the component panel and drag **EDB Knowledge Base** onto the canvas.
1. Configure the component:
   - **HM Machine User Key** → select the `hm_access_key` Global Variable.
   - **Hybrid Manager Project** → select your project.
   - **Hybrid Manager Database** → select the cluster containing your knowledge base.
   - **Database Connection Type** → select the appropriate connection type.
   - **User for the database** → select the `db_user` Global Variable.
   - **Password for database** → select the `db_password` Global Variable.
   - **Database Name** → select the database where the knowledge base was created.
   - **Query Value** → connect this to the **Chat Input** output.
1. Connect **Chat Input** → **EDB Knowledge Base**.

---

## Step 6 — Add an EDB Model Server component

1. Select **EDB** in the component panel and drag **EDB Model Server** onto the canvas.
1. Configure the component:
   - **HM Machine User Key** → select the `hm_access_key` Global Variable.
   - **Hybrid Manager Model Server Cluster Instance** → select your LLM model cluster.
   - **Input** → connect this to the **EDB Knowledge Base** output (retrieved context) and the **Chat Input** output (the original question).
1. Connect both **EDB Knowledge Base** and **Chat Input** → **EDB Model Server**.

---

## Step 7 — Add a Chat Output

1. Select **Outputs** in the component panel and drag **Chat Output** onto the canvas.
1. Connect **EDB Model Server** → **Chat Output**.

---

## Step 8 — Run and test the flow

1. Select **Playground** in the top toolbar to open the interactive test panel.
1. Type a question that relates to your source data and press **Enter**.
1. Verify the response is grounded in your content — you should see the EDB Knowledge Base retrieving relevant rows and the EDB Model Server generating a contextualised answer.
1. Check that all components show green status indicators after running.

If any component shows an error, select it and review the configuration. The most common issues are an incorrect access key, a mismatched database connection, or a model cluster that isn't active yet.

## Next steps

- **Add more components** — extend the flow with an **EDB Database** component to query structured data alongside the knowledge base, or an **EDB MCP Server** component to expose the flow as an MCP server. See [EDB components](./components/) for the full component reference.
- **Promote to production** — once the flow is tested, hand it off to a Production Operator. See [Promote a Langflow application to production](./promoting-to-production).
- **Monitor the flow's dependencies** — set up Prometheus alert rules and Grafana dashboards for the model cluster and Langflow pods. See [Observability](../observability).
