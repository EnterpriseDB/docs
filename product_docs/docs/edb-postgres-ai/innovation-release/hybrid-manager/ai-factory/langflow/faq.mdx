## Building flows

### Where do I start with Langflow in Hybrid Manager?

Access Langflow via **Launchpad** in the HM console. From the Langflow home screen you can create a new blank flow, import an existing flow from a `.json` file, or browse templates.

For a step-by-step walkthrough of building your first RAG flow, see [Langflow quickstart](./quickstart).

### What EDB components are available in Langflow?

Langflow in HM includes the following EDB-specific components, available in the **EDB** folder in the left-side component panel:

| Component           | Purpose                                                                                                     |
| ------------------- | ----------------------------------------------------------------------------------------------------------- |
| EDB Knowledge Base  | Semantic similarity search over an AIDB knowledge base in a Postgres cluster.                               |
| EDB Database        | Standard data connector for querying any table in an HM-managed Postgres cluster.                          |
| EDB Vector Index    | Creates and manages vector indexes on Postgres tables with pgvector.                                       |
| EDB Embeddings      | Generates vector embeddings using a model cluster deployed in your HM project.                             |
| EDB Model Server    | Sends prompts to a deployed LLM and returns generated responses.                                           |
| EDB Embedded Models | Runs CPU-based text and image inference using AIDB embedded models (BERT, CLIP, T5, Llama, Qwen2, SmolLM2). |
| EDB Airman MCP      | Provides the Airman MCP server's tools to a flow, for use by agents. Replaces the former EDB MCP Server.   |
| EDB Platform        | Provides MCP tools on behalf of the current user in the EDB platform. Authenticates automatically as the current user — no HM Machine User Key input required. |

### How do I connect EDB components to my HM project?

Most EDB components authenticate using an HM Machine User Key. Store this as a Langflow Global Variable and reference it in each component — you don't enter credentials directly on individual components. The **EDB Platform** component is an exception: it authenticates automatically as the current user and does not require an HM Machine User Key.

1.  Create a machine user in your HM project and generate an access key.
1.  In Langflow, go to **Profile** → **Settings** → **Global Variables** and save the key.
1.  In each EDB component, set **HM Machine User Key** to the Global Variable.

If you name the Global Variable `HM_API_KEY` then EDB components will select it automatically.

See [Set up Global Variables](https://docs.langflow.org/configuration-global-variables#create-a-global-variable) for instructions.

### What roles does the machine user need?

The required roles depend on which EDB components the flow uses:

| Component                        | Required role                         |
| -------------------------------- | ------------------------------------- |
| EDB Embeddings, EDB Model Server | AI Model Manager (organization level) |
| EDB Knowledge Base, EDB Database | Project Viewer (project level)        |

For the EDB Airman MCP, EDB Platform, EDB Embedded Models, and EDB Vector Index components, see the individual component pages for their role requirements.

Assign roles in the HM console: **Profile → User management → Assign Organization Roles** (for org-level) or **Project → Users → Assign Project Roles** (for project-level).

## Knowledge bases and retrieval

### How do I create a knowledge base for use in Langflow?

Knowledge bases are created outside Langflow using the AIDB SQL API directly in your HM Postgres cluster. Once created, the EDB Knowledge Base component in Langflow can query it.

```sql
CREATE EXTENSION IF NOT EXISTS aidb CASCADE;

SELECT aidb.create_table_knowledge_base(
    name               => 'my_knowledge_base',
    model_name         => 'bert',
    source_table       => 'public.my_content_table',
    source_data_column => 'content',
    source_data_format => 'Text'
);

SELECT aidb.bulk_embedding('my_knowledge_base');
```

See [Knowledge Base Creation](/edb-postgres-ai/1.3/hybrid-manager/ai-factory/gen-ai/create-knowledge-base/) for full options.

### How do I improve retrieval quality in a RAG flow?

Poor retrieval results are usually caused by one of the following:

-   **Wrong embedding model** — the model used to create the knowledge base embeddings must match the model used at query time. Check the `model_name` used in `aidb.create_table_knowledge_base`.
-   **Chunk size** — if source documents were ingested as large chunks, retrieval scores may be noisy. Re-index with smaller chunks using `aidb.chunk_text_config`.
-   **Top-k too low** — the EDB Knowledge Base component returns the top-k most similar results. Increase this if relevant content is being missed.
-   **Index not up to date** — if source data has changed since the last `aidb.bulk_embedding` run, retrieved content may be stale. Re-run the bulk embedding or set up incremental syncing.

## Models

### How do I use a model in a Langflow flow?

Deploy a model cluster from **AI Factory → Model Library** in the HM console first, then reference it in the **EDB Model Server** or **EDB Embeddings** component using the **Hybrid Manager Model Server Cluster Instance** dropdown.

See [Using models in Langflow](./using-models-in-langflow) for the full walkthrough.

### What's the difference between EDB Model Server and EDB Embeddings?

**EDB Model Server** sends a prompt to an LLM and returns a text response — use it for generation, summarisation, and reasoning steps. **EDB Embeddings** converts text into a vector using an embedding model — use it when you need to compute similarity or store vectors in Postgres. Both require a model cluster to be deployed in your HM project, but they use different model types (LLM vs embedding model).

## Deployment

### How do I run a flow as a production service?

Publish the flow as a bundle and deploy it. A deployed flow runs as a managed workload in an HM project, isolated in its own Kubernetes namespace, with its own HTTP and MCP endpoints, independent scaling, and credentials supplied at deploy time.

See [Flow deployment](./flow-deployment) for the publish and deploy workflow.

## Common errors

### "Can't select a project" — the portal domain defaults to the API endpoint

Select **Controls** at the top of the affected EDB Database or Knowledge Base component and set **HM Portal URL** to:

```
http://istio-internalgateway.istio-system.svc.cluster.local:7080
```

### A component shows an error after running

Select the component to expand its error message. The most common causes are an invalid or expired access key, a model cluster that isn't active, a mismatched database connection, or a knowledge base that hasn't been populated yet.

Check the model cluster status in **AI Factory → Model Server Clusters** and verify the machine user key hasn't expired in **Project Settings → Secrets**.
