The **Pipeline Designer** is a visual interface in Hybrid Manager for building and managing [AIDB](/aidb/latest/) data pipelines. 
It provides a guided, no-code workflow for creating pipelines that process data through AI and ML transformation steps, optionally storing the results in a vector knowledge base for retrieval-augmented generation (RAG).

With Pipeline Designer you can:

-   Create single-step or multi-step data pipelines through a visual wizard

-   Build knowledge bases with embedded vector storage for semantic search

-   Configure processing schedules (live, background, or manual execution)

-   Monitor pipeline status, record counts, and error conditions

-   Test knowledge base retrieval with built-in query tools
-   Use HM-hosted models, HM external inference service proxies, AIDB-native models, or built-in on-database models
-   Run pipelines on primary HM clusters, secondary HM locations, or self-managed PostgreSQL instances

## How Pipeline Designer relates to AIDB

Pipeline Designer is a graphical front end for the [AIDB extension](/aidb/latest/). 
Every pipeline you create through the designer translates directly to AIDB SQL calls (`aidb.create_pipeline`, `PipelineStepOperation`, `PipelineAutoProcessingMode`, and related functions) running in your PostgreSQL database. Pipeline Designer does not introduce its own conceptual vocabulary; it presents AIDB pipelines, knowledge bases, and models through a visual interface. For the underlying concepts, see the [AIDB documentation](/aidb/latest/).

Pipeline Designer manages only the pipelines it creates. 
Pipelines created directly through SQL (outside the Pipeline Designer UI) do not appear in the designer interface. 
See [VPU and permissions](vpu-and-permissions) for details on why this separation exists.

## Prerequisites

Before using Pipeline Designer, ensure the following are in place:

1.  **AIDB extension**: The AIDB extension must be available on your PostgreSQL cluster. Pipeline Designer requires AIDB 5.0.0 or later; multi-step pipeline support requires AIDB 6.0 or later (AIDB 6.1 adds the PdfToImage step type).

1.  **AI Factory enabled**: AI Factory must be enabled in your Hybrid Manager project. See [How to enable AI Factory](../how-to-enable-ai-factory).

1.  **Inference models available**: At least one inference model must be registered in the AIDB model registry for steps that require a model (KnowledgeBase, SummarizeText, PerformOCR). Models can come from HM Model Serving, HM external inference service proxies, AIDB-native registration via SQL, or AIDB's built-in local models. See [Executing pipelines: How models reach pipeline steps](executing-pipelines#how-models-reach-pipeline-steps) for how each path works and which deployment types support it.

1.  **Source data**: You need a PostgreSQL table containing the data you want to process. The table must have a key column (unique identifier) and a data column (the content to process).

1.  **Table permissions for VPU**: The `visual_pipeline_user` role must have access to your source table. See [VPU and permissions](vpu-and-permissions) for the required grants.

## Current limitations

Pipeline Designer provides a subset of the full AIDB feature set. Key limitations include volume-based sources and multi-pipeline knowledge bases not yet being supported in the UI. For the full list, see [Limitations](limitations).

## Supported cluster types

Pipeline Designer supports the following cluster configurations:

-   **HM-managed clusters** (PSR and PGD/AHA): Clusters provisioned and managed by Hybrid Manager on the primary location.

-   **Secondary HM locations**: HM-managed clusters running on additional HM locations in a multi-location deployment.

-   **Self-managed PostgreSQL instances**: Single-instance PostgreSQL servers registered with HM through the EDB Postgres AI agent (`beacon-agent`), with AIDB installed. External PGD clusters and Cloud Native PostgreSQL (CNP) clusters are not supported.

The minimum supported PostgreSQL major version is 16. For details on how execution works across these cluster types, see [Executing pipelines](executing-pipelines).
