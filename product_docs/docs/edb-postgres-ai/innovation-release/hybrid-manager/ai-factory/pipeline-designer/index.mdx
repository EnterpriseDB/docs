The **Pipeline Designer** is a visual interface in Hybrid Manager for building and managing [AIDB](/edb-postgres-ai/current/ai-factory/pipeline/) data pipelines. It provides a guided, no-code workflow for creating pipelines that process data through AI and ML transformation steps, optionally storing the results in a vector knowledge base for retrieval-augmented generation (RAG).

With Pipeline Designer you can:

-   Create single-step or multi-step data pipelines through a visual wizard
-   Build knowledge bases with embedded vector storage for semantic search
-   Configure processing schedules (live, background, or manual execution)
-   Monitor pipeline status, record counts, and error conditions
-   Test knowledge base retrieval with built-in query tools
-   Use HM-hosted models, external inference services (OpenAI, AWS Bedrock), or on-database models
-   Run pipelines on primary HM clusters, secondary HM locations, or self-managed PostgreSQL instances

## How Pipeline Designer relates to AIDB

Pipeline Designer is a graphical front end for the [AIDB extension](/edb-postgres-ai/current/ai-factory/pipeline/). Every pipeline you create through the designer translates to SQL calls against the AIDB extension running in your PostgreSQL database. The following table maps Pipeline Designer concepts to their AIDB equivalents:

| Pipeline Designer term | AIDB equivalent | Description |
| ---------------------- | --------------- | ----------- |
| Pipeline | Data pipeline (`aidb.create_pipeline`) | A sequence of processing steps applied to source data |
| Pipeline step | Pipeline step operation (`PipelineStepOperation`) | An individual AI/ML transformation within a pipeline |
| Knowledge base | Knowledge base (`aidb.knowledge_bases`) | Vector storage for embeddings produced by a KnowledgeBase step |
| Processing type | Auto-processing mode (`PipelineAutoProcessingMode`) | Controls when and how the pipeline executes |
| Model | AIDB model (`aidb.models`) | An inference model used by steps that require AI capabilities |

Pipeline Designer manages only the pipelines it creates. Pipelines created directly through SQL (outside the Portal) do not appear in the designer interface. See [VPU and permissions](vpu-and-permissions) for details on why this separation exists.

## Prerequisites

Before using Pipeline Designer, ensure the following are in place:

1.  **AIDB extension**: The AIDB extension must be available on your PostgreSQL cluster. Pipeline Designer requires AIDB 5.0.0 or later; multi-step pipeline support requires AIDB 6.1 or later.
2.  **AI Factory enabled**: AI Factory must be enabled in your Hybrid Manager project. See [How to enable AI Factory](../how-to-enable-ai-factory).
3.  **Inference models deployed**: At least one inference model must be available for steps that require a model (KnowledgeBase, Summarize, OCR). Deploy models through [Model Serving](../models) or register [external inference services](external-inference-services).
4.  **Source data**: You need a PostgreSQL table containing the data you want to process. The table must have a key column (unique identifier) and a data column (the content to process).
5.  **Table permissions for VPU**: The `visual_pipeline_user` role must have access to your source table. See [VPU and permissions](vpu-and-permissions) for the required grants.

## Supported cluster types

Pipeline Designer supports the following cluster configurations:

-   **HM-managed appliance clusters** (PSR and PGD/AHA): Pipelines run on the primary writable node. For PGD clusters, you select a data group and pipelines execute on the write leader.
-   **HM-managed secondary locations**: If your HM deployment spans multiple data centers, you can create pipelines targeting clusters in secondary locations.
-   **Self-managed PostgreSQL instances**: PostgreSQL instances registered with HM through a beacon-agent can host pipelines, provided AIDB is installed and a beacon-agent is connected.

The minimum supported PostgreSQL major version is 16.
