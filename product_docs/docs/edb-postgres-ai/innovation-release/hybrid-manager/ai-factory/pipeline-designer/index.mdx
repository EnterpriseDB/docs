The **Pipeline Designer** is a visual interface in Hybrid Manager for building and managing [AIDB](/edb-postgres-ai/current/ai-factory/pipeline/) data pipelines. It provides a guided, no-code workflow for creating pipelines that process data through AI and ML transformation steps, optionally storing the results in a vector knowledge base for retrieval-augmented generation (RAG).

With Pipeline Designer you can:

-   Create single-step or multi-step data pipelines through a visual wizard
-   Build knowledge bases with embedded vector storage for semantic search
-   Configure processing schedules (live, background, or manual execution)
-   Monitor pipeline status, record counts, and error conditions
-   Test knowledge base retrieval with built-in query tools
-   Use HM-hosted models, HM external inference service proxies, AIDB-native models, or built-in on-database models
-   Run pipelines on primary HM clusters, secondary HM locations, or self-managed PostgreSQL instances

## How Pipeline Designer relates to AIDB

Pipeline Designer is a graphical front end for the [AIDB extension](/edb-postgres-ai/current/ai-factory/pipeline/). Every pipeline you create through the designer translates directly to AIDB SQL calls (`aidb.create_pipeline`, `PipelineStepOperation`, `PipelineAutoProcessingMode`, and related functions) running in your PostgreSQL database. Pipeline Designer does not introduce its own conceptual vocabulary; it presents AIDB pipelines, knowledge bases, and models through a visual interface. For the underlying concepts, see the [AIDB documentation](/edb-postgres-ai/current/ai-factory/pipeline/).

Pipeline Designer manages only the pipelines it creates. Pipelines created directly through SQL (outside the Portal) do not appear in the designer interface. See [VPU and permissions](vpu-and-permissions) for details on why this separation exists.

## Prerequisites

Before using Pipeline Designer, ensure the following are in place:

1.  **AIDB extension**: The AIDB extension must be available on your PostgreSQL cluster. Pipeline Designer requires AIDB 5.0.0 or later; multi-step pipeline support requires AIDB 6.0 or later (AIDB 6.1 adds the PdfToImage step type).
2.  **AI Factory enabled**: AI Factory must be enabled in your Hybrid Manager project. See [How to enable AI Factory](../how-to-enable-ai-factory).
3.  **Inference models available**: At least one inference model must be registered in the AIDB model registry for steps that require a model (KnowledgeBase, SummarizeText, PerformOCR). Models can come from HM Model Serving, HM external inference service proxies, AIDB-native registration via SQL, or AIDB's built-in local models. See [Executing pipelines: How models reach pipeline steps](executing-pipelines#how-models-reach-pipeline-steps) for how each path works and which deployment types support it.
4.  **Source data**: You need a PostgreSQL table containing the data you want to process. The table must have a key column (unique identifier) and a data column (the content to process).
5.  **Table permissions for VPU**: The `visual_pipeline_user` role must have access to your source table. See [VPU and permissions](vpu-and-permissions) for the required grants.

## Current limitations

Pipeline Designer provides a subset of the full AIDB feature set. The following capabilities are available through AIDB SQL but are not supported in Pipeline Designer:

-   **Volume-based sources**: Pipeline Designer supports only table-based sources. AIDB's volume source type (for processing files from object storage) is not available through the Pipeline Designer UI.
-   **Multi-pipeline knowledge bases**: AIDB 7 supports knowledge bases fed by multiple pipelines. Pipeline Designer does not yet expose this capability; it is planned for the June 2026 LTS release. Each knowledge base created through Pipeline Designer is currently associated with exactly one pipeline.

## Supported cluster types

Pipeline Designer supports the following cluster configurations:

-   **HM-managed clusters** (PSR and PGD/AHA): Clusters provisioned and managed by Hybrid Manager on the primary location.
-   **Secondary HM locations**: HM-managed clusters running on additional HM locations in a multi-location deployment.
-   **Self-managed PostgreSQL instances**: External PostgreSQL instances registered with HM through the EDB Postgres AI agent (`beacon-agent`), with AIDB installed.

The minimum supported PostgreSQL major version is 16. For details on how execution works across these cluster types, see [Executing pipelines](executing-pipelines).
