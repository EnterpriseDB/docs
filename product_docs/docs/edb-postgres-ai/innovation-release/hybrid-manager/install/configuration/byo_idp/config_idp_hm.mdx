The IdP configuration workflow in the Hybrid Manager (HM) console automates the integration process, removing the requirement for manual Helm chart modifications and manual workload restarts to ensure consistent deployment and reduced operational downtime.

## Configuring the SAML app integration in the Okta admin console

To create the connection point with the HM console, configure the the SAML integration in the Okta admin console:

1. In the section **Applications**, select **Create a new app integration**.

1. Select ** SAML 2.0** > **Next**.

1. Under **General Settings**, enter the name of the SAML integration and select **Next**.

1. In the **Configure SAML** tab, enter the **Assertion Consumer Service URL** in the field **Single sign-on URL**.

1. In the **Configure SAML** tab, enter the audience URI in the **Audience URI (SP Entity ID)** field.

1. Next to **Name ID format**, select **EmailAddress**.

1. Under **Attribute Statements**, set the values for **name** and **email** (Optional) .

   Where:
   
   - **name** is `user.firstName`
   - **email** is `user.email`

1. Select **Next** > **Finish**.

1. In the new window that pops up, select **More details** to copy the **Sign on URL**.

1. In the new window that pops up, download the signing certificate by selecting **Download**.



## Configuring the IdP provider through the SAML connector in the HM console

You can configure your IdP provider with SAML in the **Settings**:

1. Select the **Identity Providers** tab.

1. In the right-hand of your screen, select **Add Identity Provider**.

1. Select the **SAML** tab and in the ** Name** field, enter the name of your new identity provider.

1. In the field **Description**, enter the description of your new Identity Provider.

1. Under **Select a Protocol**, select the **SAML** tab.

1. In the field **Single Sign-On URL**, paste the URL created during the configuration of the [SAML connector in the Okta admin console](#configuring-the-saml-app-integration-in-the-okta-admin-console. 

1. Under **Signature Certificate File**, upload the signing certificate created during the configuration of the [SAML connector in the Okta admin console](#configuring-the-saml-app-integration-in-the-okta-admin-console).

1. Under **NameID Format**, select **EMAIL ADDRESS**.

1. Enter the values for **Username Attribute** and **Email Attribute**.

   Where:
   
   - **Username Attribute** is `name`
   - **Email Attribute** is `email`

1. Select **Save**.

## Configuring the IdP provider through the LDAP connector

You can configure your IdP provider with LDAP in the **Settings** tab:

1. Select the **Identity Providers** tab.

1. In the right-hand of your screen, select **Add Identity Provider**.

1. Select the **LDAP** tab and enter the required parameters.

   Where:

   - **Protocol** is LDAP.
   - **Quick Sign-in URL** 
   [Where do customers get this from?]: #
 
   - **LDAP Server** is the LDAP server hostname. For example: `identity.mycompany.com`.
   - **LDAP scheme** uses LDAP or LDAPS for secure data transmission. For example: `ldaps`.
   - **Base DN** is `(&(objectClass=person)(uid=<username>))`.
   - **Bind DN** is the domain name (DN) and password for an application service account.
   - **User Name** is the username attribute used for comparing user entries. It is translated and combined with the other filter as `(<attr>=<username>)`.
   - **ID Attribute** is the LDAP entry attribute to map to the HM user ID.
   - **Email Attribute** is the LDAP entry attribute to map to HCP user email.
   - **Name Attribute** is the LDAP entry attribute to map to HCP user name.
 
1. Select **Save**.

## Managing the IdP provider in the HM console

Assign users or remove the IdP provider from the HM console to make secure access to the authorized users only.


### Assigning users to the IdP

To assign users to the IdP you configured and give them access to HM:

1. In the Okta admin console, select **Application** and select the **Assignments** tab.

1. Select **Assign to People** and look for the user you want to assign the IdP to.

1. Next to the user's name, select **Assign** > **Done**.

1. In the HM console, select the **Identity Providers** tab.

1. Select the IdP you created and copy the **Quick Sign-in URL**.


### Removing the IdP provider from the HM console

Removing the IdP Provider from your HM console will revoke any access to the HM console for users that were assigned to the IdP provider.

1. In the HM console, select the **Identity Providers** tab.

1. Select the IdP you want to remove and select **Delete** (trashcan icon).

1. Follow in the instructions in the pop-up window and select **Delete Identity Provider**.


!!! important
Removing the IdP provider in the HM console will delete all users assigned to the IdP provider.
