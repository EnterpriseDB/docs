The IdP configuration workflow in the Hybrid Manager (HM) console automates the integration process. When you use this workflow, you don't need to manually modify the Helm chart or restart the workload. This method makes deployment consistent and reduces downtime.


## Configuring a SAML IdP

You need to register a new application or client in your IdP (for example, Okta, Microsoft Entra ID, or Google Workspace) before configuring your IdP in HM:

1. Create a new SAML 2.0 app integration in your IdP.

1. Enter a name for the SAML integration.

1. Enter the assertion consumer service (ACS) URL in the **Single sign-on URL** field.

1. Enter the audience URI in the **Audience URI (SP Entity ID)** field.

1. Set the **Name ID format** to **EmailAddress**.

1. (Optional) Enter the **name** and **email** values for the corresponding user profile fields in your IdP.

1. Save the integration.

1. Copy the sign-on URL provided by your IdP.

1. Download the signing certificate provided by your IdP.


### Configuring a SAML IdP in HM

To configure your SAML IdP in HM:

1. In the HM console, go to **Settings** then select **Identity Providers**.

1. Select **Add Identity Provider**.

1. In the **SAML** tab, in the **Name** field, enter your new identity provider.

1. In **Description**, enter the description of your new identity provider.

1. Under **Select a Protocol**, select the **SAML** tab.

1. In **Single Sign-On URL**, paste the sign-on URL obtained when you configured the [SAML IdP](#configuring-a-saml-idp).

1. Under **Signature Certificate File**, upload the signing certificate obtained when you configured the [SAML IdP](#configuring-a-saml-idp).

1. Under **NameID Format**, select **Email Address**.

1. Enter the values for **Username Attribute** and **Email Attribute**.

   Where:

   - **Username Attribute** is `name`.
   - **Email Attribute** is `email`.

1. Select **Save**.

## Configuring an LDAP IdP

### Configuring an LDAP IdP in HM

To configure your LDAP IdP in HM:

1. In the HM console, select **Settings** then select **Identity Providers**.

1. Select **Add Identity Provider**.

1. Select the **LDAP** tab and enter the required parameters.

   Where:

   - **Protocol** is LDAP.
   - **Quick Sign-in URL**.
   - **LDAP Server** is the LDAP server hostname. For example: `identity.mycompany.com`.
   - **LDAP scheme** uses LDAP or LDAPS for secure data transmission. For example: `ldaps`.
   - **Base DN** is `dc=example,dc=com`.
   - **Bind DN** is the domain name (DN) and password for an application service account.
   - **User Name** is the username attribute used for comparing user entries.
   - **ID Attribute** is the LDAP entry attribute to map to the HM user ID.
   - **Email Attribute** is the LDAP entry attribute to map to HCP user email.
   - **Name Attribute** is the LDAP entry attribute to map to HCP user name.

1. Select **Save**.


## Configuring an OIDC IdP

You need to register a new application or client in your OIDC provider (for example, Okta, Microsoft Entra ID, or Google Workspace) before configuring your IdP in HM:

1. Set the **Redirect URI** to the value displayed in the HM console when adding an OIDC identity provider.

1. Note the **Issuer URL**, **Client ID**, and **Client Secret** provided by your OIDC provider.

1. Ensure the application is configured to include `name` and `email` in the ID token or UserInfo endpoint.

1. Assign the users you want to have access to HM to the registered application.


### Configuring an OIDC IdP in HM

To configure your OIDC IdP in HM:

1. In the HM console, select **Settings** > **Identity Providers**.

1. Select **Add Identity Provider**.

1. Select the **OIDC** tab.

1. In the **Name** field, enter a name for your identity provider.

1. In the **Description** field, enter a description for your identity provider.

1. Enter the following values from your OIDC provider:

   Where:

   - **Issuer URL** is the issuer URL of your OIDC provider (for example, `https://accounts.google.com`).
   - **Client ID** is the client ID assigned to your registered application.
   - **Client Secret** is the client secret assigned to your registered application.

1. Select **Save**.


## Testing the IdP integration

After configuring an identity provider, you can verify if the integration is working correctly using the built-in test feature. The test performs a sign-in attempt through the configured IdP and validates that user attributes such as `email` and `name` are returned correctly.

1. In the HM console, select **Settings** then select **Identity Providers**.

1. Select the IdP you want to test.

1. Select **Test Connection**.

1. Follow the sign-in prompts from your IdP. After authentication is complete, the HM console displays the test result indicating whether the integration is configured correctly.


## Managing the IdP

In HM, manage IdP providers by editing or deleting them, and assign users to restrict access to authorized users only.


### Assigning users to the IdP

To assign users to the IdP you configured and give them access to HM:

1. In your IdP's admin console, assign the users you want to give access to HM.

1. Next to the name of the user you want to assign the IdP to, select **Assign**. Select **Done**.

1. In the HM console, select to **Settings** then select **Identity Providers**.

1. Select the IdP you created and copy the value in the **Quick Sign-in URL**.


### Removing the IdP from HM

!!! warning
Removing the IdP in HM deletes all users assigned to the IdP.
!!!

To remove the IdP from HM:

1. In the HM console, select **Settings** then select **Identity Providers**.

1. Select the IdP you want to remove and select **Delete**.

1. Follow the instructions in the pop-up and select **Delete Identity Provider**.
