For production deployments, we strongly recommend configuring an external IdP like LDAP, SAML, or another [supported Dex connector](https://dexidp.io/docs/connectors/). Doing so ensures robust and secure user management. Here are some examples:

[Is this still relevant with the new changes?]: #

-   [LDAP connector with Okta](ldap)

-   [SAML connector with Okta](saml)

!!! note
You can now configure your [IdP in the HM console](/edb-postgres-ai/latest/hybrid-manager/install/configuration/byo_idp/config_idp_hm/).
!!! 

For testing or demo environments, you can simplify operations by [creating additional native users](/edb-postgres-ai/current/hybrid-manager/using_hybrid_manager/managing_users/native_users/create_native_user/).
