For production deployments, we strongly recommend configuring an external IdP like LDAP, SAML, or another [supported Dex connector](https://dexidp.io/docs/connectors/). Doing so ensures robust and secure user management. Here are some examples:

#[Is this still relevant with the new changes?]:

The IdP configuration workflow in the HM console automates the integration process, removing the requirement for manual Helm chart modifications and manual workload restarts to ensure consistent deployment and reduced operational downtime.

## Configuring the the SAML app integration in the Okta admin console

To create the connection point with the HM console, configure the the SAML integration in the Okta admin console:

1. In the section **Applications**, select **Create a new app integration**

1. Select ** SAML 2.0** > **Next**.

1. Under **General Settings**, enter the name of the SAML integration > **Next**.

1. In the **Configure SAML** tab, enter the **Assertion Consumer Service URL** in the field **Single sign-on URL**.

1. In the **Configure SAML** tab, enter the *Audience URI** in the field **Single sign-on URL**.

1. Next to **Name ID format**, select **EmailAddress**.

1. Under **Attribute Statements**, set the values for **name** and **email** (Optional) .

Where 

- **name** is `user.firstName`
- **email** is `user.email`

1. Select **Next** > **Finish**.

1. In the new window that pops up, select **More details** to copy the **Sign on URL**.

1. In the new window that pops up, download the signing certificate by selecting **Download**.



## Configuring the IdP provider through the SAML connector in the HM console

You can configure your IdP provider with SAML in the **Settings**:

1.  Select the **Identity Providers** tab.

1. In the right-hand of your screen, select **Add Identity Provider**.

1. In the field ** Name**, enter the name of your new Identity Provider.

1. In the field **Description**, enter the description of your new Identity Provider.

1. Under **Select a Protocol**, select the *SAML* tab.

1. In the field **Single Sign-On URL**, paste the URL created during the configuration of the [SAML connector in the Okta admin console](#configuring-the-the-saml-app-integration-in-the-okta-admin-console). 

1. Under **Signature Certificate File**, upload the signing certificate created during the configuration of the [SAML connector in the Okta admin console](#configuring-the-the-saml-app-integration-in-the-okta-admin-console).

1. Under **NameID Format**, select **EMAIL ADDRESS**

1. Enter the values for **Username Attribute** and **Email Attribute**

Where 

- **Username Attribute** is `name`
- **Email Attribute** is `email`

1. Select **Save**.
