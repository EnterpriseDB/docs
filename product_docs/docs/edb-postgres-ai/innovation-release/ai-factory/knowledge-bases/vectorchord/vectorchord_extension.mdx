---
title: VectorChord extension
---

VectorChord extends the standard capabilities of `pgvector` by offering specialized index types that handle high-dimensional data more effectively. In an AI pipeline, VectorChord is typically configured during the **KnowledgeBase** step to ensure that the resulting embeddings are immediately searchable with minimal latency.

## Key benefits

- **Scalability:** Optimized for datasets containing millions of vectors.
  
- **Speed:** Faster query execution times compared to standard IVFFlat or HNSW for specific workloads.

- **Flexibility:** Supports multiple data types and specialized centroid configurations.

## Integration and configuration

Integration occurs by passing specific configuration objects to the `vector_index` parameter within the `aidb.knowledge_base_config()` helper function.

It supports two primary index types:

### HNSW (Hierarchical Navigable Small Worlds)

The VectorChord implementation of HNSW offers granular control over graph connectivity and search depth. Use `aidb.vector_index_chord_hnsw_config()` function to configure parameters such as `m` (the number of connections per node) and `ef_construction` (the depth of the search graph during index construction).

**Example**:

```sql
SELECT aidb.vector_index_chord_hnsw_config(
    vector_data_type => 'float32',
    m => 16,
    ef_construction => 64,
    max_connections => 32
);
```

### Vchordq

A specialized index type designed for high-efficiency quantization and clustering. Use `aidb.vector_index_chord_vchordq_config()` function to configure parameters such as `lists` (number of clusters) and `spherical_centroids` (boolean for centroid configuration).

**Example**:

```sql
SELECT aidb.vector_index_chord_vchordq_config(
    vector_data_type => 'float32',
    lists => '1000',
    spherical_centroids => true
);
```

## Contextualizing hybrid search

VectorChord indices are compatible with the building blocks of **Hybrid Search**. By using `aidb.kb_query_encode()`, you can generate a query vector and use VectorChord-specific operators to perform similarity joins against filtered metadata.

**Example: Direct querying**

The `aidb.knowledge_bases` (alias `kbs`) view exposes the `distance_operator_sql` necessary to query VectorChord-indexed tables directly, bypassing the standard retrieval functions when custom SQL logic is required.
