---
title: Hybrid search
---

In EDB Postgres AI, hybrid search is not a standalone function but a flexible architecture built from several key SQL building blocks. These components allow you to construct custom queries that combine semantic vector distance with traditional full-text search or metadata filtering.

## Core building blocks

- **`aidb.kb_query_encode()`**: This function takes a Knowledge Base (KB) name and a text query, returning the corresponding vector embedding (as `real[]`) using the specific model configured for that KB. This is essential for manually executing similarity searches in custom SQL.

- **`aidb.distance_operator_to_sql()`**: A helper function that converts a DistanceOperator enum into the specific pgvector syntax required for SQL queries. For example, it transforms `L2` into `<- ->`, `Cosine` into `<=>`, and `InnerProduct` into `<#>`.

- **`aidb.knowledge_bases` view**: This view provides direct access to the `distance_operator_sql`, `vector_data_column`, and `vector_key_column` associated with your KBs, facilitating direct queries against the underlying vector tables.

## Reranking integration

While advanced reranking models such as `nim_reranking` or `hf_tei_reranking` are supported, they are not natively bundled into the standard `retrieve_text()` or `retrieve_key()` functions. Instead, you can optimize your search results by calling `aidb.rerank_text()` as a separate post processing step on your retrieved data.
