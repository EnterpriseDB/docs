As a core component of EDB Postgres AI, Pipelines transforms Postgres into a robust platform for AI data management. It abstracts away the complexities of AI workflows by combining `pgvector's` search capabilities with automated data preparation and lifecycle management.

## Integrated data lifecyle

Pipelines automates the journey from raw data to searchable insights. By managing data ingestion, embedding generation, and indexing, it ensures your vector stores stay synchronized with your source data.

- **Flexible model integration**: Choose from supported local models (HuggingFace) or connect to OpenAI-compatible external APIs.

- **Preprocessing automation**: Built-in preparation steps handle data cleaning and transformation directly within the AI Accelerator.

- **Eliminate stale data**: Auto-processing keeps embeddings current, significantly reducing the "hallucinations" often caused by outdated vector data.

- **Unified semantic search**: The Intelligent Knowledge Base feature allows you to query text and images using a single function call, regardless of whether the data lives in Postgres tables or S3-compatible object storage.

## How Pipelines work

Pipelines are powered by the AIDB extension embedded in the Postgres server. This extension provides the functions and views necessary to build and manage multi-step data flows.

### Pipeline architecture

The AIDB extension introduces the concept of a Pipeline that consists of one or more sequential steps:

- **Data sources**: Pipelines ingest data from standard Postgres columns or S3-compatible object storage.

- **Sequential processing**: The output of one step serves as the input for the next step.

- **Knowledge bases**: If a Pipeline's final step is a Knowledge base type, the entire pipeline is referred to as a Knowledge base.

### Auto-processing

While manual bulk processing is available using `aidb.run_pipeline()`, you can enable Auto-processing to handle updates automatically.

- **Configurability**: Supports synchronous, asynchronous, and batch processing.

- **Control**: Use `aidb.update_pipeline()` to toggle or tune automation based on your source and destination types.

For more information, see [Auto-processing](auto-processing.mdx).

### Intelligent Knowledge base & retrieval

Once your Knowledge base is established and embeddings are processed, retrieval is handled via simple function calls:

- **`aidb.retrieve_key()`**: Executes a similarity search based on your query input (text or image) and returns the IDs of the matched items.

- **`aidb.retrieve_text()`**: (Postgres tables only) Directly retrieves the source text for the matched items.

- **Multi-Modal Support**: Using models like CLIP, you can perform cross-media searches (e.g., using text to find images or vice versa).

- **Consistency**: The AIDB pipeline automatically ensures the same encoder model is used for both the initial embedding and the search query, guaranteeing mathematical accuracy.

### Supported models

The AIDB extension supports a diverse range of Open Encoder LLMs from HuggingFace (running locally on your Postgres node) and OpenAI Encoders (via cloud API). You can view all available options in the `aidb.model_providers` table.
