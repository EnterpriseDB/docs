Data pipelines in EDB Postgres AI automate the lifecycle of transforming raw data into vector-searchable insights. They remove the manual overhead of cleaning, chunking, and embedding data, ensuring your embeddings (knowledge bases) stay synchronized with your source records.

## Creating a pipeline

The core building block of a pipeline is the data processing configuration. This step defines the blueprint for how data is handled, transformed, and cleaned before it is indexed.

Depending on your data source, you can define two types of processing steps:

- **Table processing step**: Use `aidb.create_table_preparer()` to source data directly from existing Postgres tables.

- **Volume processing step**: Use `aidb.create_volume_preparer()` to source unstructured data such as PDFs or images from external object storage via PGFS.

**Key configuration parameters**

- **Source**: The specific Postgres table or PGFS volume containing the raw input data.

- **Embedding model**: The specific model used to generate vectors (for example OpenAI, HuggingFace, or local models).

- **Destination**: The designated embeddings (knowledge base) where the processed vectors will be stored and indexed.

## Pulling in data (PGFS)

To process files stored outside of Postgres, Pipelines utilize Postgres File System (PGFS) and volumes. PGFS acts as the file-ssytem bridge that allows the database to mount external cloud storage as if it were a local directory.

- **PGFS integration**: You first define a storage location (S3, GCS, or Azure) using `pgfs.create_storage_location`. This establishes the connection and authentication via IAM roles, environment variables, or static credentials.

- **AIDB Volumes**: Once a PGFS location is defined, you create an AIDB volume. This volume allows the pipeline to scan the external storage, detect new files, and pull them into the processing stream.

- **Unstructured data support**: This setup is specifically designed to pull in unstructured formats like PDFs, images, and text files that reside in object storage, making them available for downstream steps like OCR or chunking.
