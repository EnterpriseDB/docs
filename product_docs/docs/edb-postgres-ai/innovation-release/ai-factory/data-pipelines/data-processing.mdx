AI pipelines automate the lifecycle of transforming raw data into vector-searchable insights. They remove the manual overhead of cleaning, chunking, and embedding data, ensuring your embeddings (knowledge bases) stay synchronized with your source records.

## Creating a pipeline

Creating a pipeline in EDB Postgres AI is the process of defining a structured approach for how data moves from a source to an AI-ready destination. The `aidb.create_pipeline()` function allows you to set up this workflow by specifying the source of your data, the transformations it should undergo, and where the processed data should be stored.

1. **Data source**: Pipelines can ingest data from two distinct types of sources:
   
   - **Table sources**: Data is pulled directly from existing Postgres tables using the `source`, `source_key_column`, and `source_data_column` parameters.
   
   - **Volume sources**: For data in external cloud storage (like S3), the pipeline utilizes PGFS (Postgres File System). This requires a two-step setup: first defining the storage location and volume, then referencing that volume as the pipeline `source`.
   
1. **Modular step configuration**: The power of a pipeline lies in its ability to chain atomic operations, known as steps.

   - **Singe-step**: A basic pipeline might simply point a source to a `KnowledgeBase` to generate embeddings.

   - **Multi-step**: You can chain up to 10 operations for example, `ParseHtml` followed by `ChunkText` to transform complex, unstructured data into a more manageable format before embedding.

   - **Step options**: Each step is configured using specialized helper functions like `aidb.knowledge_base_config()` that pass a `JSONB` block of instructions to the underlying AI models.

1. **Orchestration & destination**: Once the pipeline is defined, it manages the heavy lifting of AI processing. It automatically tracks which records have been processed and stores the final output, typically high dimensional vectors into a Knowledge base for semantic search.

## Pulling in data (PGFS)

To process files stored outside of Postgres, pipelines utilize Postgres File System (PGFS) and volumes. PGFS acts as the file-sytem bridge that allows the database to mount external cloud storage as if it were a local directory.

- **PGFS integration**: You first define a storage location (S3, GCS, or Azure) using `pgfs.create_storage_location`. This establishes the connection and authentication via IAM roles, environment variables, or static credentials.

- **AIDB Volumes**: Once a PGFS location is defined, you create an AIDB volume. This volume allows the pipeline to scan the external storage, detect new files, and pull them into the processing stream.

- **Unstructured data support**: This setup is specifically designed to pull in unstructured formats like PDFs, images, and text files that reside in object storage, making them available for downstream steps like OCR or chunking.

For more information, see [PGFS documentation](/pg_extensions/pgfs/).
