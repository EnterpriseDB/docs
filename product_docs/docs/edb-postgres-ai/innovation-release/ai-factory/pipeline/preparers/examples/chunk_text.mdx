These examples use preparers with the [ChunkText operation](../primitives#chunk-text) in AI Accelerator.

!!! Tip
This operation transforms the shape of the data, automatically unnesting collections by introducing a `part_id` column. See the [unnesting concept](../concepts#unnesting) for more detail.
!!!

## Primitive

```sql
-- Only specify a desired length
SELECT * FROM aidb.chunk_text(
    'This is a simple test sentence.',
    aidb.chunk_text_config(desired_length => 10)
);

__OUTPUT__
 part_id |   chunk
---------+-----------
       0 | This is a
       1 | simple
       2 | test
       3 | sentence.
(4 rows)
```

```sql
-- Specify a desired length and a maximum length
SELECT * FROM aidb.chunk_text(
    'This is a simple test sentence.',
    aidb.chunk_text_config(desired_length => 10, max_length => 15)
);

__OUTPUT__
 part_id |    chunk
---------+-------------
       0 | This is a
       1 | simple test
       2 | sentence.
(3 rows)
```

```sql
-- Named parameters
SELECT * FROM aidb.chunk_text(
    input => 'This is a significantly longer text example that might require splitting into smaller chunks. The purpose of this function is to partition text data into segments of a specified maximum length, for example, this sentence 145 is characters. This enables processing or storage of data in manageable parts.',
    options => aidb.chunk_text_config(desired_length => 40)
);

__OUTPUT__
 part_id |                 chunk
---------+----------------------------------------
       0 | This is a significantly longer text
       1 | example that might require splitting
       2 | into smaller chunks.
       3 | The purpose of this function is to
       4 | partition text data into segments of a
       5 | specified maximum length, for example,
       6 | this sentence 145 is characters.
       7 | This enables processing or storage of
       8 | data in manageable parts.
(9 rows)
```

## Preparer with table data source

```sql
-- Create source test table
CREATE TABLE source_table__1628
(
    id      INT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    content TEXT NOT NULL
);
INSERT INTO source_table__1628
VALUES (1, 'This is a significantly longer text example that might require splitting into smaller chunks. The purpose of this function is to partition text data into segments of a specified maximum length, for example, this sentence 145 is characters. This enables processing or storage of data in manageable parts.'),
       (2, 'This is sentence number one. This is sentence number one.');

SELECT aidb.create_table_preparer(
    name => 'preparer__1628',
    operation => 'ChunkText',
    source_table => 'source_table__1628',
    source_data_column => 'content',
    destination_table => 'chunked_data__1628',
    source_key_column => 'id',
    destination_key_column => 'id',
    options => aidb.chunk_text_config(desired_length => 120)  -- Configuration for the ChunkText operation
);

SELECT aidb.bulk_data_preparation('preparer__1628');

SELECT * FROM chunked_data__1628;

__OUTPUT__
 id | source_id | part_ids |                                                         value
----+-----------+----------+-----------------------------------------------------------------------------------------------------------------------
  1 | 1         | {0}      | This is a significantly longer text example that might require splitting into smaller chunks.
  2 | 1         | {1}      | The purpose of this function is to partition text data into segments of a specified maximum length, for example, this
  3 | 1         | {2}      | sentence 145 is characters. This enables processing or storage of data in manageable parts.
  4 | 2         | {0}      | This is sentence number one. This is sentence number one.
(4 rows)
```

## Chunk text with overlap

The `overlap_length` parameter allows you to specify how much content should be duplicated between consecutive chunks, which can help preserve context across chunk boundaries.

```sql
-- Create source test table
CREATE TABLE source_table__1628_overlap
(
    id      INT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    content TEXT NOT NULL
);
INSERT INTO source_table__1628_overlap
VALUES (1, 'This is a significantly longer text example that might require splitting into smaller chunks. The purpose of this function is to partition text data into segments of a specified maximum length.');

SELECT aidb.create_table_preparer(
    name => 'preparer__1628_overlap',
    operation => 'ChunkText',
    source_table => 'source_table__1628_overlap',
    source_data_column => 'content',
    destination_table => 'destination_table__1628_overlap',
    source_key_column => 'id',
    options => aidb.chunk_text_config(desired_length => 50, overlap_length => 10)
);

SELECT aidb.bulk_data_preparation('preparer__1628_overlap');

SELECT * FROM destination_table__1628_overlap ORDER BY id;

__OUTPUT__
 id | source_id | part_ids |                       value
----+-----------+----------+---------------------------------------------------
  1 | 1         | {0}      | This is a significantly longer text example that
  2 | 1         | {1}      | that might require splitting into smaller chunks.
  3 | 1         | {2}      | The purpose of this function is to partition text
  4 | 1         | {3}      | text data into segments of a specified maximum
  5 | 1         | {4}      | maximum length.
(5 rows)
```

## Chunk text with word strategy

By default, chunking uses a character-based strategy. You can specify `"strategy": "words"` to use word-based chunking instead.

```sql
-- Using word-based chunking strategy
SELECT * FROM aidb.chunk_text(
    input => 'This is a simple test sentence for word-based chunking.',
    options => aidb.chunk_text_config(desired_length => 4, strategy => 'words')
);

__OUTPUT__
 part_id |            chunk
---------+------------------------------
       0 | This is a simple
       1 | test sentence for word-based
       2 | chunking.
(3 rows)
```

```sql
-- Combining word strategy with overlap
SELECT * FROM aidb.chunk_text(
    input => 'This is a simple test sentence for word-based chunking with overlap.',
    options => aidb.chunk_text_config(desired_length => 5, overlap_length => 2, strategy => 'words')
);

__OUTPUT__
 part_id |                 chunk
---------+---------------------------------------
       0 | This is a simple test
       1 | simple test sentence for word-based
       2 | for word-based chunking with overlap.
(3 rows)
```
