Each pipeline step operation is also available as a standalone SQL function. You can call these functions directly in queries for one-off transformations, for testing and exploration, or to build custom workflows outside of the pipeline framework.

| Function                                            | Operation     | Description                                                                  |
|-----------------------------------------------------|---------------|------------------------------------------------------------------------------|
| [`aidb.chunk_text()`](chunk-text)                   | ChunkText     | Divides long text into smaller, semantically coherent segments.              |
| [`aidb.parse_html()`](parse)                        | ParseHtml     | Extracts readable text from HTML, stripping tags while preserving structure. |
| [`aidb.parse_pdf()`](parse)                         | ParsePdf      | Extracts text from binary PDF data, with page-level `part_id` output.        |
| [`aidb.perform_ocr()`](perform-ocr)                 | PerformOcr    | Extracts text from images using an OCR-capable AI model.                     |
| [`aidb.summarize_text()`](summarize-text)           | SummarizeText | Generates concise summaries of long text passages using an AI model.         |
| [`aidb.summarize_text_aggregate()`](summarize-text) | SummarizeText | Summarizes text across multiple rows using a SQL aggregate pattern.          |

!!!Note
To use these operations as steps inside a pipeline, see [Pipeline steps](../data-pipelines/pipeline-steps/pipeline-steps.mdx).
