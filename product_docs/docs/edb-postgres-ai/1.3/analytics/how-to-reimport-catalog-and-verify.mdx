---
title: How-to  Check that catalog changes are reflected in PGAA
navTitle: Reimport catalog and verify
description: Load a new table via the loader, reimport the catalog, and verify PGAA sees it while transactional tables remain intact.
---

```bash
python loader.py \
  --ca-cert ca.crt \
  --bucket beacon-analytics-demo-data-eu-west-1-prod \
  --prefix tpch_sf_1/customer \
  --endpoint $PARAMS_URI \
  --token $API_WRITE \
  --warehouse $PARAMS_WAREHOUSE \
  --namespace tpch_sf_1 \
  --table-name customer
```

```sql
SELECT bdr.replicate_ddl_command($$
  SELECT pgaa.import_catalog('lakekeeper-test')
$$);

SELECT * FROM tpch_sf_1.customer LIMIT 1;
SELECT * FROM tpch_sf_1.sentinel_tx;  -- should return 'still alive'

-- Optionally stop auto-refresh
SELECT bdr.replicate_ddl_command($$ SELECT pgaa.detach_catalog('lakekeeper-test') $$);
```
