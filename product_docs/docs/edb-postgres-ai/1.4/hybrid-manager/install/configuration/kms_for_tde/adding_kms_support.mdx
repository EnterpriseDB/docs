Transparent Data Encryption (TDE) is a compliance standard for protecting data at rest, not an optional feature. Regulated industries and enterprise security policies require it. The security assurance of TDE depends on the key protection method: where the key is stored, who controls it, and whether its custody can be audited independently of the data.

For production, always use a KMS-backed key provider. Passphrase mode provides encryption but not assurance — the key custodian and the data custodian are the same principal, which auditors don't accept for PCI, HIPAA, or FedRAMP workloads. The passphrase is also an operational risk: losing it makes data unrecoverable, and leaking it voids the encryption entirely. This risk is especially acute in disaster recovery scenarios, where the passphrase is most likely to be unavailable — making it impossible to decrypt and restore your data. Don't automate passphrase-based TDE.

KMS integration separates key custody from data custody, supports audit trails, keeps the key independently accessible for disaster recovery, and is the only appropriate method for automated and production deployments.

When you deploy the Hybrid Manager (HM), the `HybridControlPlane` Custom Resource includes `passphrase` and `hashicorp_vault` as the default TDE methods:

```shell
  # - transparentDataEncryptionMethods: List of TDE Encryption Methods for the agent.
  #     Supported options: passphrase, aws_kms, azure_kms, gcp_kms, hashicorp_vault.
  #     Default values: passphrase, hashicorp_vault.
  transparentDataEncryptionMethods:
    - passphrase
    - hashicorp_vault
```

!!!note
The `passphrase` option is included for convenience in development and testing environments only. For production, configure one or more of the KMS providers below.
!!!

You can edit this section during installation, or add a key provider after installation by reapplying the `HybridControlPlane` CR with the updated configuration. See the [installation guides](../../install_guide/) for infrastructure-specific instructions on how to apply the CR.

## HashiCorp Vault

The Hybrid Manager configuration includes support for HashiCorp out of the box. 

## AWS KMS

To use Amazon's KMS, Amazon Key Management Service, add `aws_kms` to the list: 

```shell
  # - transparentDataEncryptionMethods: List of TDE Encryption Methods for the agent.
  #     Supported options: passphrase, aws_kms, azure_kms, gcp_kms, hashicorp_vault.
  #     Default values: passphrase, hashicorp_vault.
  transparentDataEncryptionMethods:
    - passphrase
    - hashicorp_vault
    - aws_kms
```

## Azure KMS

To use Azure's KMS, Azure Key Vault, add `azure_kms` to the list:

```shell
  # - transparentDataEncryptionMethods: List of TDE Encryption Methods for the agent.
  #     Supported options: passphrase, aws_kms, azure_kms, gcp_kms, hashicorp_vault.
  #     Default values: passphrase, hashicorp_vault.
  transparentDataEncryptionMethods:
    - passphrase
    - hashicorp_vault
    - azure_kms
```

## Google KMS

If you want to use Google's KMS, Cloud Key Management Service, add `gcp_kms` to the list: 

```shell
  # - transparentDataEncryptionMethods: List of TDE Encryption Methods for the agent.
  #     Supported options: passphrase, aws_kms, azure_kms, gcp_kms, hashicorp_vault.
  #     Default values: passphrase, hashicorp_vault.
  transparentDataEncryptionMethods:
    - passphrase
    - hashicorp_vault
    - gcp_kms
```
