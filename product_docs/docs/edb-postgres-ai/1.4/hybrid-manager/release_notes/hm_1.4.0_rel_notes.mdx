Hybrid Manager (HM) 1.4.0 (LTS) expands deployment flexibility with Microsoft AKS support, simplifies day-to-day operations with a new automations engine, and deepens AI capabilities across pipelines, migration, and the built-in chatbot. This long-term support release consolidates all features from Innovation Releases 2025.11 through 2026.5.1. It also includes numerous bug fixes and quality-of-life improvements.

**Release Date:** June 24, 2026

!!!note Postgres refers to all distributions supported by EDB

-   PostgreSQL
-   EDB Enterprise Postgres - formerly, EDB Postgres Extended Server (PGE)
-   EDB Enterprise Postgres (Oracle Compatible) - formerly, EDB Postgres Advanced Server (EPAS)
!!!

## Release highlights

-   **[Azure Kubernetes Service (AKS) support](/edb-postgres-ai/current/hybrid-manager/install/install_guide/k8s_install/)**: HM can now be deployed on Microsoft AKS, with full support for AI Pipelines, Analytics Accelerator, and the Data Migration Service.

-   **[Selective installations](/edb-postgres-ai/current/hybrid-manager/install/install_guide/install_hm/)**: Customize your HM deployment by enabling only the `scenarios` you need — `core`, `dbaas`, `migration`, `ai`, and `analytics` — at install time or later, reducing resource usage and simplifying component management.

-   **[Automations engine](/edb-postgres-ai/current/hybrid-manager/using_hybrid_manager/cluster_management/hm_automations/)**: A new automation framework handles routine database operations including disk auto-scaling, CPU and memory scaling, index application, and minor and patch database version updates — all with configurable maintenance windows and optional human-in-the-loop approvals.

-   **[OSS Library](/edb-postgres-ai/current/hybrid-manager/using_hybrid_manager/hm_overview/oss_library_apps/)**: Deploy community database tools — Apache Airflow, Apache Superset, Metabase, pgAdmin 4, pgBadger, and HammerDB — directly to HM projects from the **Asset Library**, with configurable parameters and approval workflows.

-   **[Chat Agent](/edb-postgres-ai/current/hybrid-manager/using_hybrid_manager/chat_agent/)**: A conversational assistant built into the HM console for Postgres cluster management, estate browsing, monitoring and alerts, health report retrieval, Oracle-to-Postgres SQL translation, and documentation Q&A with citation-backed answers. Supports models from OpenAI, Anthropic, and Google Gemini, as well as self-hosted models deployed through HM.

-   **[Custom load balancer configuration](/edb-postgres-ai/current/hybrid-manager/install/install_guide/install_hm/)**: HM supports custom load balancer annotations for use cases such as MetalLB address pool assignment and disabling Azure Private Link on Azure private clusters.

-   **[Three-location cluster deployments](/edb-postgres-ai/current/hybrid-manager/install/configuration/multi_dc/)**: Deploy Postgres clusters across three geographically separated locations for zero-data-loss configurations via synchronous replication. HM also supports Distributed High Availability (DHA) Witness Groups in a third location for quorum and split-brain prevention.

-   **[Unified schema migration](/edb-postgres-ai/current/hybrid-manager/using_hybrid_manager/migration/schema_migration/postgres/)**: Migrate database schemas from Oracle and Postgres sources to Postgres destinations directly through HM, with Oracle SQL compatibility assessment and API-driven automation via AI agents.

-   **[Model Context Protocol (MCP) integration](/edb-postgres-ai/current/hybrid-manager/using_hybrid_manager/interacting_programmatically/using_mcp_tools/)**: HM now exposes its API capabilities through MCP, enabling AI agents and developer tooling to consume HM APIs for agentic workflows.

-   **[Agent Factory enhancements](/edb-postgres-ai/current/hybrid-manager/ai-factory/pipeline-designer/)**: Visual AI Pipeline Designer, multistep composable pipelines, hybrid search, the `summarize_text_aggregate()` SQL function, and expanded Langflow components including EDB Airman MCP Server, EDB Embedded Models, and EDB Vector Index.

-   **[Langflow integration](/edb-postgres-ai/current/hybrid-manager/ai-factory/langflow/)**: Langflow is natively integrated into the HM Launchpad. You can build AI-powered agents and workflows within HM, deploy flows as managed workloads, and use a Langflow template for end-to-end database migrations from Oracle and Postgres sources to Postgres destinations.

-   **[Enhanced migration risk assessment](/edb-postgres-ai/current/hybrid-manager/using_hybrid_manager/migration/migrations/assessing_multiple_databases/)**: Granular assessments for source databases covering replication slot health, WAL generation rates, and risky table configurations (primary keys, `REPLICA IDENTITY`, `TOAST`, and unsupported data types for the Data Migration Service).

-   **[Advanced observability](/edb-postgres-ai/current/hybrid-manager/using_hybrid_manager/monitoring/)**: Daily cluster health reports, schema change detection, new query detection, a metrics comparison tool, and enhanced monitoring dashboards.

-   **[ARM architecture support](/edb-postgres-ai/current/hybrid-manager/install/install_guide/sys_reqs/)**: HM components now run on ARM-based architectures, enabling deployments on modern ARM platforms and Kubernetes nodes.

After the snapshot stage completes, you can select **Finalize Schema Concurrently** to apply indexes, foreign keys, and other schema objects in the background using non-blocking DDL variants, significantly reducing cutover time. |
