These are the tasks involved in an end-to-end migration from an AWS RDS for PostgreSQL or Aurora PostgreSQL database to a database cluster managed by Hybrid Manager (HM). The process uses two agents: the EDB Postgres AI agent (`beacon-agent`) to register your database with HM and ingest schema for assessment, and the EDB DMS agent (`cdcagent`) to perform the actual data migration.

1. **[Configure your auxiliary machine](auxiliary_machine)** — Configure the machine that hosts both migration agents. This machine acts as the bridge between your AWS RDS or Aurora PostgreSQL source and HM.

1. **[Prepare your source AWS RDS/Aurora PostgreSQL database](preparing_source_pg)** — Create the required users, configure schema extraction permissions, enable logical replication via an AWS DB cluster parameter group, and validate your configuration.

1. **[Configure the EDB Postgres AI agent](agent_postgres)** — Connect the agent to your source database and to HM to enable schema ingestion and migration assessment.

1. **[Prepare a destination database](creating_destination_db)** — Create or identify the HM-managed database cluster that will receive the migrated schema and data.

1. **[Migrate your schema](migrating_schema)** — Choose between the simplified path, where HM handles schema migration and constraint ordering automatically, or the manual path, where you export, prepare, and import the schema yourself.

1. **[Configure the EDB DMS agent](configuring_dms_agent)** — Create migration credentials, configure the DMS agent in reader mode to read changes from your AWS RDS or Aurora PostgreSQL source, and verify the source appears in HM.

1. **[Migrate your data](migrating_data)** — Create and run the data migration in the HM console, monitor its progress, finalize streaming, and apply post-migration constraints.
