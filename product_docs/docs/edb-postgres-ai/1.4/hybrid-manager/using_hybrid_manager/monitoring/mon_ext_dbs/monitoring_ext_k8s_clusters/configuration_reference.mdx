## TLS configuration options

These options control how the agent trusts the Hybrid Manager (HM) beacon server certificate. By default (`rootCAEnabled: true`), the agent mounts a CA secret and verifies HM using that certificate.

| Parameter                                         | Description                                                                                                                                                   | Default              | Required |
|---------------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------|----------------------|----------|
| rootCAEnabled                                     | When `true`, mounts the CA secret and sets the agent's `root_ca_path`. Set to `false` to trust HM via the system CA pool (or skip verification) without mounting a CA secret. | `true`               | No       |
| rootCASecretName                                  | Name of the secret holding the HM root CA (`ca.crt`). Must exist when `rootCAEnabled` is `true`.                                                              | `beacon-server-ca`   | No       |
| rootCA                                            | PEM-encoded root CA. When set, the chart creates the CA secret for you instead of requiring a pre-existing one.                                               | `""`                 | No       |
| agentConfig.beaconServer.rootCAPath               | Path inside the pod where the CA certificate is mounted and read from. Only used when `rootCAEnabled` is `true`.                                              | `/etc/ssl/certs/ca-certificates.crt` | No       |
| agentConfig.beaconServer.tlsInsecureSkipVerify    | When `true`, the agent skips TLS verification of the HM beacon server certificate. Typically paired with `rootCAEnabled: false` for testing.                  | `false`              | No       |
| agentConfig.beaconServer.plaintext                | When `true`, the agent connects to HM without TLS.                                                                                                           | `false`              | No       |

!!!note

When `rootCAEnabled: false`, you do not need to create the `beacon-server-ca` secret described in the [requirements](installation_requirements/#namespace-and-secrets). The agent relies on the system CA pool (or skips verification when `tlsInsecureSkipVerify: true`), and the CA volume mount is not injected into the agent, OTEL collector, or CloudNativePG™ sidecar pods.
!!!
