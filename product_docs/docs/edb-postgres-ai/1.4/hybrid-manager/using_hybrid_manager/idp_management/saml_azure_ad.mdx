This page walks you through configuring SAML single sign-on (SSO) for Hybrid Manager (HM) using Azure Active Directory (Azure AD) as your identity provider (IdP). It covers the specific field mappings between the Azure portal and the HM console.

For a general overview of the SAML configuration process in HM, see [Configuring your IdP in the HM console](config_idp_hm).

## Before you begin

Before starting, confirm the following:

1. You must be an HM organization owner to configure an IdP in HM.
1. The user accounts you want to use with HM must have the **Email** attribute populated in their Azure AD profile. This attribute isn't always set by default. See [Add or update a user's profile information](https://learn.microsoft.com/en-us/entra/fundamentals/how-to-manage-user-profile-info) in the Microsoft documentation.

## Starting the SAML configuration in HM

1. In the HM console, select **Settings** > **Identity Providers**.
1. Select **Add Identity Provider**.
1. Select the **SAML** tab.
1. Copy the following values — you'll need them for the Azure AD configuration:
   - **Assertion Consumer Service URL**
   - **Audience URI (SP Entity ID)**

!!! important
Don't close or navigate away from this page until you complete the HM configuration. The **Assertion Consumer Service URL** and **Audience URI** change each time you start a new configuration.
!!!

## Creating an enterprise application in Azure AD

In a separate browser tab:

1. Sign in to the [Azure portal](https://portal.azure.com) and go to **Azure Active Directory** > **Enterprise applications**.
1. Select **New application** > **Create your own application**.
1. Enter a name for the application, for example, `edb-hybrid-manager`.
1. Select **Integrate any other application you don't find in the gallery (Non-gallery)** and select **Create**.
1. In the application overview, select **Single sign-on** > **SAML**.

## Configuring basic SAML settings in Azure AD

In the **Basic SAML Configuration** section, select **Edit** and enter the following:

| Azure AD field | Value |
|---|---|
| **Identifier (Entity ID)** | Paste the **Audience URI** from HM |
| **Reply URL (Assertion Consumer Service URL)** | Paste the **Assertion Consumer Service URL** from HM |

Leave all other fields empty unless your organization requires them. Select **Save**.

## Configuring attributes and claims in Azure AD

The default attributes and claims that Azure AD provides are sufficient for HM. Verify that the following claims are present:

| Claim name | Value |
|---|---|
| `emailaddress` | `user.mail` |
| `name` | `user.displayname` |
| Unique User Identifier | `user.userprincipalname` |
| `groups` | `user.groups` |

!!! note
The `groups` claim is optional. Configure it only if you want to enable [role mapping](../managing_users/user_roles_authz/role_mapping) from Azure AD groups to HM roles.
!!!

!!! note
Claim names and values can vary depending on your organization's Azure AD configuration. Work with your IT department to confirm the correct values for your environment.
!!!

## Downloading the certificate and login URL from Azure AD

In the **SAML Certificates** section:

1. Next to **Certificate (Base64)**, select **Download** and save the file.

In the **Set up \<your-app-name\>** section:

2. Copy the **Login URL**.

## Completing the SAML configuration in HM

Return to the HM browser tab and enter the following:

1. In the **Name** field, enter a unique name for this identity provider.
1. In the **Description** field, enter an optional description.
1. In **Single Sign-On URL**, paste the **Login URL** from Azure AD.
1. Under **Signature Certificate File**, upload the Base64 certificate you downloaded from Azure AD.
1. Under **NameID Format**, select **Email Address**.
1. Enter the following attribute values:

   | HM field | Value |
   |---|---|
   | **Username Attribute** | `http://schemas.xmlsoap.org/ws/2005/05/identity/claims/name` |
   | **Email Attribute** | `http://schemas.xmlsoap.org/ws/2005/05/identity/claims/emailaddress` |

   !!! note
   These values reflect the default Azure AD claim URIs and can vary depending on your organization's configuration. Work with your IT department to confirm the correct values.
   !!!

1. (Optional) To enable [role mapping](../managing_users/user_roles_authz/role_mapping), configure group attribute mapping:

   | HM field | Value |
   |---|---|
   | **Groups Attribute** | The SAML attribute name containing the user's group memberships |
   | **Groups Delimiter** | The delimiter separating multiple groups in the attribute value. Leave empty if your IdP sends groups as separate attribute values. |

1. Select **Save & Test** to validate the connection.
