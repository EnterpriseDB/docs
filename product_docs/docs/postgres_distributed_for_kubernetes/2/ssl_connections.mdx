    for more details on how EDB Postgres® AI for CloudNativePG™ Global Cluster supports TLS certificates.

The EDB CloudNativePG Global Cluster operator was designed to work with TLS/SSL for both encryption in transit and
authentication on the server and client sides. PGD nodes are created as cluster
resources using the PG4K operator. This
includes deploying a certificate
authority (CA) to create and sign TLS client certificates.

See the [PG4K documentation](/postgres_for_kubernetes/latest/ssl_connections/)
for more information on issuers and certificates.
