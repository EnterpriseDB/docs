---
title: "BACKUP"
---



Use the `BACKUP` subcommand to create a full or incremental backup.

**Syntax for a Full Backup:**

    bart BACKUP –s { <server_name> | all } [ -F { p | t } ]

    [ -z ] [ –c <compression_level> ]

    [ --backup-name <backup_name> ]

    [ --thread-count <number_of_threads> ]

    [ { --with-pg_basebackup | --no-pg_basebackup } ]

**Syntax for an Incremental Backup:**

    bart BACKUP –s <server_name> [-Fp]

    [ --parent { <backup_id> | <backup_name> } ]

    [ --backup-name <backup_name> ]

    [ --thread-count <number_of_threads> ]

Please note that before performing an incremental backup, you must take a full backup. For more details about incremental backup, refer to *Block-Level Incremental Backup* in the EDB Postgres Backup and Recovery User Guide available at the [EDB website](https://www.enterprisedb.com/edb-docs/).

The following table describes the `BACKUP` options:

<table><thead><tr class="header"><th>Options</th><th>Description</th></tr></thead><tbody><tr class="odd"><td><code>-s</code> or <code>--server &#123; &lt;server_name&gt; | all &#125;</code></td><td><p>Use this option to specify the database server to be backed up.<br />
Specify <code>all</code> to take a backup of all servers.<br />
Specify <code>&lt;server_name&gt;</code> to take a backup of the database server (as specified in the BART configuration file).</p></td></tr><tr class="even"><td><code>-F</code> or <code>--format &#123; p | t &#125;</code></td><td><p>Use this option to specify the backup file format.<br />
Specify <code>p</code> option to take a backup in plain text format and specify <code>t</code> option to take a backup in tar format. If the <code>p</code> or <code>t</code> option is omitted, the default is tar format.<br />
Use <code>p</code> option with the <code>BACKUP</code> subcommand when <code>streaming</code> is used as a backup method.<br />
An incremental backup can only be taken in plain text format (<code>p</code>).</p></td></tr><tr class="odd"><td><code>-z</code> or <code>--gzip</code> (applicable only for full backup and <code>tar</code> format)</td><td>Use this option to enable gzip compression of tar files using the default compression level (typically 6).</td></tr><tr class="even"><td><code>-c</code> or <code>--compress-level &lt;compression_level&gt;</code> (applicable only for full backup and tar format)</td><td>Use this option to specify the gzip compression level on the tar file output. <code>&lt;compression_level&gt;</code> is a digit from 1 through 9, with 9 being the best compression.</td></tr><tr class="odd"><td><code>--backup-name &lt;backup_name&gt;</code></td><td><p>Use this option to assign a user-defined, alphanumeric friendly name to the backup. The maximum permitted length of backup name is 49 characters.<br />
For detailed information about this parameter, see the EDB Postgres Backup and Recovery User Guide available at the <a href="https://www.enterprisedb.com/edb-docs/">EDB website</a>.<br />
If the option <code>--backup-name</code> is not specified and the <code>backup_name</code> parameter is not set for this database server in the BART configuration file, then the backup can only be referenced in other BART subcommands by the BART assigned backup identifier.</p></td></tr><tr class="even"><td><code>--thread-count &lt;number_of_threads&gt;</code></td><td><p>Use this option to specify the number of worker threads to run in parallel to copy blocks for a backup.<br />
For detailed information about the <code>--thread-count</code> parameter, see the EDB Postgres Backup and Recovery Installation and Upgrade Guide available at the <a href="https://www.enterprisedb.com/edb-docs/">EDB website</a>.</p></td></tr><tr class="odd"><td><code>--with-pg_basebackup</code> (applicable only for full backup)</td><td><p>Use this option to specify the use of <code>pg_basebackup</code> to take a full backup. The number of thread counts in effect is ignored as given by the <code>thread_count</code> parameter in the BART configuration file.<br />
When taking a full backup, if the thread count in effect is greater than <code>1</code>, then the <code>pg_basebackup</code> utility is not used to take the full backup (parallel worker threads are used) unless the <code>--with-pg_basebackup</code> option is specified with the <code>BACKUP</code> subcommand.</p></td></tr><tr class="even"><td><code>--no-pg_basebackup</code> (applicable only for full backup)</td><td><p>Use this option to specify that <code>pg_basebackup</code> is not to be used to take a full backup.<br />
When taking a full backup, if the thread count in effect is only <code>1</code>, then the <code>pg_basebackup</code> utility is used to take the full backup unless the <code>--no-pg_basebackup</code> option is specified with the <code>BACKUP</code> subcommand.</p></td></tr><tr class="odd"><td><code>--parent &#123; &lt;backup_id&gt; | &lt;backup_name&gt; &#125;</code></td><td>Use this option to take an incremental backup. The parent backup is a backup taken prior to the incremental backup; it can be either a full backup or an incremental backup. <code>&lt;backup_id&gt;</code> is the backup identifier of a parent backup and <code>&lt;backup_name&gt;</code> is the user-defined alphanumeric name of a parent backup.</td></tr><tr class="even"><td><code>--check</code> (applicable only for incremental backup)</td><td><p>Use this option to verify if the required MBM files are present in the BART backup catalog before taking an incremental backup. However, an actual incremental backup is not taken when the <code>--check</code> option is specified.<br />
The <code>--parent</code> option must be used along with the <code>--check</code> option.</p></td></tr></tbody></table>



**Examples**

The following code sample demonstrates using variables with the `BACKUP` subcommand:

    ./bart backup -s ppas12 -Ft --backup-name "YEAR = %year MONTH =
    %month DAY = %day"

    ./bart backup -s ppas12 -Ft --backup-name "YEAR = %year MONTH =
    %month DAY = %day %%"

    ./bart show-backups -s ppas12 -i "test backup"
