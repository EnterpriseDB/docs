title: "SHOW-BACKUPS"
---

The `SHOW-BACKUPS` subcommand displays the backup information for the managed database servers.

**Syntax:**

```text
bart SHOW-BACKUPS [ â€“s { <server_name> | all } ]
  [ -i { <backup_id> | <backup_name> | all } ]
  [ -t ]
```

The following table describes the command options:

<code>--server &#123; &lt;server_name&gt; | all &#125;</code></p></td><td><p><code>&lt;server_name&gt;</code> is the name of the database server whose backup information is to be displayed.<br />
If <code>all</code> is specified or if the option is omitted, the backup information for all database servers is displayed with the exception as described by the following note:<br />
If <code>SHOW-BACKUPS</code> is invoked while the BART <code>BACKUP</code> subcommand is in progress, backups affected by the backup process are shown in progress status in the displayed backup information.</p></td></tr><tr class="even"><td><p><code>-i &#123; &lt;backup_id&gt; | &lt;backup_name&gt; | all &#125;</code><br />
<code>--backupid &#123; &lt;backup_id&gt; | &lt;backup_name&gt; | all &#125;</code></p></td><td><p><code>&lt;backup_id&gt;</code> is a backup identifier and <code>&lt;backup_name&gt;</code> is the user-defined alphanumeric name for the backup.<br />
If <code>all</code> is specified or if the option is omitted, all backup information for the relevant database server is displayed.</p></td></tr><tr class="odd"><td><p><code>-t</code><br />
<code>--toggle</code></p></td><td>Displays more backup information in a list format. If the option is omitted, the default is a tabular format.</td></tr></tbody></table>
