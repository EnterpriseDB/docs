Manage and review diagnostic data for your monitored servers. PostgreSQL Enterprise Manager (PEM) provides two distinct reporting frameworks to suit your diagnostic needs.

## [Custom Reports](/pem/10/reports/custom_reports)
The **Custom Reports** framework allows you to design and execute bespoke diagnostic templates. By combining SQL queries and host-level scripts, you can extract specific insights tailored to your unique infrastructure requirements.

- **User-defined logic** &mdash; Build templates from the ground up using SQL or shell/batch scripts.
- **Hierarchical structure** &mdash; Organize data into logical sections and items for clear reporting.
- **Portability** &mdash; Export and import templates via JSON for easy migration across environments.

## [System Reports](/pem/10/reports/reports)
**System Reports** are the pre-defined, built-in diagnostic tools provided by PEM. These reports offer out-of-the-box analysis of database performance, security audits, and configuration states.

- **Standardized metrics** &mdash; Access industry-standard reports for database health and activity.
- **No configuration required** &mdash; Execute immediate diagnostics without the need for custom script writing.
- **Deep Integration** &mdash; Fully integrated with PEM’s alerting and monitoring history.

!!! Note 
      To run a report, simply right-click a **Target Server** in the PEM Browser tree and navigate to the relevant report category.
