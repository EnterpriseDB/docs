---
title: "Version 8.0.0"
---
New features, enhancements, bug fixes, and other changes in PEM 8.0.0 include:

| Highlights | Type |
| ---------- | ---- |
| **Webhooks**: You can now use PEM to send event based notifications to webhook endpoints like third party systems or partner applications. | New Feature |
| **Performance Diagnostics**: Use the Performance Diagnostic tool with PostgreSQL 10 or later versions after installing 'edb_wait_states' plugin. | Enhancement |
| **PEM Backend Database**: The PEM Server (backend database) supports only PostgreSQL or EDB Postgres Advanced Server versions 11 or later. | Change |
| **Enhanced Alerting**: You can now replace alert placeholders inside the script and also can view more informative and contextual alert details. | Enhancement |
| **Enhanced BART Integration**: Configure 'bart_socket_name' and also parameters such as '--checksum-algorithm' and '--disable-checksum' through PEM. | Enhancement |
| **Security Best Practices**: The *PEM Security Best Practices* guide is available from our edb-docs website. It will help you to setup PEM in a secure way to minimize the risk of vulnerabilities. | Enhancement |
| You can use the Macros feature in the Query tool. | Enhancement |
| You can use the trigger function under the respective trigger node. | Enhancement |
| You can download the utility files at the client side using Storage Manager. | Enhancement | 
| You can use the open Query tool tab to change the database connection. | Enhancement |
| You can rename the Query tool and Debugger tab. | Enahncement |
| You can ignore the owner while comparing the objects through Schema diff tool. | Enhancement |
