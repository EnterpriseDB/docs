---
title: "Configuring a PEM Server on a Linux Host"
---

<div id="configuring_the_pem_server_on_linux_platforms" class="registered_link"></div>


After upgrading the PEM Server you can use the following command to configure the PEM Server:

> ``` text
> /usr/edb/pem/bin/configure-pem-server.sh
> ```

When invoking the configuration script, you can include command line options to specify configuration properties; the script will prompt you for values that you omit on the command line. The accepted options are:

<table><colgroup><col style="width: 5%" /><col style="width: 94%" /></colgroup><thead><tr class="header"><th>Option</th><th>Description</th></tr></thead><tbody><tr class="odd"><td><blockquote><p><code>-acp</code></p></blockquote></td><td>Defines PEM Agent certificate path. The default is /root/.pem.</td></tr><tr class="even"><td><blockquote><p><code>-ci</code></p></blockquote></td><td>CIDR formatted network address range that Agents will connect to the server from, to be added to the server's pg_hba.conf file. For example, 192.168.1.0/24. The default is 0.0.0.0/0.</td></tr><tr class="odd"><td><blockquote><p><code>-dbi</code></p></blockquote></td><td>The directory for the database server installation. For example, /usr/edb/as11 for Advanced Server or /usr/pgsql-11 for PostgreSQL.</td></tr><tr class="even"><td><blockquote><p><code>-ds</code></p></blockquote></td><td>The unit file name of the PEM database server. For Advanced Server, the default file name is edb-as-11; for PostgreSQL, it is postgresql-11.</td></tr><tr class="odd"><td><blockquote><p><code>-ho</code></p></blockquote></td><td>The host address of the PEM database server.</td></tr><tr class="even"><td><blockquote><p><code>-p</code></p></blockquote></td><td>The port number of the PEM database server.</td></tr><tr class="odd"><td><blockquote><p><code>-ps</code></p></blockquote></td><td>The service name of the pemagent; the default value is pemagent.</td></tr><tr class="even"><td><blockquote><p><code>-sp</code></p></blockquote></td><td>The superuser password of the PEM database server. This value is required.</td></tr><tr class="odd"><td><blockquote><p><code>-su</code></p></blockquote></td><td>The superuser name of the PEM database server.</td></tr><tr class="even"><td><blockquote><p><code>-t</code></p></blockquote></td><td>The installation type: Specify 1 if the configuration is for web services and backend database, 2 if you are configuring web services, or 3 if you are configuring the backend database. If you specify 3, please note that the database must reside on the local host.</td></tr><tr class="odd"><td><blockquote><p><code>-un</code></p></blockquote></td><td>To unregister the PEM server.</td></tr><tr class="even"><td><blockquote><p><code>-h</code></p></blockquote></td><td>Displays help.</td></tr></tbody></table>

If you do not provide configuration properties on the command line, you will be prompted for values by the script. To view script-related help, use the command:

``` text
/usr/edb/pem/bin/configure-pem-server.sh --help
```

After executing the PEM server configuration file, use your version-specific service control command to restart the `httpd` service.

For detailed information about using an RPM package to install or configure the PEM server or PEM Agent, please see the *PEM Linux Installation Guide*, available at:

> <https://www.enterprisedb.com/edb-docs/p/edb-postgres-enterprise-manager>
