title: "Release notes for EDB Postgres Distributed version 4.2.1"
navTitle: "Version 4.2.1"
---

EDB Postgres Distributed version 4.2.1 is a patch release of EDB Postgres Distributed 4, which includes bug fixes for issues identified in previous versions.

| Component | Version |  Type   |                                                                                                                                                                          Description                                                                                                                                     |
| --------- | ------- | ------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| BDR       | 4.2.1   | Bug fix | Grant `bdr.replication_status to `bdr_read_all_stats` (BDR-2523) <p>  Alllow the `read_all_stats` role to check BDR replication information. </p>                                                                                                                                                                             |
| BDR       | 4.2.1   | Bug fix | Resolve hang related to `bdr.standby_slots_min_confirmed` (BDR-2445, RT80692) <p> Allow the WAl sender to proceed without the need to wait for the timeout after a Subscriber Only node is parted, while using `standby_slot_names`. </p>                                                                                  |
| BDR       | 4.2.1   | Bug fix | Use two_phase slots in `bdr_init_physical` on PG14+ () <p> Improve prepared transaction handling during `bdr_init_physical`. </p>                                                                                                                                                                                        |
| BDR       | 4.2.1   | Bug fix | Add schema name to relation name in on all error messages (BDR-765)                                                                                                                                                                                                                                                      |
| BDR       | 4.2.1   | Bug fix | Allow parting a node only if the node didn't already start the part process (BDR-2629) <p> Return with an error if a node is already in at least PART_START state. </p>                                                                                                                                                  |
| BDR       | 4.2.1   | Bug fix | Don't wait for ADD CONSTRAINT progress if DDL replication is off (BDR-2645, RT86043) <p> The constraint validation from all nodes is not needed if DDL is not replicated, nor do PARTED or STANDBY nodes need to validate. </p>                                                                                          |
| BDR       | 4.2.1   | Bug fix | Adjust `replication_set_add_ddl_filter` (BDR-2755, RT86825) <p> Replicate all the columns needed for the ddl_filter. </p>                                                                                                                                                                                                |
| BDR       | 4.2.1   | Bug fix | Add maintanance to bdr.queue catalog table (BDR-2475, RT84105) <p> Automatically remove rows older than the `bdr.max_queue_days` (default disabled) on bdr.queue. </p>                                                                                                                                                   |
| BDR       | 4.2.1   | Bug fix | Add timeout option to `bdr_init_physical` (RT8277) <p> The timeout does not take the time spent in other activities of `bdr_init_physical` such as creating a base backup and sending the join request. </p>                                                                                                             |
| BDR       | 4.2.1   | Bug fix | Abort unresolved commit scope transactions on node part (BDR-2597) <p> Add a new peer state PART_TX_RESOLVE. This state will abort any remaining unresolved group commit transactions from the node that's parting on the current raft leader and replicates that to the rest of the cluster. </p>                       |
| HARP      | 2.2.1   | Bug fix | Fix connection leak issue (BDR-2530)                                                                                                                                                                                                                                                                                     |
