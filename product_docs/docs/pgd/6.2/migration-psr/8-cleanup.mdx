In this final phase, you verify that the new PGD cluster is independent and decommission the legacy replication infrastructure.

## Stopping replication from the source node

To confirm that data is no longer flowing through the source node, you must disable the logical subscriptions on all PGD nodes that connect back to the original PSR source.

On all PGD nodes, run:

```sql
SET bdr.ddl_replication = off;
DISABLE SUBSCRIPTION migration_seed_sub DISABLE;
```

Once you have verified that the source node is no longer receiving or forwarding transactions, you can safely drop these subscriptions to fully decouple the clusters.

## Cleaning up the seed node

After the migration is finalized and the cluster is stable, the seed node no longer requires the legacy version of Postgres. You can now remove the corresponding outdated packages from the node to free up resources and simplify future maintenance.
