Released: 14 Nov 2023
Updated: 18 Jan 2024

EDB Postgres Distributed version 4.3.3 is a patch release of EDB Postgres Distributed 4, which includes bug fixes for issues identified in previous versions.

!!! Note
This version is required for EDB Postgres Advanced Server versions 12.15, 13.11, 14.10, 15.5 and later.
!!!

| Component | Version |  Type        |                                                   Description                                                           |
| --------- | ------- | ------------ | ------------------------------------------------------------------------------------------------------------------------|
| BDR       | 4.3.3   | Bug&nbsp;fix | Ensure that the WalSender process does not request locks on the partitions, thus avoiding a deadlock with user process waiting on sync commit. (RT97952) |
| BDR       | 4.3.3   | Bug&nbsp;fix | Only consider CAMO transactions to be asynchronous when the CAMO setup was degraded to local mode. This solves the split brain problem when deciding fate of transactions that happened during failover. (RT78928) |
| BDR       | 4.3.3   | Bug&nbsp;fix | Handle partitions with different attribute numbers when batch inserting. (RT99115) |
| BDR       | 4.3.3   | Bug&nbsp;fix | Fix unsafe CAMO decisions in remote_write mode. |
| BDR       | 4.3.3   | Bug&nbsp;fix | Ensure that the taskmgr process does not ignore SIGINT. |
| BDR       | 4.3.3   | Bug&nbsp;fix | Clean up stale records in bdr.taskmgr_local_work_queue during physical join. |
| BDR       | 4.3.3   | Bug&nbsp;fix | Fix a bug in copying bdr.autopartition_rules during logical join. |
| BDR       | 4.3.3   | Bug&nbsp;fix | Override bdr.ddl_replication=off in taskmgr worker. |
| BDR       | 4.3.3   | Bug&nbsp;fix | Avoid a potential deadlock between `bdr.autopartition_wait_for_partitions()` and taskmgr|
| BDR       | 4.3.3   | Bug&nbsp;fix | Block new EPAS automatic partitioning on PGD cluster. |
| BDR       | 4.3.3   | Bug&nbsp;fix | Allow existing automatically partitioned tables when cluster is created or upgraded. |
| BDR       | 4.3.3   | Bug&nbsp;fix | Block PGD Autopartition on EPAS INTERVAL partitioned table. |
| BDR       | 4.3.3   | Bug&nbsp;fix | Fix memory leak in streaming transaction processing. |
| BDR       | 4.3.3   | Bug&nbsp;fix | Allow force parting of nodes that are already being parted normally. |
| BDR       | 4.3.3   | Bug&nbsp;fix | Fix nodes getting stuck on PART_CATCHUP due to replication slot conflict. |
| BDR       | 4.3.3   | Bug&nbsp;fix | Fix nodes getting stuck on PART_CATCHUP due to node_catchup_info conflict. |
| BDR       | 4.3.3   | Bug&nbsp;fix | Fix row filter failure for partitions created after a table column was dropped (RT95149) |
| BDR       | 4.3.3   | Bug&nbsp;fix | Ensure that replication receiver worker reloads configuration correctly when `pg_reload_conf()` is called. |
| BDR       | 4.3.3   | Bug&nbsp;fix | Prevent duplicate Raft request ID generation which could break replication. |
| BDR       | 4.3.3   | Bug&nbsp;fix | Fix several rare memory access issues that could potentially cause crashes of workers. |
| CLI       | 1.1.2   | Bug&nbsp;fix | Fixed `connect_timeout` issue when `sslmode=allow` or `sslmode=prefer` by upgrading database driver library version. |
| Utilities | 1.2.0   | Bug&nbsp;fix | Remove the pre-upgrade step which sets the CONNECTION LIMIT to 0 at database level in `bdr_pg_upgrade` |
