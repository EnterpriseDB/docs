Switchover to new write leader.

### Synopsis

Switchover to new write leader. Use switchover method fast for immediate
switchover, or strict to wait until lag is less than route_writer_max_lag
on given target node. If switchover fails due to timeout or any other
issue, BDR may elect a write leader which is different than given target
node.

```sh
pgd switchover [flags]
```

### Examples

```text

  Example 1 (removes the current write leader and waits until given node has
  lag less than route_writer_max_lag, after that makes it a new write leader)

  $ pgd switchover --group-name group_a --node-name bdr-a1 --method strict --timeout 15s
  switchover is complete


  Example 2 (with required arguments, default method is strict and default timeout is 10s)

  $ pgd switchover --group-name group_a --node-name bdr-a1
  switchover is complete


  Example 3 (immediate switchover, route_writer_max_lag is ignored)

  $ pgd switchover --group-name group_a --node-name bdr-a1 --method fast
  switchover is complete

```

### Options

```text
      --group-name string   group name
  -h, --help                help for switchover
      --method string       switchover method (strict, fast)
                            strict - waits until lag on given node is less than route_writer_max_lag
                            fast - immediate switchover, route_writer_max_lag is ignored (default "strict")
      --node-name string    node name
      --timeout duration    timeout period when switchover method is strict (default 10s)
```
