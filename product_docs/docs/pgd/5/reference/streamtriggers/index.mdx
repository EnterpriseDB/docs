Both [conflict triggers](../../striggers/#conflict-triggers) and [transform triggers](../../striggers/#transform-triggers) have access to information about
rows and metadata by way of the predefined variables provided by the trigger API and
additional information functions provided by PGD.

In PL/pgSQL, you can use the predefined variables and functions that follow:

* [Row variables](rowvariables)
* [Row Information functions](rowfunctions)
  * [`bdr.trigger_get_row`](rowfunctions/#bdrtrigger_get_row)
  * [`bdr.trigger_get_committs`](rowfunctions/#bdrtrigger_get_committs)
  * [`bdr.trigger_get_xid`](rowfunctions/#bdrtrigger_get_xid)
  * [`bdr.trigger_get_type`](rowfunctions/#bdrtrigger_get_type)
  * [`bdr.trigger_get_conflict_type`](rowfunctions/#bdrtrigger_get_conflict_type)
  * [`bdr.trigger_get_origin_node_id`](rowfunctions/#bdrtrigger_get_get_origin_node_id)
  * [`bdr.ri_fkey_on_del_trigger`](rowfunctions/#bdrri_fkey_on_del_trigger)

Creating and dropping stream triggers is managed through the manipulation interfaces:

* [Manipulation interfaces](interfaces)
  * [`bdr.create_conflict_trigger`](interfaces/#bdrcreate_conflict_trigger)
  * [`bdr.create_transform_trigger`](interfaces/#bdrcreate_transform_trigger)
  * [`bdr.drop_trigger`](interfaces/#bdrdrop_trigger)
