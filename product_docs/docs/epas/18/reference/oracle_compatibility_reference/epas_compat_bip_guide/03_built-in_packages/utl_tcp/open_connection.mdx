---
title: "OPEN_CONNECTION"
---

The `OPEN_CONNECTION` function establishes a network connection to a specified host and port, returning a `connection` handle used for subsequent data transfer operations.

```sql
OPEN_CONNECTION (
  remote_host      IN VARCHAR2(255) DEFAULT NULL,
  remote_port      IN INTEGER,
  local_host       IN VARCHAR2(255) DEFAULT NULL,
  local_port       IN INTEGER      DEFAULT NULL,
  in_buffer_size   IN INTEGER      DEFAULT NULL,
  out_buffer_size  IN INTEGER      DEFAULT NULL,
  charset          IN VARCHAR2(30)  DEFAULT NULL,
  newline          IN VARCHAR2(2)   DEFAULT NULL,
  tx_timeout       IN INTEGER      DEFAULT NULL,
  wallet_path      IN VARCHAR2(255) DEFAULT NULL,
  wallet_password  IN VARCHAR2(255) DEFAULT NULL,
  ssl_cert_path    IN VARCHAR2(255) DEFAULT NULL
) RETURN connection;

```

## Parameters

`remote_host` 

 The DNS name or IP address of the remote host to which you are connecting.

`remote_port`
 
 The TCP port number on the remote host.

`local_host`
 
 The local network interface address to bind to. If `NULL`, the system chooses the default interface.

`local_port`

 The local TCP port number to bind to. If `NULL`, an ephemeral port is assigned.

`in_buffer_size`

 The size (in bytes) of the input buffer used for receiving data.

`out_buffer_size`

 The size (in bytes) of the output buffer used for sending data.

`charset`

 The character set to be used for data transmission (e.g., 'UTF-8').

`newline`

 The character sequence used to denote a new line (e.g., `CRLF` or `LF`).

`tx_timeout`
 
 The timeout duration (in seconds) for transmission operations.

`wallet_path`

 The file system path to the security wallet if using SSL/TLS.

`wallet_password`
 
 The password required to access the specified security wallet.

`ssl_cert_path`

 The path to a specific SSL certificate file, used as an alternative or supplement to a wallet.

## Return Value

| Type | Description |
| ---- | ----------- |
| `connection` | A record handle representing the active network session. This handle must be passed to subsequent read/write or close procedures. |

## Notes
    
- **SSL/TLS Configuration** &mdash; If `wallet_path` or `ssl_cert_path` are provided, the function attempts to negotiate a secure connection. Ensure the certificate paths are accessible by the database process.

- **Defaults** &mdash; Most parameters are optional. At a minimum, `remote_port` must be specified if `remote_host` is handled by the underlying environment or defaults.

- **Resource Management** &mdash; Every connection opened with this function should eventually be closed using the corresponding `CLOSE_CONNECTION` procedure to prevent resource leaks.

- **Buffer Tuning** &mdash; For high-throughput applications, adjusting `in_buffer_size` and `out_buffer_size` can significantly impact performance based on the network MTU.

## Example

```sql
DECLARE
 c      utl_tcp.connection;
BEGIN   
  c := utl_tcp.open_connection('mail-arts.enterprisedb.com', 25);
  utl_tcp.close_connection(c);
end;
```
