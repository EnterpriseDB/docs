---
title: "CLOSE_ALL_CONNECTIONS"
---

The `CLOSE_ALL_CONNECTIONS` procedure terminates all active network sessions currently managed by the session and releases their associated resources.

```sql
CLOSE_ALL_CONNECTIONS();
```

## Notes

- **Bulk Cleanup** &mdash; This procedure is an efficient way to ensure no "orphan" connections remain open at the end of a process or in the event of an error recovery sequence.

- **Scope** &mdash; Only connections opened within the current session are affected. It will not close connections established by other database sessions.

- **Handle Invalidation** &mdash; After calling this procedure, any existing connection handles previously returned by `OPEN_CONNECTION` become invalid. Attempting to use them for read or write operations will result in an error.

- **Propagation Delay** &mdash; As with other probe and connection modifications, keep in mind that underlying system state changes may take a few moments to fully synchronize.

## Example

```sql
BEGIN   
  utl_tcp.close_all_connections();
end;
```
