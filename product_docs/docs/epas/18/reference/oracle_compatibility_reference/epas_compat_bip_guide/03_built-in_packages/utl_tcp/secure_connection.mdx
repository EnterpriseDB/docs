---
title: "SECURE_CONNECTION"
---

The `SECURE_CONNECTION` procedure upgrades an existing, open connection to use SSL/TLS encryption. This is typically used to initiate "StartTLS" style security on an unencrypted session.

```sql
SECURE_CONNECTION (
  c IN OUT connection
);
```

## Parameters

`c`

The connection handle of the active session, originally returned by OPEN_CONNECTION. Because this is an IN OUT parameter, the internal state of the connection handle is updated to reflect the new secure status.

## Notes

- **Prerequisites:** To successfully negotiate a secure handshake, the connection must have been initialized with valid wallet_path and wallet_password (or ssl_cert_path) parameters during the OPEN_CONNECTION call.

- **Handshake Timing:** This procedure performs the SSL/TLS handshake. If the remote host does not support encryption or the certificates are invalid, the procedure will raise an exception.

- **Operational Flow:** This is commonly used in protocols like SMTP or LDAP where the initial connection is plaintext, and the session is upgraded to a secure channel before sensitive data (like credentials) is transmitted.

- **Impact of Timeout:** The handshake is subject to the tx_timeout value defined at the start of the connection.
