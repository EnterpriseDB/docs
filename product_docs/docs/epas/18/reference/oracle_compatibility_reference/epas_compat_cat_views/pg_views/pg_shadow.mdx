---
title: "PG_SHADOW"
---

The `PG_SHADOW` view provides information about properties of all roles that are marked as `rolcanlogin` in `PG_AUTID`. It provides information specific to EDB Postgres Advanced Server in addition to the information provided in [PostgreSQL pg_shadow view](https://www.postgresql.org/docs/current/view-pg-shadow.html).

| Name            | Type                     | Description                                                |
|-----------------|--------------------------|------------------------------------------------------------|
| useredwoodprivs | integer                  | Specified whether the role has redwood specific privileges |
| uselastlogin    | timestamp with time zone | Specifies the time of the last user login.                 |

## Functions to check redwood specific privileges

Here are the functions to check whether a role has redwood specific privileges:

### has_createdblink_privilege(name)

This function checks if the **current user** has the necessary privilege to create a private database link. It takes the intended name of the database link as an argument.

- **Input:** The name of the database link to be created (as a string).
- **Outcome:** Returns `true` if the current user has `CREATE DATABASE LINK` privilege; otherwise, it returns `false`.

### has_createdblink_privilege(oid)

This function checks if the **current user** has the necessary privilege to create a private database link. It takes the intended Object Identifier (OID) of the database link as an argument.

- **Input:** The OID for the database link.
- **Outcome:** Returns `true` if the current user has `CREATE DATABASE LINK` privilege; otherwise, it returns `false`.

### has_createpublicdblink_privilege(name)

This function checks if the **current user** has the privilege to create a **public** database link, which is a link accessible to all users. It takes the intended name of the database link as an argument.

- **Input:** The name of the public database link (as a string).
- **Outcome:** Returns `true` if the current user has `CREATE PUBLIC DATABASE LINK` privilege; otherwise, it returns `false`.

### has_createpublicdblink_privilege(oid)

This function checks if the **current user** has the privilege to create a **public** database link. It takes the intended Object Identifier (OID) of the database link as an argument.

- **Input:** The OID for the public database link.
- **Outcome:** Returns `true` if the current user has `CREATE PUBLIC DATABASE LINK` privilege; otherwise, it returns `false`.

### has_droppublicdblink_privilege(name)

This function checks if the **current user** has the necessary privilege to drop a **public** database link. It takes the name of the link as an argument.

- **Input:** The name of the public database link to be dropped (as a string).
- **Outcome:** Returns `true` if the current user has `DROP PUBLIC DATABASE LINK` privilege; otherwise, it returns `false`.

### has_droppublicdblink_privilege(oid)

This function checks if the **current user** has the necessary privilege to drop a **public** database link. It takes the Object Identifier (OID) of the link as an argument.

- **Input:** The OID of the public database link.
- **Outcome:** Returns `true` if the current user has `DROP PUBLIC DATABASE LINK` privilege; otherwise, it returns `false`.


### has_exemptaccesspolicy_privilege(name)

This function checks if the **current user** has the `EXEMPT ACCESS POLICY` privilege for a specific role. This privilege allows the user to bypass Row-Level Security (RLS) policies.

- **Input:** The name of the role (as a string).
- **Outcome:** Returns `true` if the current user has the `EXEMPT ACCESS POLICY` privilege for the specified role; otherwise, it returns `false`.

### has_exemptaccesspolicy_privilege(oid)

This function checks if the **current user** has the `EXEMPT ACCESS POLICY` privilege for a specific role. This privilege allows the user to bypass Row-Level Security (RLS) policies. It takes the Object Identifier (OID) of the role as an argument.

- **Input:** The OID of the role.
- **Outcome:** Returns `true` if the current user has the `EXEMPT ACCESS POLICY` privilege for the specified role; otherwise, it returns `false`.
