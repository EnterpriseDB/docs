Released: 12 Feb 2026

!!! note "Why is RHEL 9.6 not supported for EPAS 18?"

    - *Version Pivot:* Due to OpenSSL and LLVM breakages introduced in RHEL 9.7, our current minor releases for back-branches (<= 17) were built on RHEL 9.6 to maintain stability for existing users.
    - *EPAS 18 Requirement:* Moving forward, EPAS 18 is specifically designed for RHEL 9.7+. It relies on the newer toolsets available in 9.7 that are incompatible with the 9.6 environment.

EDB Postgres Advanced Server 18.2.0 includes the following enhancements and bug fixes:

| Type           | Description                                                                                                                                                                                                                                    | Addresses&nbsp; |
|----------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|-----------------|
| Upstream merge | Merged with community PostgreSQL 18.2. See the [PostgreSQL 18.2 Release Notes](https://www.postgresql.org/docs/release/18.2/) for more information.                                                                                            |                 |
| Bug&nbsp;fix   | Fixed an issue where memory was allocated in an inappropriate context during parallel execution, leading to rare crashes when the system released data still in use by active workers; memory is now correctly scoped to a long-lived context. | #52519          |
| Bug&nbsp;fix   | Fixed an issue with stack overflow in `edb_partition` due to circular references during partition creation.                                                                                                                                    |                 |
