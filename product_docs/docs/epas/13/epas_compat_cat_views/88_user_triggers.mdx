title: "USER_TRIGGERS"

<div id="user_triggers" class="registered_link"></div>

The `USER_TRIGGERS` view displays information about all triggers on tables owned by the current user.

<table><tbody><tr class="odd"><td>Name</td><td>Type</td><td>Description</td></tr><tr class="even"><td>schema_name</td><td>TEXT</td><td>The name of the schema in which the trigger resides.</td></tr><tr class="odd"><td>trigger_name</td><td>TEXT</td><td>The name of the trigger.</td></tr><tr class="even"><td><p>trigger_type</p></td><td><p>TEXT</p></td><td><p>The type of the trigger. Possible values are:</p><blockquote><p>BEFORE ROW</p><p>BEFORE STATEMENT</p><p>AFTER ROW</p><p>AFTER STATEMENT</p></blockquote></td></tr><tr class="odd"><td>triggering_event</td><td>TEXT</td><td>The event that fires the trigger.</td></tr><tr class="even"><td>table_owner</td><td>TEXT</td><td>The user name of the owner of the table on which the trigger is defined.</td></tr><tr class="odd"><td>base_object_type</td><td>TEXT</td><td>Included for compatibility only. Value will always be TABLE.</td></tr><tr class="even"><td>table_name</td><td>TEXT</td><td>The name of the table on which the trigger is defined.</td></tr><tr class="odd"><td>referencing_names</td><td>TEXT</td><td>Included for compatibility only. Value will always be REFERENCING NEW AS NEW OLD AS OLD.</td></tr><tr class="even"><td>status</td><td>TEXT</td><td>Status indicates if the trigger is enabled (VALID) or disabled (NOTVALID).</td></tr><tr class="odd"><td>description</td><td>TEXT</td><td>Included for compatibility only.</td></tr><tr class="even"><td>trigger_body</td><td>TEXT</td><td>The body of the trigger.</td></tr><tr class="odd"><td>action_statement</td><td>TEXT</td><td>The SQL command that executes when the trigger fires.</td></tr></tbody></table>
