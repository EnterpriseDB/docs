title: "DBMS_SCHEDULER"
---

The `DBMS_SCHEDULER` package provides a way to create and manage Oracle-styled jobs, programs and job schedules. The `DBMS_SCHEDULER` package implements the following functions and procedures:

|                                                                                                                                                                                                                              |             |                                                                                                                                 |
| ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------- | ------------------------------------------------------------------------------------------------------------------------------- |
| Function/Procedure                                                                                                                                                                                                           | Return Type | Description                                                                                                                     |
| CREATE_JOB(\\&lt;job_name>, \\&lt;job_type>, \\&lt;job_action>, \\&lt;number_of_arguments>, \\&lt;start_date>, \\&lt;repeat_interval>, \\&lt;end_date>, \\&lt;job_class>, \\&lt;enabled>, \\&lt;auto_drop>, \\&lt;comments>) | n/a         | Use the first form of the CREATE_JOB procedure to create a job, specifying program and schedule details by means of parameters. |
| CREATE_JOB(\\&lt;job_name>, \\&lt;program_name>, \\&lt;schedule_name>, \\&lt;job_class>, \\&lt;enabled>, \\&lt;auto_drop>, \\&lt;comments>)                                                                                  | n/a         | Use the second form of CREATE_JOB to create a job that uses a named program and named schedule.                                 |
| CREATE_PROGRAM(\\&lt;program_name>, \\&lt;program_type>, \\&lt;program_action>, \\&lt;number_of_arguments>, \\&lt;enabled>, \\&lt;comments>)                                                                                 | n/a         | Use CREATE_PROGRAM to create a program.                                                                                         |
| CREATE_SCHEDULE( \\&lt;schedule_name>, \\&lt;start_date>, \\&lt;repeat_interval>, \\&lt;end_date>, \\&lt;comments>)                                                                                                          | n/a         | Use the CREATE_SCHEDULE procedure to create a schedule.                                                                         |
| DEFINE_PROGRAM_ARGUMENT( \\&lt;program_name>, \\&lt;argument_position>, \\&lt;argument_name>, \\&lt;argument_type>, \\&lt;default_value>, \\&lt;Mout_argument>)                                                              | n/a         | Use the first form of the DEFINE_PROGRAM_ARGUMENT procedure to define a program argument that has a default value.              |
| DEFINE_PROGRAM_ARGUMENT( \\&lt;program_name>, \\&lt;argument_position>, \\&lt;argument_name>, \\&lt;argument_type>, \\&lt;out_argument>)                                                                                     | n/a         | Use the first form of the DEFINE_PROGRAM_ARGUMENT procedure to define a program argument that does not have a default value.    |
| DISABLE(\\&lt;name>, \\&lt;force>, \\&lt;commit_semantics>)                                                                                                                                                                  | n/a         | Use the DISABLE procedure to disable a job or program.                                                                          |
| DROP_JOB(\\&lt;job_name>, \\&lt;force>, \\&lt;defer>, \\&lt;commit_semantics>)                                                                                                                                               | n/a         | Use the DROP_JOB procedure to drop a job.                                                                                       |
| DROP_PROGRAM(\\&lt;program_name>, \\&lt;force>)                                                                                                                                                                              | n/a         | Use the DROP_PROGRAM procedure to drop a program.                                                                               |
| DROP_PROGRAM_ARGUMENT( \\&lt;program_name>, \\&lt;argument_position>)                                                                                                                                                        | n/a         | Use the first form of DROP_PROGRAM_ARGUMENT to drop a program argument by specifying the argument position.                     |
| DROP_PROGRAM_ARGUMENT( \\&lt;program_name>, \\&lt;argument_name>)                                                                                                                                                            | n/a         | Use the second form of DROP_PROGRAM_ARGUMENT to drop a program argument by specifying the argument name.                        |
| DROP_SCHEDULE(\\&lt;schedule_name>, \\&lt;force>)                                                                                                                                                                            | n/a         | Use the DROP SCHEDULE procedure to drop a schedule.                                                                             |
| ENABLE(\\&lt;name>, \\&lt;commit_semantics>)                                                                                                                                                                                 | n/a         | Use the ENABLE command to enable a program or job.                                                                              |
| EVALUATE_CALENDAR_STRING( \\&lt;calendar_string>, \\&lt;start_date>, \\&lt;return_date_after>, \\&lt;next_run_date>)                                                                                                         | n/a         | Use EVALUATE_CALENDAR_STRING to review the execution date described by a user-defined calendar schedule.                        |
| RUN_JOB(\\&lt;job_name>, \\&lt;use_current_session>, \\&lt;manually>)                                                                                                                                                        | n/a         | Use the RUN_JOB procedure to execute a job immediately.                                                                         |
| SET_JOB_ARGUMENT_VALUE( \\&lt;job_name>, \\&lt;argument_position>, \\&lt;argument_value>)                                                                                                                                    | n/a         | Use the first form of SET_JOB_ARGUMENT value to set the value of a job argument described by the argument's position.           |
| SET_JOB_ARGUMENT_VALUE( \\&lt;job_name>, \\&lt;argument_name>, \\&lt;argument_value>)                                                                                                                                        | n/a         | Use the second form of SET_JOB_ARGUMENT value to set the value of a job argument described by the argument's name.              |

Advanced Server's implementation of `DBMS_SCHEDULER` is a partial implementation when compared to Oracle's version. Only those functions and procedures listed in the table above are supported.

The `DBMS_SCHEDULER` package is dependent on the pgAgent service; you must have a pgAgent service installed and running on your server before using `DBMS_SCHEDULER`.

Before using `DBMS_SCHEDULER`, a database superuser must create the catalog tables in which the `DBMS_SCHEDULER` programs, schedules and jobs are stored. Use the `psql` client to connect to the database, and invoke the command:

> `CREATE EXTENSION dbms_scheduler`;

By default, the `dbms_scheduler` extension resides in the `contrib/dbms_scheduler_ext` subdirectory (under the Advanced Server installation).

Note that after creating the `DBMS_SCHEDULER` tables, only a superuser will be able to perform a dump or reload of the database.
