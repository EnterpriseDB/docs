PGAA tables store their data in object storage, not in the WarehousePG cluster. The [gpbackup utility](/warehousepg/latest/admin_guide/backup_restore/), provided by the `whpg-backup` package, supports mixed databases containing both PGAA and regular heap tables. For PGAA tables, gpbackup captures only metadata (table mappings, storage locations, and path bindings), not the underlying data in object storage. This metadata is automatically restored by gprestore, so PGAA tables are immediately queryable after restore without manual reconfiguration.

Protecting the underlying data in object storage is the responsibility of your object storage provider.

## Prerequisites

`whpg-backup` version 1.33 or later installed in your WarehousePG cluster.

## Backing up

Run gpbackup on the coordinator:

```bash
gpbackup --dbname mydb --backup-dir /path/to/backup-dir
```

For a full list of options, see the [gpbackup](/warehousepg/latest/ref_guide/utility_guide/reference/gpbackup/) reference page.

Note the backup timestamp from the output. You will need it for the restore step. For example:

```
[INFO]:-Backup Timestamp = 20260417100433
```

## Restoring

Use `--redirect-db` to restore PGAA tables into a new, empty database. When restoring into an existing database that already has PGAA tables, a table is skipped if its metadata already exists, resulting in an empty table.

1. Create the target database and create the PGAA and PGFS extensions:

    ```bash
    createdb mydb_restored --template=template0
    psql -d mydb_restored -c "CREATE EXTENSION pgaa CASCADE;"
    ```

1. Run gprestore:

    ```bash
    gprestore --backup-dir /path/to/backup-dir --timestamp <backup-timestamp> \
      --redirect-db mydb_restored --on-error-continue
    ```

    For a full list of options, see the [gprestore](/warehousepg/latest/ref_guide/utility_guide/reference/gprestore/) reference page. 

    !!! Note
    The `--create-db` gprestore option isn't supported with PGAA.
    !!!

1. Confirm the restore succeeded by checking `Tables restored` in the gprestore output.

1. Verify that storage locations are restored and PGAA tables are immediately queryable against object storage:

    ```sql
    SELECT * FROM pgfs.list_storage_locations();
    SELECT count(*) FROM my_table;
    ```
