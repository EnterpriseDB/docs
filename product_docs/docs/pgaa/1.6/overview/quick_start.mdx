Quickstart:

- Prerequisites
- Installation: CREATE EXTENSION pgaa CASCADE
- Auto-starting Seafowl (pgaa.autostart_seafowl)
- First query: reading from a public sample bucket
- Verifying setup


In this quick start, you will:

1. Install the PGAA extension on your Postgres instance.
1. Create a storage location that points to our sample benchmarking datasets in object storage (TPC-H, TPC-DS, Clickbench, or 1BRC).
1. Run analytical queries.



## Unified OLTP + OLAP Architecture

Combine operational and analytical workloads in a single platform:

Hot data in PostgreSQL

```sql
CREATE TABLE sales_current (
  sale_id BIGINT PRIMARY KEY,
  sale_date DATE,
  amount DECIMAL(10,2)
);
```

Cold data in lakehouse:

```sql
CREATE TABLE sales_archive () USING PGAA
WITH (
  pgaa.format = 'iceberg',
  pgaa.managed_by = 'lakehouse_catalog',
  pgaa.catalog_table = 'sales_historical'
);
```

Unified view across tiers

```sql
CREATE VIEW sales_complete AS
SELECT * FROM sales_current
UNION ALL
SELECT * FROM sales_archive;
```
