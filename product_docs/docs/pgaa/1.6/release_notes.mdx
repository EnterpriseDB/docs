Released: 6 February 2026

Analytics Accelerator 1.6 includes the following enhancements and bug fixes:

## Features

- Enable experimental join pushdown in CompatScan (pgaa.enable_join_pushdown GUC to disable)
- Enable experimental groupby pushdown in CompatScan (pgaa.enable_groupby_pushdown GUC to disable)
- (BEAC-837, BEAC-649, BEAC-901) Added catalog connection checks to pgaa.add_catalog() and pgaa.update_catalog() functions.
- (BEAC-710) Allow pgaa.tiered_table and pgaa.purge_data_if_exists to be specified without an explicit value, defaulting to true when present.
- (BEAC-906) Wired-up pgd.purge_analytics_target in pgaa.enable_analytics_replication and pgaa.convert_to_tiered_table
- (BEAC-959) Add support for BYTEA replication and inference
- (BEAC-972) Add replication fallback to text for unsupported types
- (BEAC-973) Add support for replicating PG array types to native lakehouse types
- (BEAC-976, BEAC-977, BEAC-1008) Add support for format: 'parquet' PGFS tables with file:// and s3:// protocols
- (BEAC-525) Add pgaa.list_catalog_tables() SQL function that allows users to list all tables and views in a catalog, optionally filtering by namespace.
- (BEAC-526) Add cascade parameter to pgaa.detach_catalog() and pgaa.delete_catalog() to optionally drop all catalog-managed tables; add pgaa.drop_catalog_tables() function.
- BREAKING: pgaa.delete_catalog() now requires explicit cascade := true when tables exist, preventing accidental data loss.
- (BEAC-626) Add support for NOT LIKE and NOT ILIKE operators
- (BEAC-1002) Use native PG array types when inferring Delta/Iceberg schemas with array types
- (BEAC-1005) Add support for replicating BPCHAR and UUID PG array types
- (BEAC-528) Add namespace filtering to pgaa.import_catalog() SQL functions, allowing users to selectively import from specific namespaces within a catalog.
- (BEAC-934) Update DataFusion to 51

## Deprecations

- Consolidate migrations to remove upgrade paths from PGAA versions before 1.3. Users of earlier versions should first upgrade to PGAA 1.3.1 and then to 1.6.0.
- Deprecate the pgaa.experimental_enable_spark_parquet_tables GUC: defaults to TRUE since both Seafowl and Spark executor engines support Parquet
