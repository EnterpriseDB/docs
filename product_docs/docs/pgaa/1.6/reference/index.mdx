The reference section provides detailed technical specifications for the Postgres Analytics Accelerator (PGAA). Use these resources to fine-tune your configuration, understand supported data types, and manage your analytical tables.

For information on the PGFS functions for managing storage location, see the [AI Factory PGFS documentation](/edb-postgres-ai/latest/ai-factory/pipeline/pgfs/).

* [Configuration parameters](configuration_parameters): The behavior of the PGAA extension is governed by Grand Unified Configuration (GUC) variables. These parameters allow you to switch executors, enable performance optimizations, and manage security credentials.

* [Functions](functions): PGAA introduces a suite of SQL functions for administrative tasks, such as mapping new tables, monitoring storage health, and launching maintenance background jobs.

* [Table options](table_options): When mapping or creating analytical tables, specific options allow you to define how data is read from or written to your object store.

* [Data types](data_types): PGAA maps native PostgreSQL data types to optimized columnar formats in the data lake.

* [Datasets](datasets): Access pre-configured schemas and data loading instructions for analytical datasets to baseline your performance.
