title: Creating a machine user in Hybrid Manager
navTitle: Create a machine user
description: Create a machine user and access key in Hybrid Manager so the EDB Postgres AI agent can authenticate and ingest data into your estate.
---

The EDB Postgres AI agent authenticates with Hybrid Manager (HM) using a machine user account and access key. You need to create this user before configuring the agent, whether you're setting up monitoring or migration assessment.

## Creating a machine user

1.  Select your profile icon in the upper-right corner.
1.  Select **User Management > Add New User**.
1.  Enter a name for the new user.
1.  Optionally, enter an email address.
1.  Select **Create Access Key**. Enter an **Access Key Name** and an **Expiry** time.
1.  Select **Add User**.
1.  Store the access key safely — you won't be able to retrieve it again. Select **Key stored safely**.

## Assigning the estate ingester role

1.  Select the **Projects** tab and select the project from which you want to monitor or assess the database.
1.  Select **Users** in the left navigation and locate the machine user you just created.
1.  Select the edit button (pen icon) for the user entry.
1.  Select **Assign Roles**.
1.  On the **Assign Project Roles** window, select **estate ingester**.
1.  Select **Submit**.

Store the access key in an environment variable on the machine running the agent so the agent can access it at runtime:

```shell
export BEACON_AGENT_ACCESS_KEY="baak_your_key_here"
```
