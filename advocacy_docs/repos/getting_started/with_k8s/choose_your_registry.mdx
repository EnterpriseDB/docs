title: Choosing the correct EDB container registry
navTitle: Choose a registry
description: Overview of available EDB container registries and their purposes.
---

EDB currently provides the following container registries (repositories of container images used to provision Kubernetes / container orchestration systems):

- **pgai-platform** - `docker.enterprisedb.com/pgai-platform`: use this registry when deploying [EDB Postgres AI Hybrid Manager](https://www.enterprisedb.com/docs/edb-postgres-ai/latest/hybrid-manager/) (HM). 

- **k8s** - `docker.enterprisedb.com/k8s`: use this registry when deploying the [EDB Postgres® AI for CloudNativePG™ Cluster](https://www.enterprisedb.com/docs/postgres_for_kubernetes/latest/) or [EDB Postgres® AI for CloudNativePG™ Global Cluster](https://www.enterprisedb.com/docs/postgres_distributed_for_kubernetes/latest/) operators. Compatible operand images are also available through this repo. 

The images available on each registry depend on your subscription level as represented by your [token](../get_your_token.mdx).

!!! Note 
    If you're deploying both HM _and_ stand-alone CNPG-* systems, you'll need to configure your platform for both registries and use the correct path and username for the operator that you're deploying. Each registry hosts independently versioned images and each operator requires access to the correct registry to function properly.

!!! Note Deprecated registries
    In the past, EDB provided several additional registries, used to separate different applications and subscription levels. For details, see [Migrating to the unified private EDB container registry](../../working_with_repos/migrating_edb_registries.mdx).

!!! SeeAlso Next steps

    Once you've identified the correct registry for your deployment, you can [deploy operators](deploy_operators).
