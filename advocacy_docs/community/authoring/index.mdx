Below are detailed steps to get started with this repository. Instructions are optimized for macOS.
- Items preceded with $ indicate the Terminal command you should input

## How to Install an IDE and Git
1. Download VS Code from https://code.visualstudio.com/download (Skip this step if you already have an IDE)
2. Install [Homebrew](https://brew.sh/)
    Go to your Mac Terminal and enter the following command
    $ `/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"`
    This will take a few minutes
3. Install [git](https://git-scm.com/) from Terminal
    $ `brew install git`

## How to Install the Repository and dependencies
4. Clone the docs repo into your local environment
    $ `git clone https://github.com/EnterpriseDB/docs.git`
5. Add the new repository folder to VS Code
    Go to VS Code > File menu > Add Folder to Workspace > browse to “docs”
    Click on > docs to expand it within VS Code
6. Install Node.js 12 from the [Node.js website](https://nodejs.org/en/download/)
7. Go to the docs directory in Terminal 
    $ `cd docs`
8. Install node 
    $ `brew install node`
9. Install python
    $ `brew install python`
10. Install yarn and gatsby
    $ `npm i -g gatsby-cli` 
    $ `npm i -g yarn`
11. Install required packages with yarn
    $ `yarn`
12. Pull down shared icon files
    $ `git submodule update --init`
13. Select sources with yarn
    $ `yarn config-sources`
    You'll be prompted to choose the packages you'd like to access; you can use comma separated values to choose more than one
    $ `yarn develop`
14. Go to the localhost port in your browser e.g. http://localhost:8000/ (your url will be mentioned in Terminal)

Tips to Use Terminal
- `cmd t` opens a new tab
- `ctrl c` closes a process

## How to Make changes and Submit Pull Requests
15. Make changes in VS Code
16. We recommend using Github Desktop unless you're familiar with git
17. To submit a pull request
    - Make changes to the repository in VS code (or your IDE of choice)
    - Save changes `ctrl s` 
    - Create a new branch and name it
    - Follow the steps in Github Desktop, most likely starting with "Commit to <branch-name>"
