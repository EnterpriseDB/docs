To benefit from advanced replication features, you must install and configure [EDB Postgres Distributed](/pgd/latest) with your Postgres database instance.

|                                                 | PostgreSQL | EDB Postgres Extended Server | EDB Postgres Advanced Server | EDB Postgres Advanced Server in Oracle compatibility mode |
|-------------------------------------------------|------------|------------------------------|------------------------------|-----------------------------------------------------------|
| Commit at most once                             | No         | Yes                          | Yes                          | Yes                                                       |
| Group commit                                    | No         | Yes                          | Yes                          | Yes                                                       |
| Eager replication                               | No         | Yes                          | Yes                          | Yes                                                       |
| Decoding worker                                 | No         | Yes                          | Yes                          | Yes                                                       |
| Assessment tooling                              | No         | Yes                          | Yes                          | Yes                                                       |
| Tracker                                         | No         | Yes                          | Yes                          | Yes                                                       |
| Lag control                                     | No         | Yes                          | Yes                          | Yes                                                       |
| Timestamp snapshots                             | No         | Yes                          | Yes                          | Yes                                                       |
| Transaction streaming                           | Yes        | Yes                          | Yes                          | Yes                                                       |
| Missing partition conflicts                     | No         | Yes                          | Yes                          | Yes                                                       |
| No need for UPDATE trigger on tables with TOAST | No         | Yes                          | Yes                          | Yes                                                       |
| Automatically hold back freeze                  | No         | Yes                          | Yes                          | Yes                                                       |


To know which plan to get based on your Postgres distribution requirements, see [EDB Subscription Plans Comparison](https://www.enterprisedb.com/products/plans-comparison).
