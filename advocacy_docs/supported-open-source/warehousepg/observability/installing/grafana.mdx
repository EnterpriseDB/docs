Configure Grafana to point to the data sources required for fetching metrics, and load the pre-configured dashboards to visualize the collected data.

## Add data sources

**Prometheus**
- Log in to Grafana with admin privileges.
- Go to **Connections** > **Data Sources** > **Add new data source**.
- Select **Prometheus**.
- Enter the Prometheus server URL (e.g., `http://<prometheus_host>:9090`).
- Select **Save & Test**.

**Loki**
- Log in to Grafana with admin privileges.
- Go to **Connections** > **Data Sources** > **Add new data source**.
- Select **Loki**.
- Enter the Loki server URL (e.g., `http://<loki_host>:3100`).
- Select **Save & Test**.


**Infinity data source for WHPG Exporter**
- Log in to Grafana with admin privileges.
- Navigate to **Connections** > **Add new connection**, search for **Infinity**, and select **Install**.
- Go to **Connections** > **Data Sources** > **Add new data source**.
- Select **Infinity**.
- Enter the Exporter details under **Base URL** (e.g., `http://</whpg_exporter:<9187>/api/v1/query`).
- Select **Save & Test**.


## Import the dashboards

EDB provides a set of JSON configuration files for Grafana. We understand that users have different needs for monitoring. The provided Grafana configurations provide a starting point for customization.

- Download [WarehousePG dashboard JSON files](https://github.com/warehouse-pg/warehouse-pg-grafana-dashboards).
    - **WHPG Cluster Dashboard**
    - **WHPG Cluster Details**
    - **WHPG Log Details**
    - **WHPG Query Details**
    - **WHPG Recommendations**
    - **WHPG Resource Details**
    - **WHPG Segment Details**

- In Grafana, go to **Dashboards**.
- Select the **New** dropdown menu and select **Import**.
- Upload the JSON file or paste its content.
- Select **Import**.
- Repeat the same steps for all the provided dashboards.

## Verify the installation

You should now be able to view the metrics obtained from your WHPG cluster by using the Grafana dashboards.
