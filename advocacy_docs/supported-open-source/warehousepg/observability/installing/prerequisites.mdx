!!! Note
This guide assumes you have installed Grafana, Loki and Prometheus. You can use the following [ready-to run stack](https://github.com/warehouse-pg/warehouse-pg-grafana-dashboards) using Docker Compose to quickly spin up an environment that comprises Grafana, Prometheus, and Loki.

!!!

## Prerequisites

- WarehousePG version 6.x running on RH7 or RH8.
- WarehousePG version 7.x running on RH8 or RH9.
- [Grafana](https://grafana.com/docs/grafana/latest/setup-grafana/installation/) 10.4.8 or later.
- [Loki](https://grafana.com/docs/loki/latest/setup/install/) 3.5 or later.
- [Prometheus](https://prometheus.io/docs/prometheus/latest/installation/) 3.5.0 or later.
- A superuser role to connect to your WHPG cluster from the Exporter host.

## Network requirements

The following table lists the connection requirements among the different components. Note that the ports listed are default vaues, you can customize them according to your environment:

| Source | Destination | Use  
| ------ | ----------- | ---
| WHPG Coordinator | Prometheus:9090 | Push host level metrics
| WHPG Coordinator | Loki:3100       | Push log files
| Exporter         | Prometheus:9090 | Push SQL metrics for historical data
| Grafana          | Exporter:9187   | Expose live query endpoint
| Grafana          | Prometheus:9090 | Display host level metrics
| Grafana          | Loki:3100       | Display log metrics
| User             | Grafana:3000    | Access to Observability
