Install the WarehousePG Enterprise Manager (WEM) package on your designated host.

## Downloading and installing WEM 

1. Configure the EDB repository on your host:

    ```bash
    export EDB_SUBSCRIPTION_TOKEN=<your-token>
    export EDB_REPO=gpsupp
    curl -1sSLf "https://downloads.enterprisedb.com/$EDB_SUBSCRIPTION_TOKEN/$EDB_REPO/setup.rpm.sh" | sudo -E bash
    sudo dnf download whpg-enterprise-manager
    ```

1. Install the WEM package:

    ```bash 
    sudo dnf install -y whpg-enterprise-manager
    ```

## Performing post-installation setup

To run WEM management commands (such as `setup`, `doctor`, or `setup --verify`) from any location in your terminal, add the WEM binary directory to your system `PATH`. 

Update your current session:

```bash
export PATH=/usr/local/greenplum-db/wem:$PATH
```

!!! tip
    To make this change persistent, add the command above to your `~/.bashrc` or `~/.bash_profile` file.

!!! note
    Starting from WarehousePG 7.4, the default WHPG installation directory is `/usr/edb/whpg7`. For backward compatibility, a symlink is provided at `/usr/local/greenplum-db`. The WEM binary path above resolves through this symlink, so no changes are required on WarehousePG 7.4 or later systems.

## Configuring WEM

Configure the WEM service by editing `/etc/wem/wem.conf`. For a complete list of all supported parameters, defaults, and types, see the [Configuration reference](../reference/configuration).

### Target cluster

Set the values of `WHPG_HOST`, `WHPG_PORT`, `WHPG_DATABASE`, `WHPG_USER`, and `WHPG_PASSWORD` as per your WHPG cluster. For example:

```ini
WHPG_HOST=whpg-coordinator-host
WHPG_PORT=5432
WHPG_DATABASE=postgres
WHPG_USER=gpadmin
WHPG_PASSWORD=postgres
```

!!! Note
    You can specify any database on your WHPG cluster. However, `WHPG_USER` must be a superuser.

### Observability endpoints

Set the URLs for your Prometheus and Loki instances. For example:

```ini
PROMETHEUS_URL=http://prometheus.hostname:9090
LOKI_URL=http://loki.hostname:3100
```

### Application state (optional)

By default, WEM stores its internal state in the database defined by `WHPG_DATABASE`. To use a dedicated database for WEM state, configure these optional parameters. For example:

```ini
WEM_HOST=wem-db-hostname
WEM_PORT=5432
WEM_DATABASE=wem
WEM_USER=gpadmin
WEM_PASSWORD=postgres
```

### Alerting

To enable centralized notifications, point WEM to your Alertmanager endpoint:

```ini
ALERTMANAGER_URL=http://prometheus.hostname:9093
```

### AI Assistant

To enable the AI Assistant for query writing and optimization, manually add the `ANTHROPIC_API_KEY` parameter (requires an active Anthropic account).

```ini
ANTHROPIC_API_KEY=YOUR_API_KEY_HERE
```

### Advanced tuning

Optionally, adjust these parameters to fine-tune logging and data intervals:

- `WEM_EXPORTER_LOG_LEVEL`: Sets the verbosity of Exporter logs (`debug`, `info`, `warn`, or `error`).

- `WEM_EXPORTER_REMOTE_WRITE_ENABLED`: Controls whether WEM actively pushes SQL exporter metrics to Prometheus via remote write. Defaults to `true`.
    - Set to `false` when you prefer Prometheus to scrape WEM via the `/prom/metrics` endpoint instead of WEM pushing to Prometheus, or when `PROMETHEUS_URL` isn't set.
    - Setting this parameter to `true` doesn't disable the `/prom/metrics` endpoint. Cluster metrics remain available for Prometheus to scrape on demand.
    - When set to `false`, all `WEM_EXPORTER_*` parameters are ignored.

- `WEM_EXPORTER_REMOTE_WRITE_INTERVAL`: Frequency for pushing metrics. For example, 15s, 1m, 5m.

- `WEM_EXPORTER_REMOTE_WRITE_TIMEOUT`: Maximum time allowed for a data push. For example, 30s, 1m, 2m.

- `LONG_RUNNING_THRESHOLD`: Duration after which a query is flagged as long-running in the **Connection & Query Metrics** section on the **System Metrics** page. For example, 5m, 30m, 1h. This can also be configured directly from the WEM interface.

### Remote write receiver (optional)

WEM can accept inbound metrics from Grafana Alloy or any Prometheus-compatible remote write client. Received time series are stored in memory and exposed at `/prom/metrics` alongside WEM's own cluster metrics.

To enable, set the following parameters:

```ini
WEM_REMOTE_WRITE_RECEIVER_ENABLED=true
WEM_REMOTE_WRITE_RECEIVER_PORT=9091
WEM_REMOTE_WRITE_RECEIVER_TTL=5m
WEM_REMOTE_WRITE_RECEIVER_TOKEN=eU3zV8mK2pL9nQ4rX1yA6wB0cF5hJ7dT
```

- `WEM_REMOTE_WRITE_RECEIVER_ENABLED`: Master switch for the receiver. Defaults to `false`. When `false`, no `/write` endpoint is registered and no additional port is opened. The `/prom/metrics` endpoint remains available regardless. WEM's own cluster metrics are always exposed there for Prometheus to scrape even when the receiver is off.

- `WEM_REMOTE_WRITE_RECEIVER_PORT`: Port on which WEM listens for `POST /write` requests. Defaults to `9091`.
    - **Different from `PORT` (default):** WEM opens a dedicated HTTP listener on this port. Only `/write` and the `/-/healthy` health endpoint are available there. This setup lets you restrict Alloy traffic independently of the main UI port using firewall rules.
    - **Same as `PORT`:** The `/write` route is registered on the main web server and no second listener is opened. Use this in environments where opening an additional port isn't possible.

- `WEM_REMOTE_WRITE_RECEIVER_TTL`: How long WEM retains a received time series in memory after its last sample. When the TTL expires and no new sample has arrived, the series is evicted from `/prom/metrics`. Defaults to `5m`. Set this to at least twice your Alloy `remote_write` interval to avoid gaps during temporary restarts. With Alloy's default 60-second interval, `2m` is the minimum safe value. Accepts `s` (seconds), `m` (minutes), and `h` (hours).

- `WEM_REMOTE_WRITE_RECEIVER_TOKEN` *(sensitive)*: Bearer token that inbound `POST /write` requests must present in the `Authorization` header:

    ```
    Authorization: Bearer <token>
    ```

    Defaults to empty (no authentication). When empty, WEM logs a warning at startup and any client that can reach the receiver port can push metrics. When set, requests without a valid token are rejected with HTTP 401. Generate a token with `openssl rand -base64 32`. You can also pass this value as the `--remote-write-token` CLI flag instead of storing it in `wem.conf`.

Once the receiver is enabled, configure Collector to push metrics to WEM by setting `PROMETHEUS_ENDPOINT` in `/var/lib/whpg-observability-collector/collector.conf` to WEM's `/write` endpoint:

```ini
PROMETHEUS_ENDPOINT="http://<wem-host>:9091/write"
```

Redeploy Collector to apply the change:

```bash
cd /var/lib/whpg-observability/
./deploy-observability
```


### WEM portal access

By default, the WEM web portal listens on port 8080. To use a different port, set the `PORT` parameter. For example:

```ini
PORT=8082
```

### Session security

To start, WEM requires `WEM_COOKIE_SECRET` to be set. It returns an error with setup instructions if it's missing.

- Generate a unique 32-byte key using `openssl rand -base64 32` and set it as the `WEM_COOKIE_SECRET`.
- If you aren't using HTTPS, you must also set `WEM_INSECURE_COOKIES` to `1`.

```ini
WEM_COOKIE_SECRET=your-generated-secret-key
WEM_INSECURE_COOKIES=1
```

### Setting the administrator password

Before initializing WEM, you must establish the initial administrative credentials. Choose one of the following methods:

- **Interactive setup:** Skip editing the configuration file and provide the password directly when running `wem setup` in the following section.

- **Static configuration:** Define the password in the WEM configuration file. This parameter is auto-generated if empty at install.

    ```ini
    WEM_ADMIN_PASSWORD=your-secure-password
    ```

- **File-based configuration:** For enhanced security in automated deployments, point WEM to an external password file.

    1. Create a file containing your password:

        ```bash
       echo "your-secure-password" > /etc/wem/admin.pw
        ```

    1. Manually add the following parameter to the WEM configuration file:
        
        ```ini
        WEM_ADMIN_PASSWORD_FILE=/etc/wem/admin.pw
        ```

!!! Note
    After you initialize the WEM portal and log in for the first time, perform all user management, such as adding users and modifying roles, through the WEM interface.

## Initializing WEM

Initialize the database and internal configurations. Choose the method that fits your deployment style:

- **Manual interactive setup:** Use this method if you want to provide configuration details, such as the admin password, during the process.

    ```shell
    /usr/local/greenplum-db/wem/wem setup
    ```

- **Automated systemd setup:** Use this method for automated installations. When you start the service using `systemd`, the system automatically detects if a setup is required and executes it using the parameters defined in your configuration file.

    ```shell
    systemctl enable wem
    systemctl start wem
    ```

!!! Important
    `wem setup` doesn't read `wem.conf`. When run directly, it uses built-in defaults (host `localhost`, port `5432`, database `wem`) regardless of what's in your configuration file. `wem.conf` is only sourced when WEM starts via systemd.

    To use your configured values during manual setup, pass them explicitly as flags:

    ```shell
    /usr/local/greenplum-db/wem/wem setup --host <WHPG_HOST> --port <WHPG_PORT> --database <WHPG_DATABASE> --user <WHPG_USER>
    ```

    Alternatively, use the automated systemd setup, which reads `wem.conf` automatically.

## Verifying the installation

Verify that the service is active:

```bash
sudo systemctl status wem
```

Use the following diagnostic tools to ensure all components are communicating correctly and the environment is healthy.

- Validate your configuration settings:

    ```bash
    wem setup --verify
    ```
    See [wem setup command reference](../reference/commands#wem-setup) for details.

- Perform a comprehensive check for configuration errors, missing dependencies, or connectivity issues:

    ```bash
    wem doctor
    ```

    See [wem doctor command reference](../reference/commands#wem-doctor) for details.


!!! Note Post-installation changes: 
    To modify your environment after the initial setup, see [Configuring WEM](../configuring).
