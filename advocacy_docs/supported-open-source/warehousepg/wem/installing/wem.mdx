Install the WarehousePG Enterprise Manager (WEM) package on your designated host.

## Downloading and installing WEM 

1. Download the package from the EDB repository:

    ```bash
    export EDB_SUBSCRIPTION_TOKEN=<your-token>
    export EDB_REPO=gpsupp
    curl -1sSLf "https://downloads.enterprisedb.com/$EDB_SUBSCRIPTION_TOKEN/$EDB_REPO/setup.rpm.sh" | sudo -E bash
    sudo dnf download edb-wem
    ```

1. Install WHPG Exporter on the designated host:

    <TabContainer syncKey="install">
    <Tab title="RHEL 8, 9">

    ```bash 
    sudo dnf install -y edb-wem
    ```

    </Tab>
    <Tab title="RHEL 7">

    ```bash 
    sudo yum install -y edb-wem
    ```

    </Tab>
    </TabContainer>

## Configuring WEM

1. Export the Prometheus and Loki endpoints: 

    ```bash
    export PROMETHEUS_URL=
    export LOKI_URL=
    ```

1. Update the configuration file `/etc/wem/wem.conf`:

    ```
    # WarehousePG connection config
    WHPG_HOST=0.0.0.0
    WHPG_PORT=6432
    WHPG_DATABASE=postgres
    WHPG_USER=gpadmin
    WHPG_PASSWORD=postgres

    # web portal config
    WEM_HOST=localhost
    WEM_USER=postgres
    WEM_PASSWORD=postgres
    WEM_PORT=5432

    # Prometheus URL (enables exporter when set, uses PROMETHEUS_URL/api/v1/write for remote write)
    PROMETHEUS_URL="http://localhost:9090"
    LOKI_URL="http://localhost:3100"

    # Exporter Service config (uses WHPG connection settings and PROMETHEUS_URL above)
    # Log level for the exporter (debug, info, warn, error)
    WEM_EXPORTER_LOG_LEVEL=warn
    WEM_EXPORTER_REMOTE_WRITE_INTERVAL=15s
    WEM_EXPORTER_REMOTE_WRITE_TIMEOUT=30s
    # Comma-separated list of database names to monitor by Exporter.
    # If empty, all databases are monitored.
    WEM_EXPORTER_WHPG_DB_LIST=""
    ```

    - Set the values of `WHPG_HOST`, `WHPG_PORT`, `WHPG_DATABASE`, and `WHPG_USER`.
    - Set `WHPG_PASSWORD` with your database password.

1. Enable and start the WEM service:

    ```bash
    systemctl enable wem
    systemctl start wem
    ```

1. Verify WEM is running
