**Release Date:** May 5, 2026

WarehousePG Enterprise Manager (WEM) 1.1 includes new features, enhancements, and bug fixes focused on improving stability and reliability.

### New features

[PTT-881]: #
- **Background Processes tab:** The **Query Monitor** now includes a dedicated **Background Processes** tab that separates internal database background processes from active user queries.

[PTT-923]: #
- **Visual query plan graphs:** The **SQL Editor** now renders a visual graph for static query execution plans, making it easier to understand and optimize query performance.

[PTT-877]: #
- **Remote pg_hba.conf editing:** The **HBA Editor** now supports editing `pg_hba.conf` when WEM runs on a different host than the database server, with the same automatic backup and safe reload behavior as local editing.

[PTT-892]: #
- **Database size growth chart:** A new **Database Size Growth** chart on the **System Metrics** page shows how each database's size has changed over time.

[PTT-896]: #
- **Role management:** You can now create, edit, and delete database roles directly from the **Access Management** page.

[PTT-897]: #
- **Group role membership:** The **Access Management** page now displays the members of each group role.

[PTT-969]: #
- **Build details in version display:** The WEM version display now includes the commit ID and build date alongside the version number.

[PTT-899]: #
- **Configuration parameter source:** The **Settings** tab now includes source and source file columns for each configuration parameter, sorted alphabetically by name.

### Enhancements

[PTT-900]: #
- **Materialized views:** Views and materialized views are now displayed separately in the **Data Analysis** page. Materialized views now include size and storage options.

[PTT-903]: #
[PTT-904]: #
- **Actionable data analysis:** The **Missing Stats** and **Bloat** sections on the **Data Analysis** page now offer direct remediation actions.

[PTT-902]: #
- **Clickable log severity cards:** Clicking a severity count card on the **Logs** page now filters the log list to that severity.

[PTT-981]: #
- **Configurable long-running query threshold:** The **Connection & Query Metrics** section has been improved, and the threshold for flagging long-running queries is now configurable.

[PTT-957]: #
[PTT-944]: #
- **Canary monitoring:** The **Canary** monitoring page is reorganized with tabs and additional admin controls. Canary check updates now use pointer-based partial updates, and system-defined checks are protected from modification.

[PTT-978]: #
- **Sortable Storage Details columns:** Columns in the **Storage Details** table can now be sorted.

[PTT-975]: #
- **Segment Details filters:** The **Cluster Segment Details** table now supports filtering by **Role**, **Status**, and **Hostname**.

[PTT-955]: #
- **Management page permission enforcement:** Non-admin users who navigate to a restricted **Management** page now receive a 403 access denied page. Destructive actions on the **Management** page now prompt for confirmation before proceeding.

[PTT-956]: #
- **Settings form validation and unsaved-changes warning:** The **Settings** tab now validates URL, integer, and duration fields inline. Navigating away or closing the browser with unsaved changes triggers a warning, and changes to critical connectivity settings require explicit confirmation.

[PTT-961]: #
- **Query Monitor session improvements:** The **Sessions** table now displays an **Active** column with a matching filter. Canceling a query is disabled for sessions in an idle state.

[PTT-991]: #
- **Viewer profile defaults:** The **Logs** tab is no longer included in the **Viewer** profile by default.

### Changes

### Security

- Added input validation and injection prevention across API handlers, including PromQL and Loki query parameters, **Alertmanager** silence IDs, EXPLAIN format values, and CSV export cells.
- WEM user passwords now require at least one uppercase letter, lowercase letter, digit, and special character. The cluster segments endpoint, which exposes internal hostnames, ports, and data directories, is now restricted to **Admin** users. Existing scripts or automation that create WEM users with simple passwords will need to be updated.
- WEM now requires the `WEM_COOKIE_SECRET` environment variable to be set, and returns an actionable error if it's missing.
- Debug endpoints are now only registered when `DEBUG` mode is enabled.
- Updated pgx and Prometheus dependencies to address security vulnerabilities.
