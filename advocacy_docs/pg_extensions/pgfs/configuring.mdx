To configure `PGFS`, install the extension using the `CREATE EXTENSION` command:

```sql
edb=# CREATE EXTENSION pgfs;
CREATE EXTENSION
```

You can check whether the extension was installed by running the `\dx` command in psql:

```sql
edb=# \dx
__OUTPUT__
                                     List of installed extensions
       Name       | Version |   Schema   |                        Description
------------------+---------+------------+------------------------------------------------------------
 pgfs             | 1.0.6   | pgfs       | pgfs: enables access to filesystem-like storage locations
```

In addition to the `PGFS` extension listed in the output, other extensions are typically listed as well.

## PGFS components

Once the `PGFS` extension is created, the following components are created or initialized:

### pgfs schema

To keep the database organized, all internal objects related to the file system are grouped under a dedicated `pgfs` schema. This prevents naming conflicts with your existing tables and functions.

### Management functions

A set of SQL functions is added to allow you to manage storage from within the `psql`:

- **`pgfs.create_storage_location()`**: Used to define the connection string and credentials for an S3-comaptible bucket.

- **`pgfs.list_files()`**: Allows you to browse the contents of the connected external storage.

- **`pgfs.read_file()`**: Enables the database to pull specific data into a query.

### Metadata catalog tables

The extension creates internal catalog tables that can help track your storage locations. These tables store:

- **Endpoint URLS**: Where the storage is located. For example, AWS S3, Google Cloud Storage (GCP), or Azure Blob.

- **Authentication config**: References to the keys of IAM roles needed to access the data.

- **Path mapping**: Definitions of which virtual paths in Postgres map to which physical paths in the cloud.

### Integration points for Pipelines

`PGFS` is a dependency for AI Accelerator Pipelines. It creates the hooks necessary for:

- **Volumes**: Virtual storage abstractions that Pipelines use as data sources.

- **Format handlers**: The logic required to identify and process different data formats like `Text` or `Image` stored in the cloud.


For more information on using these components, see [Using PGFS](./using/).
