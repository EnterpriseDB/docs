`tds_fdw` is supported on the same platforms as the Postgres distribution you're using. Support starts with PostgreSQL 9.2. For details, see:

- [EDB Postgres Advanced Server product compatibility](https://www.enterprisedb.com/platform-compatibility#epas)
- [PostgreSQL product compatibility](https://www.enterprisedb.com/resources/platform-compatibility#pg)
- [EDB Postgres Extended Server product compatibility](https://www.enterprisedb.com/resources/platform-compatibility#epas_extended)

## Prerequisites

Before you begin the installation process:

1. Install Postgres. See:

   - [Installing EDB Postgres Advanced Server](/epas/latest/installing/)
   - [Installing PostgreSQL](../../supported-open-source/postgresql/installing/)
   - [Installing EDB Postgres Extended Server](/pge/latest/installing/)

1. Install the FreeTDS library, which `tds_fdw` requires to communicate with TDS databases.

   On RHEL/Rocky Linux and derivatives:

   ```shell
   sudo yum install epel-release
   sudo yum install freetds-devel
   ```

   On Debian and Ubuntu:

   ```shell
   sudo apt-get install freetds-dev
   ```

1. Set up the EDB repository.

   Setting up the repository is a one-time task. If you've already set up your repository, you don't need to perform this step.

   To set up the repository, go to [EDB repositories](https://www.enterprisedb.com/repos-downloads) and follow the instructions provided there.

## Install the package

The syntax for the install command is:

```shell
# For SLES, CentOS, RHEL and its derivatives
sudo <package-manager> -y install edb-<postgres><postgres_version>-tds-fdw

# For Debian and Ubuntu
sudo <package-manager> -y install edb-<postgres><postgres_version>-tds-fdw
```

Where:

- `<package-manager>` is the package manager used with your operating system:

  | Package manager | Operating system                 |
  | --------------- | -------------------------------- |
  | dnf             | RHEL 8/9 and derivatives         |
  | yum             | RHEL 7 and derivatives, CentOS 7 |
  | zypper          | SLES                             |
  | apt-get         | Debian and derivatives           |

- `<postgres>` is the distribution of Postgres you're using:

  | Postgres distribution        | Value            |
  | ---------------------------- | ---------------- |
  | PostgreSQL                   | pg               |
  | EDB Postgres Advanced Server | as               |
  | EDB Postgres Extended Server | postgresextended |

- `<postgres_version>` is the version of Postgres you're using.

## Install from source

If you'd prefer to build `tds_fdw` from source, use the following steps.

1. Install the required build dependencies:

   ```shell
   # On RHEL/Rocky Linux and derivatives
   sudo yum install clang llvm make redhat-rpm-config wget freetds-devel

   # On Debian/Ubuntu
   sudo apt-get install build-essential libfreетds-dev postgresql-server-dev-all
   ```

1. Download the release package:

   ```shell
   export TDS_FDW_VERSION="2.0.5"
   wget https://github.com/tds-fdw/tds_fdw/archive/v${TDS_FDW_VERSION}.tar.gz
   tar -xvzf v${TDS_FDW_VERSION}.tar.gz
   cd tds_fdw-${TDS_FDW_VERSION}
   ```

1. Build and install:

   ```shell
   make USE_PGXS=1
   sudo make USE_PGXS=1 install
   ```

   If you have multiple PostgreSQL versions installed and want to target a specific one, set `PG_CONFIG` to the full path of the `pg_config` binary for that version:

   ```shell
   make USE_PGXS=1 PG_CONFIG=/usr/pgsql-17/bin/pg_config
   sudo make USE_PGXS=1 PG_CONFIG=/usr/pgsql-17/bin/pg_config install
   ```
