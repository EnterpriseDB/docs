## Create the extension

After installing the package, connect to your database and create the extension:

```sql
CREATE EXTENSION tds_fdw;
```

## Configure FreeTDS

`tds_fdw` uses FreeTDS to connect to TDS databases. You can configure connection parameters either directly in the foreign server definition or in the FreeTDS configuration file `freetds.conf`.

To configure a named server entry in `freetds.conf`:

```ini
[mssql_server]
    host = 192.168.1.100
    port = 1433
    tds version = 7.4
    client charset = UTF-8
```

You can then refer to this entry by its section name (`mssql_server`) as the `servername` option in your foreign server definition.

For more information on FreeTDS configuration, see the [FreeTDS documentation](https://www.freetds.org/userguide/freetdsconf.html).

## Character sets and encoding

If you encounter errors like the following when working with Unicode data on Microsoft SQL Server:

```
NOTICE: DB-Library notice: Msg #: 4004, Msg state: 1, Msg: Unicode data in a
Unicode-only collation or ntext data cannot be sent to clients using DB-Library
```

You may need to set `tds version` to `7.0` or higher in `freetds.conf`. See [Choosing a TDS protocol version](https://www.freetds.org/userguide/ChoosingTdsProtocol.html).

To set the client character set, set `client charset` in `freetds.conf`. See [Localization and TDS 7.0](https://www.freetds.org/userguide/Localization.html).

## Encrypted connections to Microsoft SQL Server

Encrypted connections are handled by FreeTDS. To enable encryption, configure the `encryption` setting in `freetds.conf`. See the [FreeTDS freetds.conf reference](https://www.freetds.org/userguide/freetdsconf.html) for details.

## Create a foreign server

Create a foreign server definition to represent the remote TDS database:

```sql
CREATE SERVER mssql_svr
    FOREIGN DATA WRAPPER tds_fdw
    OPTIONS (servername '127.0.0.1', port '1433', database 'mydb', tds_version '7.4');
```

## Create a user mapping

Create a user mapping to associate a local PostgreSQL role with credentials on the remote server:

```sql
CREATE USER MAPPING FOR postgres
    SERVER mssql_svr
    OPTIONS (username 'sa', password 'yourpassword');
```

### User mapping options

| Option | Required | Description |
| ------ | -------- | ----------- |
| `username` | Yes | The username on the remote server. For Azure SQL Database, use the format `username@servername`. |
| `password` | Yes | The password for the remote account. |
